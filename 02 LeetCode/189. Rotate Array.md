> [!warning] Medium

> [!note] Task
> Given an integer array `nums`, rotate the array to the right by `k` steps, where `k` is non-negative.
> 
> **Example 1:**
> 
> **Input:** nums = [1,2,3,4,5,6,7], k = 3
> **Output:** [5,6,7,1,2,3,4]
> **Explanation:**
> rotate 1 steps to the right: [7,1,2,3,4,5,6]
> rotate 2 steps to the right: [6,7,1,2,3,4,5]
> rotate 3 steps to the right: [5,6,7,1,2,3,4]
> 
> **Example 2:**
> 
> **Input:** nums = [-1,-100,3,99], k = 2
> **Output:** [3,99,-1,-100]
> **Explanation:** 
> rotate 1 steps to the right: [99,-1,-100,3]
> rotate 2 steps to the right: [3,99,-1,-100]
> 
> **Constraints:**
> 
> - `1 <= nums.length <= 105`
> - `-231 <= nums[i] <= 231 - 1`
> - `0 <= k <= 105`
> 
> **Follow up:**
> 
> - Try to come up with as many solutions as you can. There are at least **three** different ways to solve this problem.
> - Could you do it in-place with `O(1)` extra space?

---
# Solutions:
First solution (slower):
```java
class Solution {
    public void rotate(int[] nums, int k) {
        int n = nums.length;
        int[] rotated = new int[n];

        for(int i = 0;i < n;i++){
            rotated[(i + k) % n] = nums[i];
        }

        for(int i = 0;i < n;i++){
            nums[i] = rotated[i];
        }
    }
}
```

Second solution (O(1) space):
```java
class Solution {
    public void rotate(int[] nums, int k) {
        int n = nums.length;
        k = k % n;
        reverse(nums, 0, n - 1); //whole array
        reverse(nums, 0, k - 1); //first half
        reverse(nums, k, n - 1); //second half
    }
    public void reverse(int arr[], int start, int end){
        while(start < end){
            int temp = arr[start];
            arr[start] = arr[end];
            arr[end] = temp;
            start++; 
            end--;
        }
    }
}
```

> [!warning] Нащо саме **k = k % n**
> Якщо довжина масиву $n = 5$, а тебе попросять повернути його на $k = 100$ позицій, і ти не зробиш `k = k % n`, то в наступному рядку коду станеться біда:
> 
> `reverse(nums, 0, k - 1);` — перетвориться на `reverse(nums, 0, 99);`
> 
> Програма спробує знайти елемент під номером **99** у масиві, де їх всього **5**. Вона просто "впаде" з помилкою, бо такого індексу не існує.
