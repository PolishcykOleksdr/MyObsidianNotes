> Java IO (Input/Output) — це початковий API для роботи з операціями введення-виведення, що базується на потоках байтів та символів. З часом він був доповнений Java NIO (New IO) та NIO.2 для забезпечення неблокуючого та асинхронного доступу до ресурсів, що критично для продуктивності сучасних систем.

---

> [!abstract] Core Framework 
> Колекції та структури введення-виведення в Java реалізують механізми передачі даних між програмою та зовнішніми джерелами:
> 
> - **Java IO (java.io):** Орієнтований на потоки (Stream-oriented). Працює з `File`, `InputStream` та `OutputStream`.
>     
> - **Java NIO (java.nio):** Орієнтований на буфери та канали (Buffer-oriented). Запроваджений у Java 1.4 для неблокуючого IO.
>     
> - **NIO.2 (java.nio.file):** Запроваджений у Java 1.7. Додає асинхронність та високорівневі об'єкти: `Path`, `Files`, `Paths`.
>     

---

> [!info] Key Insights
> 
> - **Блокування (Blocking):** У Java IO потік виконання блокується на методах `read()` або `write()`, доки операція не завершиться. Потік не може виконувати іншу роботу в цей час.
>     
> - **Неблокуючий режим (Non-blocking):** У Java NIO потік може запитати дані з каналу і отримати тільки те, що є в наявності (або нічого). Це дозволяє потоку переключатися на інші задачі.
>     
> - **Орієнтація на дані:** IO зчитує по одному або кілька байтів за раз безпосередньо з потоку. NIO використовує буферизацію — зчитування даних невеликими фрагментами.
>     
> - **Мультиплексування:** NIO дозволяє використовувати `SelectableChannel` для моніторингу багатьох каналів одним потоком.
>     

---

> [!example] Практичне застосування
> 
> **Java IO: Робота з потоками (InputStream/OutputStream)** Призначені для доставки інформації з кроком в один байт. Мають зручні оболонки `Reader` та `Writer`.
> 
> **Java NIO: Канали та Буфери** Канал виступає як середовище зв’язку із зовнішнім світом.
> 
> **NIO.2: Високорівневі маніпуляції** Використання `Interface Path` та `Class Files` для полегшення роботи з файловою системою.

---

> [!warning] Обмеження та ризики
> 
> - **Режим очікування в IO:** Використання режиму блокування в `java.io` забезпечує отримання всього повідомлення перед продовженням, але це зупиняє потік ("заморожує" процес).
>     
> - **Складність NIO:** Перехід на неблокуючу модель та роботу з селекторами/буферами вимагає складнішої логіки управління станом даних порівняно з лінійним читанням у IO.
>     

---

> [!tip] Порівняльна характеристика **Java IO (java.io) ✅:**
> 
> - Простота реалізації для послідовного читання.
>     
> - Автоматичне блокування до завершення операції (гарантія отримання даних).
>     
> - Підтримка `Reader`/`Writer` для символьних даних.
>     
> 
> **Java NIO (java.nio) ✅:**
> 
> - Підтримка мультиплексування (`SelectableChannel`).
>     
> - Використання `CharsetDecoder` для конвертації символів у байти.
>     
> - Ефективність при великій кількості з'єднань завдяки неблокуючому режиму.
>     
> - Асинхронний підхід (у NIO.2).
>
