> Java NIO базується на взаємодії **Buffers** (контейнери для даних) та **Channels** (шлюзи для передачі). На відміну від стандартного IO, де робота йде безпосередньо з потоками, у NIO дані завжди зчитуються з каналу в буфер або записуються з буфера в канал.

---

> [!abstract] Core Framework
> 
> Робота з файловою системою в NIO реалізується через дві ключові сутності:
> 
> - **Buffer:** Фіксований масив пам'яті для тимчасового зберігання примітивних даних. Кожен тип має свій клас (наприклад, `ByteBuffer`, `IntBuffer`).
>     
> - **Channel:** Двонаправлене з'єднання з файлом, яке підтримує неблокуючі операції. Основний клас для файлів — `FileChannel`.
>     

---

> [!info] Key Insights
> 
> Стан буфера визначається чотирма показниками:
> 
> 1. **Capacity (Ємність):** Максимальна кількість елементів. Встановлюється при створенні і $не змінюється$.
>     
> 2. **Limit (Ліміт):** Індекс першого елемента, який **не можна** читати/записувати (межа активних даних).
>     
> 3. **Position (Позиція):** Індекс наступного елемента для операції. Починається з $0$.
>     
> 4. **Mark (Позначка):** Запам'ятована позиція для повернення.
>     
> 
> **Ключові стани:**
> 
> - **Flip():** Переводить буфер з режиму запису в режим читання.
>     
>     - `limit = position`
>         
>     - `position = 0`
>         
> - **Clear():** Готує буфер для нового запису (дані не видаляються, але показники скидаються).
>     
>     - `position = 0`
>         
>     - `limit = capacity`
>

> [!example] Практичне застосування
> 
> **Типи буферів:**
> 
> Для кожного примітивного типу існує свій клас: `ByteBuffer`, `CharBuffer`, `DoubleBuffer`, `FloatBuffer`, `IntBuffer`, `LongBuffer`, `ShortBuffer`. Найчастіше використовується `ByteBuffer`.
> 
> **Робота з FileChannel та ByteBuffer:**
> 
> Канали, на відміну від потоків IO, є двонаправленими — один і той самий канал може і читати, і записувати.
> 
> Java
> 
> ```java
> import java.nio.ByteBuffer;
> import java.nio.channels.FileChannel;
> import java.io.RandomAccessFile;
> 
> public class NioExample {
>     public static void main(String[] args) throws Exception {
>         /* Створення каналу через RandomAccessFile */
>         RandomAccessFile aFile = new RandomAccessFile("data.txt", "rw");
>         FileChannel inChannel = aFile.getChannel();
> 
>         /* Створення буфера з фіксованою ємністю (Capacity) */
>         ByteBuffer buf = ByteBuffer.allocate(48);
> 
>         /* Читання з каналу в буфер (запис у буфер) */
>         int bytesRead = inChannel.read(buf);
>         
>         while (bytesRead != -1) {
>             /* Flip: перемикаємо буфер на режим читання (limit = position, position = 0) */
>             buf.flip();
> 
>             while(buf.hasRemaining()){
>                 /* Читання з буфера по одному байту */
>                 System.out.print((char) buf.get());
>             }
> 
>             /* Clear: готуємо буфер для наступного запису (position = 0, limit = capacity) */
>             buf.clear();
>             bytesRead = inChannel.read(buf);
>         }
>         aFile.close();
>     }
> }
> ```

---

> [!warning] Обмеження та ризики
> 
> - **Фіксований розмір:** Буфер має обмежену ємність. Якщо дані перевищують `capacity`, їх потрібно зчитувати ітеративно через цикл з використанням `clear()`.
>     
> - **Забутий flip():** Найпоширеніша помилка — спроба читання з буфера без виклику `flip()`. У цьому випадку програма намагатиметься читати порожні комірки після записаних даних.
>     
> - **Ресурсний менеджмент:** Канали (`FileChannel`) — це відкриті з'єднання, які потребують обов'язкового закриття для уникнення витоку ресурсів.
>     

---

> [!tip] Чому саме NIO Канали?
> 
> - **Двонаправленість:** Один канал замінює і `InputStream`, і `OutputStream`.
>     
> - **Ефективність:** Можливість виконувати неблокуючі операції.
>     
> - **Буферизація:** Дані обробляються частинами, що оптимізує роботу з пам'яттю при великих обсягах файлів.
>
