> **Observer** — поведінковий патерн, що визначає залежність типу «один-до-багатьох» між об'єктами: коли стан одного об'єкта змінюється, всі залежні від нього об'єкти автоматично отримують сповіщення та оновлюються.

---

> [!abstract] Core Framework 
> **Патерн Observer будується на трьох ролях:**
> 
> - **Subject (Видавець)** — зберігає список спостерігачів, надає методи підписки та відписки, сповіщає всіх при зміні стану.
> - **Observer (Спостерігач)** — інтерфейс із методом `update()`, який викликається видавцем.
> - **ConcreteObserver** — конкретна реалізація спостерігача, що реагує на зміни.

---

> [!info] Key Insights
> 
> - **Слабка зв'язаність.** Видавець знає лише про інтерфейс `Observer`, а не про конкретні класи підписників. Нові спостерігачі можна додавати без зміни коду видавця.
>     
> - **Відношення один-до-багатьох.** Один видавець може мати довільну кількість підписників — список `observers` розширюється динамічно через `addObserver()`.
>     
> - **Push-модель.** У цьому прикладі видавець сам передає дані (`news`) у метод `update()` — спостерігачу не потрібно нічого запитувати.
>     
> - **Порядок сповіщень.** Спостерігачі отримують `update()` у тому порядку, в якому були додані до списку. Не покладайся на цей порядок у бізнес-логіці.
>     

---

> [!example] Практичне застосування
> 
> **Підписка на новини** — один видавець, багато підписників:
> 
> - `NewsPublisher` — видавець, що зберігає список `Observer`
> - `ConcreteObserver` — конкретний підписник з іменем
> - При виклику `publishNews()` усі підписники отримують повідомлення через `update()`

---

## Реалізація

```java
import java.util.ArrayList;
import java.util.List;

// Інтерфейс спостерігача
interface Observer {
    void update(String news);
}

// Видавець новин
class NewsPublisher {
    private List<Observer> observers = new ArrayList<>();

    // Додати спостерігача
    void addObserver(Observer observer) {
        observers.add(observer);
    }

    // Опублікувати новину
    void publishNews(String news) {
        System.out.println("Новина оприлюднена: " + news);
        notifyObservers(news);
    }

    // Повідомити всіх спостерігачів про новину
    private void notifyObservers(String news) {
        for (Observer observer : observers) {
            observer.update(news);
        }
    }
}

// Конкретний спостерігач
class ConcreteObserver implements Observer {
    private String name;

    ConcreteObserver(String name) {
        this.name = name;
    }

    @Override
    public void update(String news) {
        System.out.println(name + " отримав нову новину: " + news);
    }
}

public class Main {
    public static void main(String[] args) {
        // Створення видавця новин
        NewsPublisher publisher = new NewsPublisher();

        // Створення спостерігачів
        Observer observer1 = new ConcreteObserver("Користувач 1");
        Observer observer2 = new ConcreteObserver("Користувач 2");

        // Додавання спостерігачів до списку підписників
        publisher.addObserver(observer1);
        publisher.addObserver(observer2);

        // Оприлюднення новини
        publisher.publishNews("У Києві відбудеться концерт.");
    }
}

/* Результат:
   Новина оприлюднена: У Києві відбудеться концерт.
   Користувач 1 отримав нову новину: У Києві відбудеться концерт.
   Користувач 2 отримав нову новину: У Києві відбудеться концерт.
*/
```

---

> [!warning] Обмеження та ризики
> 
> - **Витік пам'яті.** Якщо спостерігач не відписується (`removeObserver`), видавець тримає сильне посилання — GC не може зібрати об'єкт. Завжди реалізовуй механізм відписки.
>     
> - **Непередбачуваний порядок.** Порядок виклику `update()` залежить від порядку реєстрації у списку — ніколи не покладайся на нього.
>     
> - **Каскадні оновлення.** Якщо спостерігач у своєму `update()` також змінює стан видавця, можна отримати нескінченний цикл сповіщень.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Існує відношення «один-до-багатьох» між об'єктами
> - Кількість підписників динамічно змінюється під час виконання
> - Потрібна слабка зв'язаність між видавцем та підписниками
> 
> **Ні** ❌:
> 
> - Є лише один підписник, що ніколи не зміниться
> - Важливий строгий порядок обробки подій
> - Підписники мають залежати один від одного
