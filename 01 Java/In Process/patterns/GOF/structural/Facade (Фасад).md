> **Фасад (Facade)** — це **структурний патерн**, який надає **спрощений єдиний інтерфейс** до складної підсистеми або набору класів, приховуючи їхню внутрішню складність від клієнтського коду.

---

> [!abstract] Core Framework 
> **Facade виступає єдиною точкою входу до підсистеми:**
> 
> - **Subsystem classes** — складні класи з власною логікою, про яку клієнт не повинен знати (`CPU`, `Memory`, `HardDrive`);
> - **Facade** — клас, що знає про всі підсистеми, координує їх роботу і надає простий API (`ComputerFacade`);
> - **Client** — взаємодіє лише з Facade, не звертаючись до підсистем напряму.

---

> [!info] Key Insights
> 
> - **Ізоляція складності.** Клієнт викликає один метод на Facade, а той сам координує потрібну послідовність операцій між кількома підсистемами.
>     
> - **Підсистеми залишаються доступними.** Facade не закриває підсистеми повністю — за потреби клієнт може звернутися до них напряму.
>     
> - **Зменшення зв'язності.** Клієнтський код залежить лише від Facade, а не від усіх класів підсистеми, що спрощує підтримку і тестування.
>     

---

> [!example] Практичне застосування
> 
> **Структура патерну:**
> 
> - `CPU` — підсистема з методами `start()` і `execute()`;
> - `Memory` — підсистема з методом `load()`;
> - `HardDrive` — підсистема з методом `readData()`;
> - `ComputerFacade` — фасад, що у методі `start()` оркеструє правильну послідовність викликів усіх підсистем.

```java
// Складна підсистема
class CPU {
    public void start() {
        System.out.println("CPU: запуск обчислювального процесу");
    }

    public void execute() {
        System.out.println("CPU: виконання обчислювальних операцій");
    }
}

// Складна підсистема
class Memory {
    public void load() {
        System.out.println("Memory: завантаження даних в пам'ять");
    }
}

// Складна підсистема
class HardDrive {
    public void readData() {
        System.out.println("HardDrive: читання даних з жорсткого диска");
    }
}

// Фасад, який надає простий інтерфейс до складних підсистем
class ComputerFacade {
    private CPU cpu;
    private Memory memory;
    private HardDrive hardDrive;

    public ComputerFacade() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.hardDrive = new HardDrive();
    }

    public void start() {
        cpu.start();
        memory.load();
        hardDrive.readData();
        cpu.execute();
    }
}

public class Main {
    public static void main(String[] args) {
        // Використання фасаду для запуску комп'ютера
        ComputerFacade computer = new ComputerFacade();
        computer.start();
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Ризик перетворення на «Бога».** Якщо Facade починає містити бізнес-логіку замість простої оркестрації — він стає надмірно відповідальним класом.
>     
> - **Неповне приховування.** Facade не забороняє прямий доступ до підсистем, тому дисципліна використання залежить від команди розробників.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Потрібно надати простий інтерфейс до складної або заплутаної підсистеми
> - Потрібно зменшити кількість залежностей між клієнтом і підсистемою
> - Потрібно розбити систему на шари і визначити точки входу до кожного
> 
> **Ні** ❌:
> 
> - Підсистема проста і не потребує спрощення
> - Клієнту все одно потрібен тонкий контроль над підсистемою — Facade лише заважатиме
