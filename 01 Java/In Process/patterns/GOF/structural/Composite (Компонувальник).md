> **Компонувальник (Composite)** — це **структурний патерн**, який дозволяє компонувати об'єкти у **деревоподібні структури** для представлення ієрархій «частина–ціле» і працювати з окремими об'єктами та їх групами **однаковим чином**.

---

> [!abstract] Core Framework 
> **Composite будує дерево з двох типів вузлів через єдиний інтерфейс:**
> 
> - **Component** — спільний інтерфейс для листків і складових об'єктів (`Graphic`);
> - **Leaf (листок)** — елементарний об'єкт без дочірніх елементів (`Line`);
> - **Composite (складовий)** — об'єкт, що може містити колекцію дочірніх компонентів і делегує їм виклики (`Picture`).

---

> [!info] Key Insights
> 
> - **Уніфікований інтерфейс.** Клієнт викликає `draw()` однаково і на листку, і на цілому дереві — він не знає і не повинен знати, з яким типом взаємодіє.
>     
> - **Рекурсивна структура.** Складовий об'єкт (`Picture`) може містити інші `Picture` як дочірні елементи, утворюючи дерево довільної глибини.
>     
> - **Делегування вниз по дереву.** При виклику методу на Composite він автоматично обходить усіх дочірніх і делегує їм виклик.
>     

---

> [!example] Практичне застосування
> 
> **Структура патерну:**
> 
> - `Graphic` — компонент-інтерфейс із методом `draw()`;
> - `Line` — листок, що реалізує `draw()` безпосередньо;
> - `Picture` — складовий об'єкт, що зберігає список `Graphic` і делегує `draw()` кожному з них.

```java
import java.util.ArrayList;
import java.util.List;

// Компонент
interface Graphic {
    void draw();
}

// Листок
class Line implements Graphic {
    @Override
    public void draw() {
        System.out.println("Відображення лінії");
    }
}

// Складовий об'єкт
class Picture implements Graphic {
    private List<Graphic> graphics = new ArrayList<>();

    // Додати графічний об'єкт в групу
    public void add(Graphic graphic) {
        graphics.add(graphic);
    }

    @Override
    public void draw() {
        System.out.println("Відображення картини:");
        for (Graphic graphic : graphics) {
            graphic.draw(); // Виклик методу відображення для кожного графічного об'єкта
        }
    }
}

public class Main {
    public static void main(String[] args) {
        // Створення складової картини
        Picture picture = new Picture();
        picture.add(new Line()); // Додати лінію до картини
        picture.add(new Line()); // Додати ще одну лінію до картини

        // Відображення картини
        picture.draw();
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Надмірна узагальненість інтерфейсу.** Спільний інтерфейс для листків і Composite може змусити листки реалізовувати методи керування дочірніми елементами (`add`, `remove`), які їм не потрібні.
>     
> - **Складність обмежень типів.** Якщо певні вузли дерева мають приймати лише конкретні типи дочірніх елементів — це доведеться перевіряти вручну в рантаймі.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Потрібно представити ієрархію об'єктів типу «частина–ціле» (файлова система, UI-дерево, організаційна структура)
> - Клієнтський код має однаково обробляти окремі елементи і їх групи
> - Потрібна рекурсивна структура довільної глибини
> 
> **Ні** ❌:
> 
> - Ієрархія має лише один рівень і не передбачає вкладеності
> - Листки і Composite мають принципово різну поведінку, яку не можна об'єднати в один інтерфейс
