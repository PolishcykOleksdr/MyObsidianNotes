> **Абстрактна фабрика (Abstract Factory)** — це **породжувальний патерн**, який надає інтерфейс для створення **сімейств пов'язаних об'єктів** без прив'язки до їх конкретних класів. Дозволяє замінювати цілі родини продуктів, не змінюючи клієнтський код.

---

> [!abstract] Core Framework 
> **Abstract Factory вирішує задачу створення груп взаємопов'язаних об'єктів:**
> 
> - Оголошується **абстрактна фабрика** з набором методів для створення кожного типу продукту;
> - **Конкретні фабрики** реалізують цей інтерфейс і створюють певне сімейство об'єктів;
> - Клієнт працює виключно через **абстрактний інтерфейс** фабрики та продуктів.

---

> [!info] Key Insights
> 
> - **Сімейство продуктів.** На відміну від Factory Method, Abstract Factory відповідає одразу за створення кількох пов'язаних об'єктів (наприклад, наземний і повітряний транспорт разом).
>     
> - **Заміна родини об'єктів.** Щоб переключитися на інший набір продуктів, достатньо підставити іншу конкретну фабрику — клієнтський код не змінюється.
>     
> - **Гарантія сумісності.** Об'єкти, створені однією фабрикою, гарантовано сумісні між собою, оскільки належать до одного сімейства.
>     

---

> [!example] Практичне застосування
> 
> **Структура патерну:**
> 
> - `TransportFactory` — абстрактна фабрика з методами `createLandTransport()` та `createAirTransport()`;
> - `RoadAndAirTransportFactory` — конкретна фабрика, яка створює `Car` та `Plane`;
> - `Transport` — інтерфейс продукту;
> - `Car`, `Plane` — конкретні реалізації продуктів одного сімейства.

```java
// Абстрактний клас фабрики транспортних засобів
interface TransportFactory {
    Transport createLandTransport();
    Transport createAirTransport();
}

// Конкретна реалізація абстрактної фабрики для створення дорожніх та повітряних транспортних засобів
class RoadAndAirTransportFactory implements TransportFactory {
    @Override
    public Transport createLandTransport() {
        return new Car();
    }

    @Override
    public Transport createAirTransport() {
        return new Plane();
    }
}

// Базовий клас для всіх видів транспортних засобів
interface Transport {
    void deliver();
}

// Конкретний клас автомобіля
class Car implements Transport {
    @Override
    public void deliver() {
        System.out.println("Доставка автомобілем");
    }
}

// Конкретний клас літака
class Plane implements Transport {
    @Override
    public void deliver() {
        System.out.println("Доставка літаком");
    }
}

public class Main {
    public static void main(String[] args) {
        // Створення фабрики для дорожніх та повітряних транспортних засобів
        TransportFactory factory = new RoadAndAirTransportFactory();

        // Створення дорожнього транспорту через фабрику
        Transport landTransport = factory.createLandTransport();
        landTransport.deliver(); // Виведе: Доставка автомобілем

        // Створення повітряного транспорту через фабрику
        Transport airTransport = factory.createAirTransport();
        airTransport.deliver(); // Виведе: Доставка літаком
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Складність розширення продуктів.** Якщо потрібно додати новий тип продукту до сімейства — доведеться змінити абстрактний інтерфейс фабрики і всі її конкретні реалізації.
>     
> - **Велика кількість класів.** Кожне нове сімейство продуктів потребує окремої конкретної фабрики, що збільшує кількість класів у системі.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Система має працювати з кількома сімействами пов'язаних продуктів
> - Потрібно забезпечити сумісність об'єктів усередині одного сімейства
> - Потрібна можливість легко переключатися між родинами продуктів
> 
> **Ні** ❌:
> 
> - Потрібно створювати лише один тип об'єктів (достатньо Factory Method)
> - Сімейства продуктів не плануються і не змінюватимуться
