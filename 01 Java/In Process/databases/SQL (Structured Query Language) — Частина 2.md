> **Основні DML-команди SQL** — `SELECT`, `INSERT`, `UPDATE`, `DELETE` — забезпечують повний цикл роботи з даними в реляційній базі даних: від вибірки до видалення.

---

> [!abstract] Core Framework 
> **Чотири основні DML-команди та їхнє призначення:**
> 
> - **SELECT** — вибірка даних з однієї або кількох таблиць; підтримує фільтрацію через `WHERE` та вибір конкретних стовпців або `*` для всіх;
> - **INSERT** — додавання нових рядків до таблиці з явним зазначенням значень для кожного стовпця;
> - **UPDATE** — зміна значень в існуючих рядках відповідно до умов `WHERE`;
> - **DELETE** — видалення рядків, що відповідають умові `WHERE`.

---

> [!info] Key Insights
> 
> - **SELECT — найбільш вживана команда.** Використовується для будь-якого читання даних. Без `WHERE` повертає всі рядки таблиці.
>     
> - **INSERT вставляє один або кілька рядків одночасно.** Порядок стовпців у запиті має відповідати порядку значень у `VALUES`.
>     
> - **UPDATE без WHERE — небезпечна операція.** Без умови фільтрації оновлюються всі рядки таблиці.
>     
> - **DELETE без WHERE видаляє всі рядки.** Таблиця залишається, але стає порожньою — на відміну від `DROP TABLE`, яка видаляє саму таблицю.
>     

---

> [!example] Практичне застосування

## 1. Створення таблиці `Students`

```sql
CREATE TABLE Students (
    StudentID INT,
    FirstName VARCHAR(50),
    LastName  VARCHAR(50),
    Age       INT,
    GPA       DECIMAL(3, 2)
);
```

Результат — порожня таблиця зі схемою:

|StudentID|FirstName|LastName|Age|GPA|
|---|---|---|---|---|

---

## 2. Вставка даних у таблицю

```sql
INSERT INTO Students (StudentID, FirstName, LastName, Age, GPA)
VALUES
    (1, 'John',    'Doe',     22, 3.5),
    (2, 'Alice',   'Smith',   21, 3.8),
    (3, 'Michael', 'Johnson', 23, 3.2);
```

Результат:

|StudentID|FirstName|LastName|Age|GPA|
|---|---|---|---|---|
|1|John|Doe|22|3.5|
|2|Alice|Smith|21|3.8|
|3|Michael|Johnson|23|3.2|

---

## 3. Вибірка всіх записів

```sql
SELECT * FROM Students;
```

Результат — всі рядки таблиці:

|StudentID|FirstName|LastName|Age|GPA|
|---|---|---|---|---|
|1|John|Doe|22|3.5|
|2|Alice|Smith|21|3.8|
|3|Michael|Johnson|23|3.2|

---

## 4. Вибірка з умовою фільтрації

```sql
SELECT * FROM Students WHERE Age > 21;
```

Результат — лише студенти, старші за 21 рік:

|StudentID|FirstName|LastName|Age|GPA|
|---|---|---|---|---|
|1|John|Doe|22|3.5|
|3|Michael|Johnson|23|3.2|

---

## 5. Оновлення запису

```sql
UPDATE Students SET GPA = 4.0 WHERE StudentID = 2;
```

Результат — середній бал студента з `ID = 2` змінено на `4.0`:

|StudentID|FirstName|LastName|Age|GPA|
|---|---|---|---|---|
|1|John|Doe|22|3.5|
|2|Alice|Smith|21|4.0|
|3|Michael|Johnson|23|3.2|

---

## 6. Видалення запису

```sql
DELETE FROM Students WHERE StudentID = 1;
```

Результат — студент з `ID = 1` видалений:

|StudentID|FirstName|LastName|Age|GPA|
|---|---|---|---|---|
|2|Alice|Smith|21|4.0|
|3|Michael|Johnson|23|3.2|

---

> [!warning] Обмеження та ризики
> 
> - **`UPDATE` без `WHERE`.** Оновить кожен рядок таблиці — найпоширеніша помилка при роботі з продакшн-даними. Завжди перевіряй умову `WHERE` перед виконанням.
>     
> - **`DELETE` без `WHERE`.** Видалить усі рядки таблиці без можливості відновлення (якщо транзакція вже завершена через `COMMIT`).
>     
> - **`SELECT *` у продакшн-коді.** Повертає всі стовпці, включно з непотрібними, що збільшує навантаження на мережу та пам'ять. Краще явно вказувати потрібні стовпці.
>     
> - **Відсутність перевірки типів у `INSERT`.** Спроба вставити рядок у числовий стовпець або значення, що порушує обмеження, призведе до помилки виконання.
>     

---

> [!tip] Коли використовувати кожну команду
> 
> **SELECT** ✅ — будь-яке читання та аналіз даних; з `WHERE` для точної фільтрації.
> 
> **INSERT** ✅ — додавання нових записів; підтримує масове додавання кількох рядків одним запитом.
> 
> **UPDATE** ✅ — зміна існуючих даних; **завжди** використовуй `WHERE` для точного вибору рядків.
> 
> **DELETE** ✅ — видалення застарілих або некоректних записів; **завжди** використовуй `WHERE`; для повного очищення таблиці розглянь `TRUNCATE` як альтернативу.
