> **DDL (Data Definition Language)** та **DML (Data Manipulation Language)** — дві ключові підмови SQL. DDL визначає та модифікує структуру бази даних, DML керує самими даними всередині цієї структури.

---

> [!abstract] Core Framework 
> **DDL — операції над структурою:**
> 
> - `CREATE` — створення нових об'єктів (таблиці, індекси, представлення);
> - `ALTER` — модифікація існуючих об'єктів;
> - `DROP` — повне видалення об'єктів разом з даними;
> - `TRUNCATE` — очищення всіх рядків таблиці зі збереженням структури;
> - `RENAME` — перейменування об'єктів;
> - `COMMENT` — додавання коментарів до об'єктів для документації.
> 
> **DML — операції над даними:**
> 
> - `INSERT` — вставка нових записів;
> - `UPDATE` — оновлення існуючих записів;
> - `DELETE` — видалення записів;
> - `SELECT` — вибірка даних.

---

> [!info] Key Insights
> 
> - **`DROP` vs `TRUNCATE` vs `DELETE`.** `DROP` видаляє саму таблицю разом зі структурою. `TRUNCATE` очищає всі рядки, але зберігає структуру — і є швидшим за `DELETE`, оскільки виконується як одна операція. `DELETE` видаляє рядки по одному з урахуванням `WHERE`.
>     
> - **`TRUNCATE` і зовнішні ключі.** `TRUNCATE` не виконається, якщо таблиця має залежні таблиці або зовнішні ключі — необхідно спочатку видалити залежності.
>     
> - **`RENAME` через `ALTER TABLE`.** У PostgreSQL перейменування таблиці виконується командою `ALTER TABLE ... RENAME TO`, а не окремою командою `RENAME`.
>     
> - **`COMMENT` — інструмент документації.** Коментарі до таблиць та стовпців зберігаються в метаданих бази даних та полегшують розуміння структури у великих проєктах.
>     
> - **`SERIAL` у PostgreSQL.** Тип `SERIAL` для стовпця `product_id` автоматично створює послідовність (sequence) та робить поле автоінкрементним — аналог `AUTO_INCREMENT` у MySQL.
>

> [!example] Практичне застосування

## DDL — операції над структурою

### 1. CREATE — створення таблиці

Створення таблиці `products` для зберігання інформації про продукти.

```sql
CREATE TABLE products (
    product_id   SERIAL PRIMARY KEY,
    product_name VARCHAR(255) NOT NULL,
    price        DECIMAL(10, 2) NOT NULL,
    quantity     INTEGER DEFAULT 0
);
```

Результат: таблиця `products` з чотирма стовпцями. `product_id` — автоінкрементний первинний ключ. `product_name` та `price` — обов'язкові поля (`NOT NULL`). `quantity` — за замовчуванням `0`.

---

### 2. ALTER — зміна структури

Додавання нового стовпця `manufacturer` до таблиці `products`.

```sql
ALTER TABLE products
ADD COLUMN manufacturer VARCHAR(255);
```

Результат: стовпець `manufacturer` додається до кожного існуючого рядка зі значенням `NULL` за замовчуванням.

---

### 3. DROP — видалення таблиці

Повне видалення таблиці `products` разом з усіма даними та індексами.

```sql
DROP TABLE products;
```

Результат: таблиця повністю видалена. Операція незворотна — відновлення без резервної копії неможливе.

---

### 4. TRUNCATE — очищення таблиці

Видалення всіх записів з таблиці `products` зі збереженням структури.

```sql
TRUNCATE TABLE products;
```

Результат: таблиця порожня, але схема залишається. Швидше за `DELETE` без `WHERE`, але не виконається за наявності зовнішніх ключів.

---

### 5. RENAME — перейменування таблиці

Перейменування таблиці `products` на `inventory`.

```sql
ALTER TABLE products RENAME TO inventory;
```

Результат: таблиця тепер доступна як `inventory`. Дані та структура залишаються незмінними.

---

### 6. COMMENT — документування об'єкта

Додавання коментаря до таблиці `products`.

```sql
COMMENT ON TABLE products IS 'This table stores information about products.';
```

Результат: коментар зберігається в метаданих бази даних та доступний через системні представлення для документування структури.

---

## DML — операції над даними

### 1. INSERT — вставка запису

Додавання нового продукту до таблиці `products`.

```sql
INSERT INTO products (product_id, product_name, price)
VALUES (101, 'Телефон', 500);
```

Результат: новий рядок з `product_id = 101`, назвою `'Телефон'` та ціною `500` додано до таблиці.

---

### 2. UPDATE — оновлення запису

Зміна ціни продукту з `product_id = 101`.

```sql
UPDATE products
SET price = 550
WHERE product_id = 101;
```

Результат: ціна продукту з ідентифікатором `101` змінена на `550`.

---

### 3. DELETE — видалення запису

Видалення продукту з `product_id = 101`.

```sql
DELETE FROM products
WHERE product_id = 101;
```

Результат: рядок з `product_id = 101` видалено з таблиці.

---

### 4. SELECT — вибірка даних

Вибірка всіх продуктів, ціна яких перевищує `100`.

```sql
SELECT * FROM products
WHERE price > 100;
```

Результат: повертає всі рядки таблиці `products`, де значення стовпця `price` більше `100`.

---

> [!warning] Обмеження та ризики
> 
> - **`DROP` без резервної копії.** Незворотна операція — всі дані, структура та індекси знищуються без можливості відновлення.
>     
> - **`TRUNCATE` і зовнішні ключі.** Якщо на таблицю посилаються інші таблиці через `FOREIGN KEY`, `TRUNCATE` завершиться помилкою. Необхідно спочатку видалити або відключити залежності.
>     
> - **`UPDATE` / `DELETE` без `WHERE`.** Без умови фільтрації операція застосовується до всіх рядків таблиці — одна з найпоширеніших критичних помилок у роботі з продакшн-даними.
>     
> - **`ALTER TABLE` на великій таблиці.** Деякі операції `ALTER` (наприклад, додавання стовпця без `DEFAULT`) блокують таблицю на час виконання, що може призвести до недоступності даних.
>     

---

> [!tip] Коли що використовувати
> 
> **DDL** ✅:
> 
> - `CREATE` — при проєктуванні нової схеми або додаванні нових сутностей;
> - `ALTER` — при еволюції схеми (нові поля, зміна типів);
> - `DROP` — лише за наявності резервної копії та впевненості у незворотності;
> - `TRUNCATE` — для швидкого очищення таблиці при відсутності зовнішніх ключів;
> - `RENAME` — при рефакторингу схеми без втрати даних;
> - `COMMENT` — для документування великих або складних схем.
> 
> **DML** ✅:
> 
> - `INSERT` — додавання нових даних, підтримує масову вставку кількох рядків;
> - `UPDATE` — завжди з `WHERE` для точного вибору рядків;
> - `DELETE` — завжди з `WHERE`; для повного очищення краще `TRUNCATE`;
> - `SELECT` — читання та аналіз даних; уникай `SELECT *` у продакшн-коді.
