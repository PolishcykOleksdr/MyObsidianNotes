> На відміну від байтових потоків, символьні потоки в Java призначені спеціально для роботи з текстовими даними. Вони автоматично обробляють кодування (Unicode), що дозволяє коректно читати та записувати текст на різних мовах.

---

> [!abstract] Core Framework 
> Базові компоненти для роботи з текстом:
> 
> - **Reader**: Абстрактний суперклас для читання символьних потоків.
>     
> - **Writer**: Абстрактний суперклас для запису символьних потоків.
>     
> - **InputStreamReader / OutputStreamWriter**: Класи-мости, що перетворюють байтові потоки у символьні та навпаки.
>     
> - **FileReader / FileWriter**: Спеціалізовані класи для безпосередньої роботи з текстовими файлами.
>     

---

> [!info] Key Insights
> 
> - **Unicode-орієнтованість**: Символьні потоки оперують типом `char` (16 біт), що дозволяє підтримувати міжнародні набори символів.
>     
> - **Автоматичне кодування**: `FileWriter` та `FileReader` використовують системне кодування за замовчуванням (або вказане розробником), щоб перетворити символи програми у байти на диску.
>     
> - **Ефективне збирання рядків**: При читанні по одному символу в циклі рекомендується використовувати `StringBuilder` для накопичення результату, щоб уникнути створення безлічі об'єктів `String` у пам'яті.
>

> [!example] Практичне застосування: Запис через try-with-resources
> 
> Java
> 
> ```java
> import java.io.FileWriter;
> 
> public class Main {
>    private final static String BASE_PATH = "/Username/Files/";
> 
>    public static void main(String[] args) {
>        String myFile = "myfile.txt";
>        String myText = "Some information.";
> 
>        /* >         * try-with-resources: гарантує автоматичне закриття FileWriter.
>         * Це можливо, оскільки FileWriter реалізує AutoCloseable.
>         */
>        try(FileWriter fw = new FileWriter(BASE_PATH + myFile)) {
>            fw.write(myText);
>            System.out.println("Success.");
>        } catch (Exception e) {
>            System.out.println("Exception: " + e.getMessage());
>        }
>    }
> }
> ```

---

> [!example] Практичне застосування: Читання символів
> 
> Java
> 
> ```java
> import java.io.FileReader;
> import java.io.IOException;
> 
> public class Main {
>     private final static String BASE_PATH = "/Username/Files/";
> 
>     public static void main(String[] args) {
>         String path = BASE_PATH + "myfile.txt";
>         System.out.println(readFile(path));
>     }
> 
>     private static String readFile(String path) {
>         try (FileReader reader = new FileReader(path)) {
>             int sym;
>             StringBuilder stringBuilder = new StringBuilder();
>             /* Метод read() повертає int (код символу), або -1 у кінці */
>             while ((sym = reader.read()) != -1) {
>                 stringBuilder.append((char) sym);
>             }
>             return stringBuilder.toString();
>         } catch (IOException ex) {
>            return "Error: " + ex.getMessage();
>         }
>     }
> }
> ```

> [!important] Різниця між `InputStreamReader` та `FileReader`
> 
> - **`FileReader`** — спрощена обгортка спеціально для читання текстових файлів.
> - **`InputStreamReader`** — універсальний клас, який перетворює будь-який `InputStream` (не лише файл) у потік символів.
> 
> ### Ієрархія класів
> 
> ```
> Reader
>  ├── InputStreamReader
>  │    └── FileReader  ← є підкласом InputStreamReader
> ```
> 
> `FileReader` — це просто підклас `InputStreamReader`, вже налаштований на роботу з файлами.
> 
> ### Головна різниця — кодування
> 
> java
> 
> ```java
> // FileReader — використовує кодування системи за замовчуванням
> // Не можна вказати кодування (до Java 11)
> FileReader fr = new FileReader("file.txt");
> 
> // InputStreamReader — можна явно вказати кодування
> InputStreamReader isr = new InputStreamReader(
>     new FileInputStream("file.txt"), StandardCharsets.UTF_8
> );
> ```
> 
> ⚠️ Якщо система працює не в UTF-8, `FileReader` може читати кирилицю неправильно!
> 
> ### Джерела даних
> 
> java
> 
> ```java
> // FileReader — тільки файл
> FileReader fr = new FileReader("file.txt");
> 
> // InputStreamReader — будь-який InputStream
> new InputStreamReader(new FileInputStream("file.txt")); // файл
> new InputStreamReader(System.in);                       // консоль
> new InputStreamReader(socket.getInputStream());         // мережа
> new InputStreamReader(url.openStream());                // URL
> ```
> 
> ### Порівняння
> 
> |`Різниця`|`FileReader`|`InputStreamReader`|
> |---|---|---|
> |Джерело даних|тільки файл|будь-який `InputStream`|
> |Вказати кодування|⚠️ лише з Java 11|✅ завжди|
> |Простота використання|✅ простіший|трохи складніший|
> |Гнучкість|❌ обмежена|✅ висока|
> 
> ### Практична порада
> 
> На практиці краще завжди використовувати `InputStreamReader` з явним кодуванням, обгорнутий у `BufferedReader` для ефективності:
> 
> java
> 
> ```java
> BufferedReader reader = new BufferedReader(
>     new InputStreamReader(new FileInputStream("file.txt"), StandardCharsets.UTF_8)
> );
> ```
> 

---

> [!warning] Ресурси та витоки (Leaks)
> 
> - **Закриття потоків**: Якщо не закрити `Writer`, частина даних може залишитися в буфері та не записатися на диск.
>     
> - **Interface AutoCloseable**: Тільки об'єкти, що реалізують цей інтерфейс, можуть бути оголошені всередині дужок `try(...)`.
>     
> - **Блокування файлів**: Відкритий і незакритий потік може заблокувати файл для інших програм або процесів операційної системи.
>
