> **Java IO (Input/Output)** — це пакет (`java.io`), який використовується для обробки процесів читання та запису даних. В основі лежить концепція **потоку (Stream)**, що представляє собою безперервну послідовність байтів, які передаються від джерела до приймача.

---

> [!abstract] Core Framework 
> Java IO побудований на ієрархії класів, що розділяють роботу з байтовими та символьними даними:
> 
> - **Байтові потоки (Byte Streams)**: Використовуються для роботи з необробленими бінарними даними (зображення, відео, файли).
>     
>     - `InputStream` (введення)
>         
>     - `OutputStream` (виведення)
>         
> - **Символьні потоки (Character Streams)**: Оптимізовані для роботи з текстом (Unicode).
>     
>     - `Reader` (читання)
>         
>     - `Writer` (запис)
>         

---

> [!info] Key Insights
> 
> - **Однонаправленість**: Потік у Java IO працює лише в один бік. Якщо вам потрібно і читати, і записувати, необхідно створювати два окремі потоки.
>     
> - **Потокова орієнтованість**: Дані зчитуються послідовно. Ви не можете "перестрибнути" в середину потоку або повернутися назад без попереднього кешування даних у буфері.
>     
> - **Відсутність кешування за замовчуванням**: Базові потоки зчитують по одному байту за раз прямо з джерела, що може бути повільним. Для оптимізації використовуються буферизовані класи (наприклад, `BufferedInputStream`).
>     
> - **Джерела та приймачі**: Дані можуть надходити з файлів, мережевих з'єднань, масивів байтів або консолі.
>

> [!example] Практичне застосування: Робота з файлами
> 
> Java
> 
> ```java
> import java.io.FileInputStream;
> import java.io.FileOutputStream;
> import java.io.IOException;
> 
> /* >  * Копіювання вмісту файлу через байтовий потік InputStream/OutputStream
>  */
> public class FileCopyExample {
>     public static void main(String[] args) {
>         // Використовуємо try-with-resources для автоматичного закриття потоків
>         try (FileInputStream in = new FileInputStream("source.dat");
>              FileOutputStream out = new FileOutputStream("dest.dat")) {
>             
>             int data;
>             /* Читаємо один байт за раз, поки не досягнемо кінця файлу (-1) */
>             while ((data = in.read()) != -1) {
>                 out.write(data);
>             }
>         } catch (IOException e) {
>             System.err.println("Помилка IO: " + e.getMessage());
>         }
>     }
> }
> ```

---

> [!warning] Обмеження та ризики
> 
> - **Блокування (Blocking IO)**: Коли потік викликає метод `read()`, він зупиняє (блокує) виконання програми до тих пір, поки дані не стануть доступними. Це критично для високонавантажених серверів.
>     
> - **Витік ресурсів**: Якщо потік не закрити явно через `.close()`, дескриптор файлу або мережевий порт залишиться відкритим, що може призвести до збоїв у системі.
>     
> - **Продуктивність**: Робота без буферизації з великими файлами спричиняє величезну кількість системних викликів, що сильно сповільнює програму.
>     

---

> [!tip] Коли використовувати 
> Так ✅:
> 
> - Робота з локальними файлами невеликого розміру.
>     
> - Послідовна обробка текстових конфігурацій чи логів.
>     
> - Прості інструменти командного рядка.
>     
> 
> Ні ❌:
> 
> - Високопродуктивні мережеві сервери (краще використовувати Java NIO).
>     
> - Випадковий доступ до даних у середині великого файлу (використовуйте `RandomAccessFile`).
>
