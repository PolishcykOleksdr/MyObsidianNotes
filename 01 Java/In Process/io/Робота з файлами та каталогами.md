> У Java робота з файловою системою на базовому рівні реалізована через клас `File`. Важливо розуміти, що об'єкт цього класу є лише **абстрактним представленням** шляху до файлу або папки, а не самим файлом чи потоком даних.

---

> [!abstract] Core Framework 
> Клас **File** (`java.io.File`) надає засоби для:
> 
> - Перевірки існування об'єкта (`exists()`).
>     
> - Отримання властивостей (розмір, дата зміни, права доступу).
>     
> - Маніпуляцій: створення (`createNewFile`, `mkdir`), видалення (`delete`) та перейменування (`renameTo`).
>     
> - Навігації: отримання списку файлів у директорії (`list`, `listFiles`).
>     

---

> [!info] Key Insights
> 
> - **Універсальність**: Один і той самий клас `File` описує і текстовий документ, і папку, і навіть диск.
>     
> - **Метадані**: Клас дозволяє дізнатися, чи є шлях файлом (`isFile()`) або директорією (`isDirectory()`).
>     
> - **Обмеження**: `File` API вважається застарілим (Legacy). Він не надає детальної інформації про помилки (повертає лише `boolean`) та не підтримує сучасні фічі на кшталт символічних посилань.
>     

---

> [!example] Практичне застосування: Керування папками
> 
> Java
> 
> ```java
> import java.io.File;
> 
> public class DirectoryManager {
>     private final static String BASE_PATH = "/Username/Files/";
> 
>     public static void main(String[] args) {
>         File folder = new File(BASE_PATH + "ProjectData");
> 
>         // Створення директорії
>         if (folder.mkdir()) {
>             System.out.println("Папка створена");
>         }
> 
>         // Перейменування
>         File renamedFolder = new File(BASE_PATH + "OldData");
>         folder.renameTo(renamedFolder);
> 
>         // Видалення (папка має бути порожньою!)
>         if (renamedFolder.delete()) {
>             System.out.println("Папка видалена");
>         }
>     }
> }
> ```

---

> [!example] Практичне застосування: Робота з файлами
> 
> Java
> 
> ```java
> import java.io.File;
> import java.io.IOException;
> 
> public class FileManager {
>     public static void main(String[] args) {
>         File file = new File("test.txt");
> 
>         try {
>             if (file.createNewFile()) {
>                 System.out.println("Файл створено: " + file.getAbsolutePath());
>             } else {
>                 System.out.println("Файл вже існує.");
>             }
>         } catch (IOException e) {
>             System.out.println("Помилка: " + e.getMessage());
>         }
>     }
> }
> ```

---

> [!important] Різниця між `file.mkdir()` та `file.mkdirs()`
> 
> - **`mkdir()`** — створює лише одну директорію. Якщо батьківські директорії не існують — поверне `false` і нічого не створить.
> - **`mkdirs()`** — створює директорію разом з усіма батьківськими, якщо вони відсутні.
> 
> ### Приклад
> 
> java
> 
> ```java
> File file = new File("a/b/c");
> 
> // Якщо "a" і "b" не існують:
> file.mkdir();   // ❌ поверне false, нічого не створить
> file.mkdirs();  // ✅ створить a/ → a/b/ → a/b/c/
> ```
> 
> ### Порівняння
> 
> |`mkdir()`|`mkdirs()`|
> |---|---|---|
> |Створює батьківські папки|❌|✅|
> |Повертає `true` якщо успішно|✅|✅|
> |Повертає `false` якщо вже існує|✅|✅|
> |Аналог у терміналі|`mkdir dir`|`mkdir -p dir`|
> 
> ### Коли що використовувати
> 
> - **`mkdir()`** — коли впевнений, що батьківський шлях вже існує.
> - **`mkdirs()`** — коли створюєш вкладену структуру або не знаєш, чи існують батьківські директорії. На практиці `mkdirs()` використовують частіше, бо він безпечніший.

---

> [!warning] Проблеми та недоліки File API
> 
> - **Тиха відмова**: Якщо `delete()` повертає `false`, ви не знаєте чому (брак прав, файл зайнятий іншим процесом чи папка не порожня).
>     
> - **Продуктивність**: Метод `list()` при великій кількості файлів може працювати повільно та споживати багато пам'яті.
>     
> - **Платформозалежність**: Робота з іменами файлів та шляхами може відрізнятися між Windows та Linux/macOS.
>     

---

> [!tip] Коли використовувати 
> Так ✅:
> 
> - Прості операції у старих проектах (Java 6 і нижче).
>     
> - Швидка перевірка "чи існує файл".
>     
> 
> Ні ❌:
> 
> - Сучасні додатки (рекомендується використовувати **Java NIO.2** та клас `java.nio.file.Path`).
>     
> - Робота з атрибутами файлів (власник, права доступу POSIX).
>     
> - Копіювання або переміщення великих дерев каталогів.
>
