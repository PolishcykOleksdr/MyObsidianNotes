> Алгоритм у програмуванні — це чітко визначений набір інструкцій або процедур, призначених для вирішення конкретної задачі або досягнення мети. Вони є логічним ядром, що визначає поведінку та ефективність програмних систем.

---

> [!abstract] Core Framework
> 
> Алгоритми в Java — це логічні конструкції, що оперують структурами даних для виконання обчислювальних завдань.
> 
> Основні операції, що підтримуються алгоритмами:
> 
> - **Додавання**: вставка нового елемента в структуру.
>     
> - **Видалення**: виключення елемента з колекції.
>     
> - **Зміна**: оновлення існуючого значення в структурі даних.
>     

---

> [!info] Key Insights
> 
> - **Коректність**: Здатність алгоритму видавати правильний результат для будь-яких допустимих вхідних даних.
>     
> - **Продуктивність**: Визначається обсягом споживаних ресурсів, зокрема пам'яті та обчислювальної потужності CPU.
>     
> - **Масштабованість**: Здатність алгоритму зберігати прийнятну ефективність при значному зростанні обсягу вхідних даних.
>     
> - **Розширюваність**: Можливість адаптації алгоритму до роботи з різними типами даних та в мінливих умовах оточення.
>     
> - **Стійкість**: Стабільність функціонування системи навіть у непередбачуваних або помилкових ситуаціях.
>     

---

> [!example] Практичне застосування
> 
> ### 1. Алгоритми сортування
> 
> Впорядкування елементів за певним критерієм (зростання/спадання).
> 
> Java
> 
> ```java
> /* Сортування бульбашкою (Bubble Sort) 
>    Порівняння сусідніх елементів та їх перестановка */
> public void bubbleSort(int[] arr) {
>     int n = arr.length;
>     for (int i = 0; i < n - 1; i++) {
>         for (int j = 0; j < n - i - 1; j++) {
>             if (arr[j] > arr[j + 1]) {
>                 int temp = arr[j];
>                 arr[j] = arr[j + 1];
>                 arr[j + 1] = temp;
>             }
>         }
>     }
> }
> ```
> 
> ### 2. Алгоритми пошуку
> 
> Знаходження цільового елемента в структурі.
> 
> Java
> 
> ```java
> /* Бінарний пошук (Binary Search)
>    Працює лише у відсортованих масивах */
> public int binarySearch(int[] arr, int target) {
>     int left = 0, right = arr.length - 1;
>     while (left <= right) {
>         int mid = left + (right - left) / 2;
>         if (arr[mid] == target) return mid; /* Елемент знайдено */
>         if (arr[mid] < target) left = mid + 1; /* Шукаємо в правій частині */
>         else right = mid - 1; /* Шукаємо в лівій частині */
>     }
>     return -1; /* Елемент відсутній */
> }
> ```
> 
> ### 3. Рекурсія
> 
> Метод, де функція викликає саму себе.
> 
> Java
> 
> ```java
> /* Обчислення факторіала n! 
>    Приклад рекурсивного підходу */
> public int factorial(int n) {
>     if (n <= 1) return 1; /* Базовий випадок */
>     return n * factorial(n - 1); /* Рекурсивний виклик */
> }
> ```
>

---

> [!warning] Обмеження та ризики
> 
> - **Часова складність**: Неефективні алгоритми (наприклад, Bubble Sort на великих даних) призводять до експоненціального зростання часу виконання $O(n^2)$.
>     
> - **Складність реалізації**: Алгоритми на кшталт "Швидкого сортування" або "Злиття" складніші у впровадженні та налагодженні порівняно з лінійними методами.
>     
> - **Використання пам'яті**: Рекурсивні виклики створюють навантаження на стек викликів (Stack), що може призвести до `StackOverflowError` при великій глибині.
>     

---

> [!tip] Вибір алгоритму
> 
> **Сортування:**
> 
> - Використовуйте **QuickSort/MergeSort** для великих масивів даних.
>     
> - Використовуйте **Bubble Sort** лише для навчальних цілей або дуже малих наборів даних через низьку ефективність.
>     
> 
> **Пошук:**
> 
> - **Лінійний пошук**: для невідсортованих даних невеликого розміру.
>     
> - **Бінарний пошук**: для швидкого знаходження у впорядкованих структурах.
>     
> 
> **Графи:**
> 
> - **BFS (в ширину)**: коли потрібно знайти найкоротший шлях у незваженому графі.
>     
> - **DFS (в глибину)**: для повного обходу або перевірки зв'язності.
>

# Порівняння алгоритмів та характеристик

|**Категорія**|**Алгоритм**|**Складність (Average)**|**Особливість**|
|---|---|---|---|
|**Сортування**|Bubble Sort|$O(n^2)$|Простий, але повільний|
|**Сортування**|Quick Sort|$O(n \log n)$|Стратегія "Розділяй і володарюй"|
|**Сортування**|Merge Sort|$O(n \log n)$|Стабільне сортування злиттям|
|**Пошук**|Linear Search|$O(n)$|Перевірка кожного елемента|
|**Пошук**|Binary Search|$O(\log n)$|Тільки для впорядкованих даних|
|**Графи**|BFS|$O(V + E)$|Пошук через сусідні вершини|
|**Графи**|DFS|$O(V + E)$|Рух вглиб до упору|
