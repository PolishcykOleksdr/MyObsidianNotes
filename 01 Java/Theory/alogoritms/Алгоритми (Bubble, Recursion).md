> Алгоритми сортування та рекурсивні обчислення є фундаментальними інструментами в програмуванні, що дозволяють маніпулювати структурами даних та розв'язувати математичні задачі через декомпозицію.

---

> [!abstract]
> 
> ### Core Framework
> 
> Дані алгоритми представляють два різні підходи до обробки даних: ітеративне переставлення елементів та рекурсивний поділ задачі.
> 
> **Сортування бульбашкою** — ітеративний алгоритм для впорядкування масиву шляхом порівняння сусідніх елементів.
> 
> **Рекурсія Фібоначчі** — метод обчислення чисел послідовності, де кожен елемент є сумою двох попередніх: $F(n) = F(n-1) + F(n-2)$.

---

> [!info]
> 
> ### Key Insights
> 
> - **Механіка Bubble Sort.** Алгоритм "виштовхує" найбільший елемент у кінець масиву під час кожного повного проходу, подібно до бульбашки повітря у воді.
>     
> - **Рекурсивний спуск.** У випадку з Фібоначчі програма будує дерево викликів. Кожен вузол розгалужується на два нових, поки не досягне базових випадків ($n \le 1$).
>     
> - **Ефективність пам'яті.** Сортування бульбашкою працює "на місці" (in-place), тоді як рекурсія активно використовує системний стек для зберігання контексту кожного виклику.
>     

---

> [!example]
> 
> ### Практичне застосування
> 
> **1. Bubble Sort Implementation**
> 
> Java
> 
> ```java
> public void bubbleSort(int[] arr) {
>     int n = arr.length;
>     for (int i = 0; i < n - 1; i++) {
>         for (int j = 0; j < n - i - 1; j++) {
>             /* Порівняння сусідніх елементів */
>             if (arr[j] > arr[j + 1]) {
>                 /* Перестановка (Swap) */
>                 int temp = arr[j];
>                 arr[j] = arr[j + 1];
>                 arr[j + 1] = temp;
>             }
>         }
>     }
> }
> ```
> 
> **2. Recursive Fibonacci Implementation**
> 
> Java
> 
> ```java
> public int fibonacci(int n) {
>     /* Базовий випадок: зупинка рекурсії */
>     if (n <= 1) {
>         return n;
>     }
>     /* Рекурсивний виклик для (n-1) та (n-2) */
>     return fibonacci(n - 1) + fibonacci(n - 2);
> }
> ```

---

> [!warning]
> 
> ### Обмеження та ризики
> 
> - **Stack Overflow.** При великих $n$ рекурсія Фібоначчі призводить до переповнення стека через надмірну глибину викликів.
>     
> - **Експоненціальний вибух.** Складність $O(2^n)$ означає, що додавання лише одного числа до $n$ приблизно подвоює час виконання.
>     
> - **Продуктивність.** Сортування бульбашкою є вкрай повільним для великих масивів порівняно з QuickSort або MergeSort через квадратичну складність $O(n^2)$.
>     

---

> [!tip]
> 
> ### Поради щодо оптимізації
> 
> **Bubble Sort ✅:**
> 
> - Використовуйте лише для навчальних цілей або дуже малих масивів.
>     
> - Додайте прапор (flag) для виходу з циклу, якщо за прохід не було зроблено жодного обміну.
>     
> 
> **Fibonacci ✅:**
> 
> - Для великих $n$ замінюйте рекурсію ітеративним підходом або динамічним програмуванням (мемоїзацією) для зниження складності до $O(n)$.
>
