> Алгоритми сортування — це спеціальні методи або процедури, які використовуються для впорядкування елементів у певному порядку. Вони є критично важливими в програмуванні, оскільки дозволяють ефективно обробляти та аналізувати великі обсяги даних.

---

> [!abstract] Core Framework
> 
> Колекції Java та масиви використовують алгоритми сортування як фундаментальний інструмент організації даних.
> 
> **Core Framework**
> 
> Алгоритми сортування є реалізацією логіки впорядкування, що забезпечує:
> 
> - Ефективний пошук елементів у структурах.
>     
> - Оптимізацію аналізу великих масивів даних.
>     
> - Підготовку даних для подальшої обробки (наприклад, перед бінарним пошуком).
>     

---

> [!info] Key Insights
> 
> - **Механіка "спливання"**. У Bubble Sort елементи порівнюються попарно, і найбільше значення ітеративно переміщується в кінець, що нагадує рух бульбашки повітря у воді.
>     
> - **Поступова побудова**. Insertion Sort працює за принципом сортування карт у руках: кожен новий елемент шукає своє логічне місце серед уже впорядкованих.
>     
> - **Розділяй та володарюй**. Quick Sort та Merge Sort використовують рекурсивний підхід, розбиваючи велику задачу на менші підзадачі, що кардинально знижує часову складність для великих обсягів даних.
>     
> - **In-place сортування**. Більшість простих алгоритмів мають складність за простором $O(1)$, оскільки модифікують вхідний масив без виділення додаткової пам'яті.
>     

---

> [!example] Практичне застосування
> **1. Bubble Sort (Сортування бульбашкою)**
> 
> Алгоритм кілька разів проходить по масиву, порівнюючи кожну пару сусідніх елементів.
> 
> Java
> 
> ```java
> public void bubbleSort(int[] array) {
>     int n = array.length;
>     for (int i = 0; i < n - 1; i++) {
>         for (int j = 0; j < n - i - 1; j++) {
>             /* Порівнюємо сусідні елементи */
>             if (array[j] > array[j + 1]) {
>                 /* Міняємо місцями, якщо порядок хибний */
>                 int temp = array[j];
>                 array[j] = array[j + 1];
>                 array[j + 1] = temp;
>             }
>         }
>     }
> }
> ```
> 
> **2. Insertion Sort (Сортування вставками)**
> 
> Будує впорядковану послідовність, вставляючи елемент у відповідне місце.
> 
> Java
> 
> ```java
> public void insertionSort(int[] array) {
>     int n = array.length;
>     for (int i = 1; i < n; ++i) {
>         int key = array[i];
>         int j = i - 1;
>         /* Пересуваємо елементи, що більші за key */
>         while (j >= 0 && array[j] > key) {
>             array[j + 1] = array[j];
>             j = j - 1;
>         }
>         array[j + 1] = key;
>     }
> }
> ```
> 
> **3. Quick Sort (Швидке сортування)**
> 
> Використовує опорний елемент (півот) для розділення масиву.
> 
> **4. Merge Sort (Сортування злиттям)**
> 
> Рекурсивно розбиває масив на дві половини, сортує їх та зливає.

---

> [!warning] Обмеження та ризики
> 
> - **Низька ефективність $O(n^2)$**. Алгоритми на кшталт Bubble Sort та Insertion Sort стають надто повільними при зростанні кількості елементів (неефективні для великих масивів).
>     
> - **Рекурсія**. Алгоритми "розділяй та володарюй" використовують рекурсивні виклики, що може призвести до переповнення стеку при надто великій глибині.
>     
> - **Витрати пам'яті**. Хоча прості алгоритми споживають $O(1)$, складніші методи (наприклад, Merge Sort) можуть потребувати значного додаткового обсягу пам'яті для злиття частин масиву.
>     

---

> [!tip] Переваги та недоліки
> 
> **Bubble Sort**
> 
> - Так ✅: Проста реалізація, мінімальні витрати пам'яті $O(1)$.
>     
> - Ні ❌: Дуже низька швидкість на реальних даних.
>     
> 
> **Insertion Sort**
> 
> - Так ✅: Найкраще підходить для невеликих або майже відсортованих масивів.
>     
> - Ні ❌: Складність $O(n^2)$ у середньому та найгіршому випадках.
>     
> 
> **Quick Sort**
> 
> - Так ✅: Висока швидкість завдяки стратегії "розділяй та володарюй".
>
