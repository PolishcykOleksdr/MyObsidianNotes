> Алгоритми пошуку — це методи, які використовуються для знаходження конкретного елемента або шаблону даних у колекції. Вони застосовуються в базах даних, графіці, обробці тексту та інших областях.

---

> [!abstract]
> 
> ### Core Framework
> 
> Алгоритми пошуку є реалізацією фундаментальних стратегій доступу до даних, що підтримують основні операції:
> 
> - Знаходження індексу або посилання на конкретний елемент у структурі.
>     
> - Перевірка наявності шаблону в наборі даних.
>     
> - Оптимізація швидкості доступу залежно від впорядкованості та типу контейнера.
>     

---

> [!info]
> 
> ### Key Insights
> 
> - **Послідовний перебір.** Лінійний пошук не вимагає попередньої підготовки даних, але його вартість зростає пропорційно кількості елементів $n$.
>     
> - **Розділяй і володарюй.** Бінарний пошук використовує властивість впорядкованості, щоб на кожній ітерації відсікати рівно половину простору пошуку, що дає логарифмічну складність $\log n$.
>     
> - **Пряма адресація.** Хеш-пошук трансформує значення ключа в індекс пам'яті за допомогою хеш-функції, намагаючись досягти константного часу доступу $O(1)$.
>     
> - **Проблема колізій.** Ефективність хеш-таблиць критично залежить від якості хеш-функції; при ідентичних хешах для різних об'єктів швидкість деградує до лінійної.
>     

---

> [!example]
> 
> ### Практичне застосування
> 
> **1. Linear Search Implementation**
> 
> Java
> 
> ```java
> public int linearSearch(int[] arr, int target) {
>     /* Послідовна перевірка кожного елемента */
>     for (int i = 0; i < arr.length; i++) {
>         if (arr[i] == target) {
>             return i; /* Збіг знайдено */
>         }
>     }
>     return -1; /* Елемент відсутній */
> }
> ```
> 
> **2. Binary Search Implementation**
> 
> Java
> 
> ```java
> public int binarySearch(int[] sortedArr, int target) {
>     int low = 0;
>     int high = sortedArr.length - 1;
>     
>     while (low <= high) {
>         int mid = low + (high - low) / 2; /* Знаходження середини */
>         
>         if (sortedArr[mid] == target) return mid;
>         
>         if (sortedArr[mid] < target) {
>             low = mid + 1; /* Пошук у правій половині */
>         } else {
>             high = mid - 1; /* Пошук у лівій половині */
>         }
>     }
>     return -1;
> }
> ```
> 
> **3. Hash Table Logic (Conceptual)**
> 
> Java
> 
> ```java
> /* Спрощена логіка пошуку в хеш-таблиці */
> public V get(K key) {
>     int hash = computeHash(key); /* Обчислення хеш-коду */
>     int index = hash % table.length; /* Локалізація місця в таблиці */
>     
>     /* Прямий доступ до комірки (bucket) */
>     return table[index].findValue(key); 
> }
> ```

---

> [!warning]
> 
> ### Обмеження та ризики
> 
> - **Вимога до сортування.** Бінарний пошук поверне некоректний результат, якщо масив не впорядкований.
>     
> - **Витрати на хешування.** Хеш-таблиці потребують додаткової пам'яті $O(n)$ для зберігання структури та вирішення колізій.
>     
> - **Деградація швидкості.** При великій кількості колізій у хеш-таблиці пошук сповільнюється до $O(n)$, що нівелює переваги алгоритму.
>     

---

> [!tip]
> 
> ### Коли використовувати
> 
> **Лінійний пошук ✅:**
> 
> - Малі масиви даних.
>     
> - Несортовані колекції, де сортування коштує дорожче за сам пошук.
>     
> 
> **Бінарний пошук ✅:**
> 
> - Великі статичні або рідко змінювані сортовані списки.
>     
> 
> **Хеш-таблиці ✅:**
> 
> - Швидкий пошук за унікальним ключем у великих обсягах даних.
>     
> - Системи, де пам'ять не є критичним обмеженням.
>

# Порівняння характеристик пошуку

| **Характеристика**           | **Лінійний пошук** | **Бінарний пошук**         | **Хеш-таблиці**      |
| ---------------------------- | ------------------ | -------------------------- | -------------------- |
| **Вимога до даних**          | Будь-які           | Тільки впорядковані        | Потрібна хеш-функція |
| **Часова складність (сер.)** | $O(n)$             | $O(\log n)$                | $O(1)$               |
| **Просторова складність**    | $O(1)$             | $O(1)$                     | $O(n)$               |
| **Найкраще для...**          | Малих масивів      | Великих сортованих списків | Пошуку за ключем     |
