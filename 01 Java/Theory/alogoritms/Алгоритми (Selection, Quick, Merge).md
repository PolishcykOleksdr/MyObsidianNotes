> Алгоритми сортування — це фундаментальні методи впорядкування даних, що базуються на різних стратегіях: від простого перебору до рекурсивного розділення. Вони визначають ефективність обробки великих масивів у пам'яті JVM.

---

> [!abstract] Core Framework
> 
> Алгоритми сортування є реалізацією абстрактних структурних маніпуляцій над масивами, що підтримують основні операції:
> 
> - **Порівняння**: Визначення порядку між двома елементами.
>     
> - **Перестановка (Swap)**: Зміна позицій елементів у пам'яті.
>     
> - **Розділення (Partitioning/Splitting)**: Рекурсивне розбиття структури на менші підмножини.
>     

---

> [!info] Key Insights
> 
> - **Стабільність складності**: Сортування злиттям гарантує $O(n \log n)$ навіть у найгіршому випадку, що робить його передбачуваним для критичних систем.
>     
> - **Ефект Pivot**: Ефективність Quick Sort критично залежить від вибору опорного елемента; невдалий вибір призводить до деградації продуктивності до $O(n^2)$.
>     
> - **In-place обмеження**: Сортування вибором працює безпосередньо у вхідному масиві ($O(1)$ extra space), тоді як Merge Sort вимагає копіювання даних у тимчасові буфери ($O(n)$).
>     
> - **Квадратична неминучесть**: Selection Sort виконує повний цикл порівнянь незалежно від початкового стану даних.
>     

---

> [!example] Практичне застосування
> 
> ### 1. Selection Sort (Сортування вибором)
> 
> Пошук мінімуму та перестановка на початок невпорядкованої частини.
> 
> Java
> 
> ```java
> public void selectionSort(int[] arr) {
>     int n = arr.length;
>     for (int i = 0; i < n - 1; i++) {
>         int minIdx = i; /* Індекс мінімального елемента */
>         for (int j = i + 1; j < n; j++) {
>             if (arr[j] < arr[minIdx]) {
>                 minIdx = j; /* Оновлюємо індекс найменшого */
>             }
>         }
>         /* Міняємо місцями знайдений мінімум з першим елементом */
>         int temp = arr[minIdx];
>         arr[minIdx] = arr[i];
>         arr[i] = temp;
>     }
> }
> ```
> 
> ### 2. Quick Sort (Швидке сортування)
> 
> Стратегія «розділяй та володарюй» через опорний елемент.
> 
> Java
> 
> ```java
> /* Рекурсивне розділення масиву відносно pivot */
> void quickSort(int[] arr, int low, int high) {
>     if (low < high) {
>         int pi = partition(arr, low, high); /* Індекс розділення */
>         quickSort(arr, low, pi - 1);  /* Ліва частина */
>         quickSort(arr, pi + 1, high); /* Права частина */
>     }
> }
> ```
> 
> ### 3. Merge Sort (Сортування злиттям)
> 
> Рекурсивне ділення навпіл та подальше впорядковане об'єднання.
> 
> Java
> 
> ```java
> /* Злиття двох підмасивів у тимчасовий простір */
> void merge(int[] arr, int l, int m, int r) {
>     /* Створення тимчасових масивів вимагає O(n) пам'яті */
>     int n1 = m - l + 1;
>     int n2 = r - m;
>     /* ... логіка порівняння та копіювання ... */
> }
> ```

---

> [!warning] Обмеження та ризики
> 
> - **Stack Overflow**: Швидке сортування при глибокій рекурсії (найгірший випадок) може призвести до переповнення стеку викликів.
>     
> - **Memory Overhead**: Сортування злиттям споживає значний обсяг пам'яті $O(n)$, що може бути критичним для обмежених ресурсів Heap пам'яті.
>     
> - **CPU Wastage**: Використання Selection Sort на великих даних призводить до марних обчислень через його $O(n^2)$ природу навіть для майже відсортованих списків.
>     

---

> [!tip] Коли використовувати
> 
> **Selection Sort**
> 
> - Так ✅: Невеликі масиви, де важлива простота коду.
>     
> - Ні ❌: Будь-які великі обсяги даних.
>     
> 
> **Quick Sort**
> 
> - Так ✅: Середні та великі дані; коли важлива середня швидкість $O(n \log n)$.
>     
> - Ні ❌: Коли потрібна гарантована швидкість у найгіршому випадку.
>     
> 
> **Merge Sort**
> 
> - Так ✅: Дуже великі обсяги даних, де стабільність часу дорожча за пам'ять.
>     
> - Ні ❌: Системи з жорстким лімітом RAM.
>
