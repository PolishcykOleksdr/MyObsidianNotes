> Синхронізація (Synchronization) — це процес керування доступом кількох потоків до спільних ресурсів. Основна мета полягає в уникненні втручання потоків (thread interference) та забезпеченні узгодженості даних.

---

> [!abstract] Core Framework 
> Об'єкти в багатопотоковому середовищі потребують синхронізації для захисту критичних областей коду:
> 
> - **Запобігання неузгодженості:** Гарантує, що один потік не почне опрацювання даних, поки інший не завершить їх читання або оновлення.
>     
> - **Критичні області:** Визначаються за допомогою оператора `synchronized`.
>     
> - **Механізм блокування:** Синхронізація базується на концепції монітора (locking). Коли потік входить у синхронізовану область, він захоплює блокування об'єкта.
>     

---

> [!info] Key Insights
> 
> - **Асинхронне оновлення:** Проблема виникає, коли кілька потоків одночасно намагаються змінити спільний ресурс без координації.
>     
> - **Автоматичне звільнення:** Потік автоматично звільняє блокування (lock) після завершення виконання синхронізованого методу або блоку.
>     
> - **Об'єкти синхронізації:** Блокування може відбуватися на рівні методу (весь об'єкт) або блоку (конкретна частина коду/об'єкта).
>     
> - **Продуктивність:** Використання `synchronized` може знизити швидкість роботи системи через витрати на керування блокуваннями.
>     

---

> [!example] Практичне застосування
> 
> #### 1. Проблема без синхронізації (Race Condition)
> 
> Коли метод не синхронізований, планувальник потоків може перемикати контекст посеред виконання циклу. Результати виводяться упереміш.
> 
> ```java
> public class DataHandler {
>     /* Не синхронізований метод: потоки будуть переривати один одного */
>     void printData(int n) {
>         for (int i = 1; i <= 5; i++) {
>             System.out.println(n * i);
>             try {
>                 Thread.sleep(500);
>             } catch (Exception e) {
>                 System.out.println("Exception " + e);
>             }
>         }
>     }
> }
> 
> public class MyThread01 extends Thread { 
>     DataHandler dataHandler; 
>     MyThread01(DataHandler dataHandler) {
> 	     this.dataHandler = dataHandler; 
>      } 
>     public void run() { 
> 	    dataHandler.printData(5); 
>     } 
> }
> 
> public class MyThread02 extends Thread { 
>     DataHandler dataHandler; 
>     MyThread02(DataHandler dataHandler) { 
> 	    this.dataHandler = dataHandler; 
>     } 
>     public void run() { 
> 	    dataHandler.printData(100); 
>     } 
> }
> 
> public class Main { 
>     public static void main(String[] args) { 
>         DataHandler dataHandler = new DataHandler(); 
>         MyThread01 t1 = new MyThread01(dataHandler); 
>         MyThread02 t2 = new MyThread02(dataHandler); 
>         t1.start(); 
>         t2.start(); 
>     } 
> } 
> /* Результат суперечливий (перемішаний): 5, 100, 10, 200... */
> ```
> 
> #### 2. Використання Synchronized Method
> 
> Додавання ключового слова `synchronized` до сигнатури методу автоматично блокує **весь об'єкт** (його монітор) для інших потоків, доки поточний потік не вийде з методу.
> 
> ```java
> public class DataHandler {
>     /* Синхронізований метод блокує весь об'єкт для потоку */
>     synchronized void printData(int n) {
>         for (int i = 1; i <= 5; i++) {
>             int res = n * i;
>             System.out.println("Result is " + res);
>             try {
>                 Thread.sleep(500);
>             } catch (Exception e) {
>                 System.out.println("Exception " + e);
>             }
>         }
>     }
> }
> /* Результат послідовний: спочатку всі множення для 5, потім для 100 */
> ```
> 
> #### 3. Використання Synchronized Block
> 
> Більш гнучкий підхід. Дозволяє синхронізувати не весь метод, а лише конкретний блок коду. Це корисно, якщо в методі є важкі операції, які не потребують блокування (наприклад, читання файлу або запити в мережу).
> 
> ```java
> public class DataHandler {
>     void printData(int n) {
>         /* Блокується тільки критична секція всередині методу */
>         synchronized (this) {
>             for (int i = 1; i <= 5; i++) {
>                 int res = n * i;
>                 System.out.println("Result is " + res);
>                 try {
>                     Thread.sleep(500);
>                 } catch (Exception e) {
>                     System.out.println("Exception " + e);
>                 }
>             }
>         }
>     }
> }
> ```
> 

---

> [!warning] Обмеження та ризики
> 
> - **Неузгоджений стан:** Без належної синхронізації дані можуть залишитися в стані, який не дозволить жодному з потоків отримати коректний результат.
>     
> - **Втрата швидкості:** Оскільки потоки змушені чекати на звільнення монітора, загальна пропускна здатність програми може впасти.
>     
> - **Втручання потоків:** Відсутність `synchronized` у критичних місцях призводить до непередбачуваної поведінки при асинхронному оновленні.
>     

---

> [!tip] Порівняння підходів
> 
> - **Synchronized Method ✅:** Простий у використанні, блокує весь об'єкт. Підходить, якщо весь функціонал методу є критичним.
>     
> - **Synchronized Block ✅:** Більш ефективний, оскільки дозволяє синхронізувати лише частину коду. Має меншу область блокування, що позитивно впливає на продуктивність.
>
