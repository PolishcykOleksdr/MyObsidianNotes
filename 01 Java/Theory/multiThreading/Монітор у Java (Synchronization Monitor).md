> **Монітор** — це високорівневий механізм синхронізації, який дозволяє потокам мати взаємовиключний доступ до спільних ресурсів. У Java кожен об'єкт неявно має асоційований з ним монітор, який використовується для керування доступом до критичних секцій.

---

> [!abstract]
> 
> ### Core Framework
> 
> Монітор реалізує концепцію взаємного виключення (**Mutual Exclusion**). Він гарантує, що в будь-який момент часу тільки один потік може виконувати код, захищений цим монітором. Основні стани:
> 
> - **Locked (Заблоковано):** Один потік "захопив" монітор об'єкта.
>     
> - **Unlocked (Розблоковано):** Монітор вільний для захоплення іншими потоками.
>     
> - **Wait Set (Набір очікування):** Потоки, що чекають на звільнення монітора або на певну умову (`wait()`).
>     

---

> [!info]
> 
> ### Key Insights
> 
> - **Implicit Lock.** У Java вам не потрібно створювати об'єкт монітора вручну. Будь-який `Object` може виступати в ролі замка.
>     
> - **Взаємодія з JVM.** Коли потік входить у `synchronized` блок, JVM виконує інструкцію `monitorenter`. При виході — `monitorexit`.
>     
> - **Reentrancy (Повторне входження).** Монітори в Java є реентерабельними. Якщо потік уже утримує монітор об'єкта, він може знову увійти в інший синхронізований метод того самого об'єкта без самоблокування.
>     
> - **Пам'ять.** Захоплення монітора не тільки блокує доступ, а й забезпечує "видимість" змін (Memory Visibility) — всі записи, зроблені одним потоком до виходу з монітора, будуть видимі іншому потоку після його входу.
>     

---

> [!example]
> 
> ### Практичне застосування
> 
> **Синхронізація на рівні об'єкта** У цьому прикладі монітором виступає сам екземпляр класу `Account`.
> 
> ```java
> public class Account {
>     private int balance = 1000;
> 
>     /* 1. Синхронізований метод: 
>        Потік автоматично захоплює монітор об'єкта 'this' при вході */
>     public synchronized void deposit(int amount) {
>         balance += amount;
>         // Коли баланс поповнено, повідомляємо потоки, що чекають (наприклад, у withdraw)
>         notifyAll(); 
>         System.out.println(Thread.currentThread().getName() + " поповнив на: " + amount);
>     } 
> 
>     /* 2. Синхронізований блок: 
>        Дозволяє заблокувати лише частину коду, що покращує продуктивність */
>     public void withdraw(int amount) {
>         synchronized (this) {
>             // Чекаємо, поки баланс стане достатнім
>             while (balance < amount) {
>                 try {
>                     System.out.println(Thread.currentThread().getName() + " чекає на поповнення...");
>                     /* Потік звільняє монітор і переходить у стан WAITING */
>                     wait(); 
>                 } catch (InterruptedException e) {
>                     Thread.currentThread().interrupt();
>                 }
>             }
>             balance -= amount;
>             System.out.println(Thread.currentThread().getName() + " зняв: " + amount);
>         }
>     }
> 
>     public synchronized int getBalance() {
>         return balance;
>     }
> }
> ```

---

> [!note] Методи керування монітором
> 
> Ці методи належать класу `Object` і можуть бути викликані **тільки** всередині синхронізованого контексту (інакше виникне `IllegalMonitorStateException`).
> 
> - **`wait()`** — Потік добровільно віддає монітор і засинає, поки інший потік не викличе `notify()` або `notifyAll()` на цьому ж об'єкті.
>     
> - **`notify()`** — Обирає один випадковий потік у черзі очікування цього монітора і переводить його у стан _Runnable_.
>     
> - **`notifyAll()`** — Прокидає **всі** потоки, що чекають. Це безпечніший варіант, оскільки він гарантує, що потрібний потік точно отримає сигнал.

---

> [!warning]
> 
> ### Обмеження та ризики
> 
> - **Starvation (Голодування).** Потік може нескінченно чекати на захоплення монітора, якщо інші "пріоритетні" потоки постійно його перехоплюють.
>     
> - **Overhead.** Операції з моніторами на рівні ОС (важковагові блокування) потребують багато ресурсів CPU на перемикання контексту.
>     
> - **Блокування на `this`.** Використання `synchronized(this)` робить замок публічним. Зовнішній код також може синхронізуватися на вашому об'єкті, що призведе до непередбачуваних затримок або Deadlock.
>     

---

> [!tip]
> 
> ### Коли використовувати
> 
> Так ✅:
> 
> - Коли потрібно забезпечити атомарність групи операцій.
>     
> - Для реалізації шаблону "Producer-Consumer" через `wait/notify`.
>     
> 
> Ні ❌:
> 
> - Якщо потрібно просто інкрементувати лічильник (використовуйте `AtomicInteger`).
>     
> - Якщо вистачає простої видимості змінної без блокування (використовуйте `volatile`).
>
