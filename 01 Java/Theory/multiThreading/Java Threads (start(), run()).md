> Маніпулювання потоками в Java здійснюється через методи класу `Thread`, які дозволяють керувати життєвим циклом та черговістю виконання завдань.

---

> [!abstract] Core Framework 
> Управління потоками базується на взаємодії між кодом розробника та **Thread Scheduler** (Планувальником потоків).
> 
> - **Thread Scheduler:** Частина JVM, яка вирішує, який потік отримає доступ до ЦП. Поведінка залежить від реалізації JDK та ОС, тому результат виконання може бути не детермінованим.
>     
> - **Метод `start()`:** Створює новий стек викликів для потоку та ініціює метод `run()`.
>     
> - **Метод `run()`:** Містить логіку, яку має виконати потік. При прямому виклику працює як звичайний метод у поточному стеку.
>     

---

> [!info] Key Insights 
> - **Новий стек vs Поточний стек.** Виклик `start()` виділяє окремий ресурс (стек) для паралельного виконання. Виклик `run()` безпосередньо виконує код у стеку `main`, що призводить до послідовного (синхронного) виконання. 
> - **Обмеження запуску.** Потік у Java є одноразовим об'єктом щодо стану виконання. Спроба викликати `start()` двічі на одному об'єкті призведе до винятку. 
> - **Механіка `sleep()`.** Це блокуюча операція. JVM звертається до ОС з проханням призупинити доступ потоку до ЦП. Після завершення часу ОС "будить" потік через переривання.

---

> [!example] Практичне застосування багатопотоковості
> 
> #### 1. Реалізація через Runnable
> 
> Для цього підходу необхідно явно створити об'єкт класу `Thread` та передати ваш `Runnable` як параметр у конструктор.
> 
> ```java
> public class Main {
>     public static void main(String[] args) {
>         MyRunnable myRunnable = new MyRunnable();
>         Thread thread = new Thread(myRunnable);
>         
>         // Надається новий стек і викликається метод run()
>         thread.start();
>     }
> }
> ```
> 
> #### 2. Реалізація через Thread
> 
> Використання методу `start()` ініціює створення нового потоку.
>
> ```java
> public class Main01 {
>     public static void main(String[] args) {
>         MyThread mt = new MyThread();
>         
>         // Створюється новий стек викликів
>         mt.start(); 
>     }
> }
> ```
> 
> ---
> 
> #### 3. Варіант БЕЗ багатопотоковості
> 
> Прямий виклик методу `run()` не створює новий потік, а виконує код послідовно.
> 
> ```java
> public class Main02 {
>     public static void main(String[] args) {
>         MyThread mt = new MyThread();
>         
>         // Працює в стеку основного потоку (main)
>         mt.run(); 
>     }
> }
> ```

---

> [!IMPORTANT] Головна відмінність: метод **`start()`** запускає новий потік виконання, тоді як **`run()`** просто виконує код методу в поточному потоці.

---

> [!warning] Ризики та обмеження 
> - **IllegalThreadStateException.** Виникає при спробі повторного виклику `start()` на одному і тому ж об'єкті потоку. *
> - **Блокування.** `Thread.sleep()` робить потік повністю неактивним для будь-яких завдань до моменту завершення таймера. 
> - **Тестування.** Використання `sleep()` ускладнює написання стабільних тестів, оскільки вносить часову залежність, яка може варіюватися між системами.

---

> [!tip] Ефективність 
> Використовуйте `Thread.sleep()` як засіб надання процесорного часу іншим потокам або програмам у системі. Проте пам'ятайте про ціну контекстного перемикання та блокування поточних ресурсів.
