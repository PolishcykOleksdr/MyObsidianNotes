> Механізми переривання в Java дозволяють безпечно сигналізувати потоку про необхідність зупинки або зміни діяльності, використовуючи систему статусів та виключень.

---

> [!abstract] Core Framework Управління перериваннями базується на трьох основних елементах:
> 
> - **`interrupt()`**: Метод-команда, що встановлює прапорець (flag) переривання для цільового потоку.
> - **`Thread.interrupted()`**: Статичний метод, що перевіряє, чи було перервано **поточний** потік, і — що важливо — **очищує** цей статус.
> - **`InterruptedException`**: Виключення, яке негайно викидається, якщо потік перебуває у стані сну (`sleep`) або очікування (`wait`, `join`) під час виклику `interrupt()`.

---

> [!info] Key Insights 
> - **Відкладений ефект.** Якщо потік зараз виконує обчислення (не спить), `interrupt()` просто ставить "мітку". Вона спрацює лише тоді, коли потік спробує заснути або самостійно перевірить свій статус. 
> - **Очищення статусу.** Метод `Thread.interrupted()` є руйнівним: перший виклик поверне `true` (якщо був interrupt), а другий — уже `false`. Це дозволяє потоку "обробити" переривання і продовжити роботу, якщо це передбачено логікою. 
> - **Марні виклики.** Якщо потік за весь час роботи жодного разу не перевірив статус переривання і не входив у стан сну, виклик `interrupt()` на ньому не матиме жодного ефекту.

---

> [!example] Практичне застосування
> 
> #### 1. Переривання `interrupt()` та безпечна зупинка через `isInterrupted()`
> 
> Найкращий спосіб зупинити потік, що виконує тривалу роботу в циклі. Ми не "вбиваємо" потік примусово, а просимо його завершитися.
> 
> ```java
> Thread worker = new Thread(() -> {
>     while (!Thread.currentThread().isInterrupted()) {
>         // Виконання корисної роботи
>         System.out.println("Обробка даних...");
>     }
>     System.out.println("Потік отримав сигнал і безпечно завершився.");
> });
> 
> worker.start();
> worker.interrupt(); // Встановлюємо прапорець переривання
> ```
>
> #### 2. Обробка `InterruptedException`
> 
> Коли потік "спить" (`sleep`) або чекає (`join`), виклик `interrupt()` викидає виключення. Важливо правильно його обробити, щоб не втратити сигнал про зупинку.
> 
> ```java
> try {
>     Thread.sleep(5000);
> } catch (InterruptedException e) {
>     // Після виключення прапорець скидається, тому його варто відновити
>     Thread.currentThread().interrupt(); 
>     System.out.println("Сон перервано, готуємось до виходу.");
> }
> ```
> 
> #### 3. Використання статичного `interrupted()`
> 
> Цей метод використовується, коли потрібно перевірити стан переривання і **одразу його скинути**, щоб потік міг продовжити роботу в штатному режимі після обробки сигналу.
> 
> ```java
> if (Thread.interrupted()) {
>     // Прапорець стає false після цієї перевірки
>     System.out.println("Сигнал отримано, статус скинуто. Виконуємо очищення...");
>     cleanup(); 
> }
> ```

### 📊 Резюме методів переривання

|**Метод**|**Тип**|**Дія**|**Особливість**|
|---|---|---|---|
|**`interrupt()`**|Об'єктний|Встановлює прапорець|Не зупиняє потік миттєво|
|**`isInterrupted()`**|Об'єктний|Перевіряє прапорець|**Не скидає** статус у `false`|
|**`interrupted()`**|Статичний|Перевіряє прапорець|**Скидає** статус у `false`|

---

> [!warning] Важливе 
> - **Логіка розробника.** Java не зупиняє потік силоміць. Метод `interrupt()` — це лише "ввічливе прохання". Ви самі повинні вирішити, чи завершувати цикл `for`, чи просто вивести повідомлення в консоль. 
> - **Втрата сигналу.** Якщо ви викликали `Thread.interrupted()` і не завершили роботу потоку, ви "проковтнули" сигнал. Інші частини коду більше не дізнаються, що потік намагалися перервати.

---

> [!tip] Коли використовувати 
> **interrupt()**: Завжди використовуйте цей метод замість застарілого `stop()`. Це єдиний безпечний спосіб зупинити потік, даючи йому можливість коректно закрити ресурси (файли, сокети) перед завершенням.
