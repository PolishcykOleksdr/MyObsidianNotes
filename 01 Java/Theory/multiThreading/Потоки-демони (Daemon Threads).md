> Потік-демон — це потік з низьким пріоритетом, який працює у фоновому режимі для надання послуг потокам-користувачам. Його головна особливість полягає в тому, що він не перешкоджає завершенню роботи JVM. (Найвідоміший приклад демон-потоку - `Garbage Collection`)

---

> [!abstract] Core Framework
> 
> Потоки-демони мають специфічний механізм життєвого циклу:
> 
> - **Залежність:** Життя демона залежить від потоків-користувачів. Коли всі потоки-користувачі завершуються, JVM автоматично зупиняє всі демон-потоки.
>     
> - **Службова роль:** Призначені для фонових задач (Garbage Collection, очищення кешу).
>     
> - **Пріоритет:** Це потоки з вкрай низьким пріоритетом (хоча за замовчуванням вони можуть успадковувати пріоритет батьківського потоку).
>     

---

> [!info] Key Insights
> 
> - **Некероване завершення:** JVM завершує роботу демон-потоку миттєво, щойно останній потік-користувач «помирає». JVM «все одно», чи закінчив демон свою роботу.
>     
> - **Метод `isDaemon()`:** Повертає `true`, якщо потік позначено як демон.
>     
> - **Успадкування:** Потоки, створені в іншому потоці, можуть набути пріоритет батьківського потоку (наприклад, пріоритет $5$ від `main`).
>     
> - **Пріоритет JVM:** Хоча демони вважаються низькорівневими, фактичний числовий пріоритет встановлюється згідно з ієрархією створення.
>     

---

> [!example] Практичне застосування
> 
> ```java
> public class MyThread extends Thread {
> 	public MyThread(String name) {
> 	    super(name);
> 	}
> 	public void run() {
> 	    /* Перевірка статусу через isDaemon() */
> 	    if (Thread.currentThread().isDaemon()) {
> 	        System.out.println(getName() + " is Daemon thread");
> 	    } else {
> 	        System.out.println(getName() + " is User thread");
> 	    }
>     
> 	    /* Виведення пріоритету */
> 	    System.out.println(getName() + " priority " 
> 	            + Thread.currentThread().getPriority());
> 	}
> }
> public class Main {
> 
> 	public static void main(String[] args) {
> 
> 		MyThread thread1 = new MyThread("Thread 1");
> 
> 		MyThread thread2 = new MyThread("Thread 2");
> 
> 		MyThread thread3 = new MyThread("Thread 3");
> 		
> 	    /* Встановлення статусу ДЕМОН */
> 	    thread1.setDaemon(true);
> 	    thread1.start();
> 	    
> 	    /* Потік-користувач (за замовчуванням false) */
> 	    thread2.setDaemon(false);
> 	    thread2.start();
> 	    
> 	    /* Ще один демон */
> 	    thread3.setDaemon(true);
> 	    thread3.start();
> 	}
> }
> ```

---

> [!warning] Обмеження та ризики
> 
> - **IllegalThreadStateException:** Метод `setDaemon(boolean on)` необхідно викликати **виключно до** виклику `start()`. Якщо потік уже активний, спроба змінити його статус призведе до критичної помилки.
>     
> - **Завершення роботи:** Не використовуйте демон-потоки для операцій введення-виведення (I/O) або запису в базу даних, оскільки JVM може закрити їх у будь-який момент, що призведе до втрати даних або пошкодження файлів.
>     

---

> [!tip] Коли використовувати
> 
> Так ✅:
> 
> - Задачі збирання сміття (GC).
>     
> - Моніторинг ресурсів або очищення кешу.
>     
> - Автоматичне завершення фонової роботи при закритті програми.
>     
> 
> Ні ❌:
> 
> - Задачі, які мають гарантовано завершити свою транзакцію.
>     
> - Бізнес-логіка, критична для цілісності даних.
>
