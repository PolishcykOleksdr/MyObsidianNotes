> Пакет `java.util.concurrent.atomic` пропонує набір класів, які підтримують безблокове потокобезпечне програмування для окремих змінних. Це гарантує атомарність операцій без використання явної синхронізації або блокування.

---

> [!abstract]
> 
> ### Core Framework
> 
> Пакет надає інструменти для роботи з примітивами (`int`, `long`, `boolean`), посиланнями на об'єкти та масивами. Основний механізм:
> 
> - **CAS (Compare and Swap)** — інструкція порівняння та заміни.
>     
> - **Неблокуючі алгоритми** — дозволяють уникнути ситуацій, коли потоки простоюють в очікуванні монітора об'єкта.
>     

---

> [!info]
> 
> ### Key Insights
> 
> - **Продуктивність.** Атомарні класи вважаються швидшими за традиційне блокування, оскільки вони мінімізують накладні витрати на перемикання контексту потоків.
>     
> - **Цілісність даних.** CAS забезпечує оновлення значення лише в тому випадку, якщо поточний стан відповідає очікуваному, що виключає стан перегонів.
>     
> - **Сфера застосування.** Найкраще підходять для лічильників, прапорців стану та генераторів ідентифікаторів у високонавантажених системах.
>

> [!example]
> 
> ### Практичне застосування: AtomicBoolean
> 
> **1. Метод compareAndSet(boolean expect, boolean update)** 
> Атомарно встановлює значення, якщо поточне значення дорівнює очікуваному.
> 
> ```java
> import java.util.concurrent.atomic.AtomicBoolean;
> 
> public class Main { 
>     public static void main(String[] args) { 
>         // Встановлюємо початкове значення в конструкторі
>         AtomicBoolean val = new AtomicBoolean(true); 
>         System.out.println("Initial value is " + val);
> 
>         /* Порівнюємо поточне значення з true. 
>            Якщо збігається — записуємо false.
>         */
>         boolean res = val.compareAndSet(true, false);
>         
>         if (res) {
>             System.out.println("Updated value is " + val);
>         } else {
>             System.out.println("The value was not updated (expected value mismatch)");
>         }
>     } 
> } 
> ```
> # Результат:
> ```
> Initial value is true 
> Updated value is false 
> ```
> 
> **2. Метод `getAndSet(newValue)`**
   Цей метод атомарно встановлює нове значення і одночасно повертає те, яке було до моменту заміни.
> 
> ```java
> import java.util.concurrent.atomic.AtomicBoolean;
> 
> public class Main { 
>     public static void main(String[] args) { 
>         AtomicBoolean val = new AtomicBoolean(true); 
>         System.out.println("Initial value is " + val);
> 
>         /* Атомарно встановлює false і повертає старе значення (true) 
>         */
>         boolean previousValue = val.getAndSet(false);
>         
>         if (previousValue) {
>             System.out.println("Updated value is " + val);
>             System.out.println("Previous value was " + previousValue);
>         } else {
>             System.out.println("The value was already false");
>         }
>     } 
> } 
> ```
> # Результат:
> ```
> Initial value is true 
> Updated value is false 
> Previous value is true 
> ```

> [!example]
> 
> ### Практичне застосування: AtomicInteger
>
> #### 1. Метод `addAndGet(int delta)`
> 
> Додає вказане значення до поточного та **одразу повертає результат**. Це еквівалент операції `+=`, але виконаний атомарно.
> 
> ```java
> import java.util.concurrent.atomic.AtomicInteger;
> 
> public class Main { 
>     public static void main(String[] args) { 
>         AtomicInteger val = new AtomicInteger(2); 
>         System.out.println("Initial value is " + val); 
> 
>         /* Атомарно додає 5 (2 + 5 = 7) та повертає результат */
>         int res = val.addAndGet(5); 
>         System.out.println("Updated value is " + res); 
>     } 
> } 
> ```
> ### Результат:
> ``` 
> Initial value is 2 
> Updated value is 7 
> ```
> 
> #### 2. Метод `compareAndSet(int expect, int update)`
> 
> Працює за принципом **CAS** (Compare-And-Swap). Оновлення відбувається лише тоді, коли поточне значення дорівнює очікуваному (`expect`).
> 
> ```java
> import java.util.concurrent.atomic.AtomicInteger;
> 
> public class Main { 
>     public static void main(String[] args) { 
>         AtomicInteger val = new AtomicInteger(0); 
>         System.out.println("Initial value is " + val); 
> 
>         /* Оновлення відбудеться тільки якщо поточне значення 0 */
>         boolean res = val.compareAndSet(0, 5); 
>         
>         if (res) {
>             System.out.println("Updated value is " + val); 
>         } else {
>             System.out.println("The value was not updated (was not 0)"); 
>         }
>     } 
> }
> ```
>  ### Результат:
> ``` 
> Initial value is 0 
> Updated value is 5 
> ```
> 
> #### 3. Інкремент та Декремент
> 
> Ці методи є атомарними аналогами операторів `++` та `--`. Вони бувають двох типів:
> 
> - **`getAndIncrement()`** — спочатку повертає значення, потім збільшує (як `i++`).
>     
> - **`incrementAndGet()`** — спочатку збільшує, потім повертає результат (як `++i`).
>     
> - Аналогічно для декременту: **`decrementAndGet()`** та **`getAndDecrement()`**.
> 
> ```java
> import java.util.concurrent.atomic.AtomicInteger;
> 
> public class Main { 
>     public static void main(String[] args) { 
>         String[] items = {"orange", "mango", "kiwi", "apple"};
> 
>         /* Приклад декременту: decrementAndGet() */
>         AtomicInteger decVal = new AtomicInteger(8); 
>         System.out.println("Initial: " + decVal);
>         
>         int res = decVal.decrementAndGet(); /* 8 -> 7 */
>         System.out.println("After decrement: " + res);
> 
>         /* Приклад лічильника для масиву: getAndIncrement() */
>         AtomicInteger counter = new AtomicInteger(1);
>         for (String item : items) {
>             // Виводимо поточне значення і ТІЛЬКИ ПІСЛЯ цього збільшуємо на 1
>             System.out.printf("%d) %s%n", counter.getAndIncrement(), item);
>         }
>     }
> }
> ```
>  ### Результат:
> ``` 
> Initial: 8 
> After decrement: 7
> 1) orange 
> 2) mango
> 3) kiwi
> 4) apple
> ```

---

> [!warning]
> 
> ### Обмеження та ризики
> 
> - **Ефект ABA.** Це ситуація, коли значення змінюється з A на B, а потім знову на A. CAS може "не помітити" проміжної зміни.
>     
> - **Single Variable Only.** Атомарні змінні захищають лише одну конкретну змінну. Якщо потрібно атомарно змінити дві змінні одночасно, доведеться використовувати `synchronized`.
>     
> - **Циклічні спроби.** При великій конкуренції (багато потоків б'ються за одну змінну), потоки можуть витрачати багато CPU-часу на повторні спроби CAS.
>     

---

> [!tip]
> 
> ### Коли використовувати
> 
> Так ✅:
> 
> - Глобальні лічильники або ID-генератори.
>     
> - Статусні прапорці (Flags).
>     
> - Високонавантажені алгоритми, де блокування (locking) створює вузьке місце.
>     
> 
> Ні ❌:
> 
> - Складна логіка, що охоплює кілька полів класу.
>     
> - Коли операції з даними тривають довго (краще використати блокування, щоб "заснути", а не "крутитися" в CAS-циклі).
>
