Представлення дерев у Java варіюється від високорівневих абстракцій у `java.util` до низькорівневих ручних структур, що дозволяють повністю контролювати пам'ять та зв'язки.

---

> [!abstract] Core Framework
> 
> Java надає готові рішення для роботи з деревами, які базуються на самобалансованих структурах:
> 
> - **TreeMap / TreeSet** — реалізації інтерфейсів `Map` та `Set`, що використовують механіку **Червоно-чорних дерев**.
>     
> - **TreeNode** — допоміжна структура (доступна в деяких внутрішніх пакетах та API), що описує вузол.
>     
> - **Часова складність**: Основні операції (пошук, вставка, видалення) виконуються за $O(\log n)$.
>     

---

> [!info] Key Insights
> 
> - **Автоматичне сортування.** `TreeMap` завжди зберігає ключі в їх природному порядку (або за заданим `Comparator`), що робить його ідеальним для отримання діапазонів даних.
>     
> - **Навігаційні методи.** На відміну від `HashMap`, `TreeMap` дозволяє шукати "сусідні" елементи через методи `ceilingKey`, `floorKey`, `firstKey` та `lastKey`.
>     
> - **Рекурсивна природа.** Ручне створення через клас `TreeNode` дозволяє реалізувати будь-яку топологію дерева, використовуючи рекурсію як основний інструмент обходу.
>     
> - **In-order Traversal.** Обхід "всередину" (ліво -> корінь -> право) у бінарному дереві пошуку завжди видає елементи у відсортованому порядку.
>     

---

> [!example] Практичне застосування

### 1. Використання стандартного TreeMap

Цей підхід використовує вбудовану логіку балансування JVM для забезпечення стабільної продуктивності.

```java
import java.util.*;

public class TreeMapExample {
    public static void main(String[] args) {
        /* Створення TreeMap: Ключі автоматично сортуються */
        TreeMap<Integer, String> treeMap = new TreeMap<>();

        treeMap.put(3, "Three");
        treeMap.put(1, "One");
        treeMap.put(2, "Two");

        /* Виведення: {1=One, 2=Two, 3=Three} */
        System.out.println("Елементи TreeMap: " + treeMap);

        /* Навігаційні операції */
        System.out.println("Значення для ключа 2: " + treeMap.get(2));
        
        /* Отримання меж */
        System.out.println("Мінімальний ключ: " + treeMap.firstKey()); // 1
        
        /* Найближчий більший або рівний ключ */
        int ceilKey = 2;
        System.out.println("Найближчий більший ключ за " + ceilKey + ": " + treeMap.ceilingKey(ceilKey));
    }
}
```

### 2. Ручне створення вузла (TreeNode)

Нижче наведено приклад побудови бінарного дерева "вручну" та його рекурсивного обходу.

```java
/* Клас, що представляє вузол дерева */
class TreeNode {
    int data;
    TreeNode left;
    TreeNode right;

    public TreeNode(int data) {
        this.data = data;
        left = null;
        right = null;
    }
}

public class ManualTreeCreation {
    /* Рекурсивний вивід: Ліве піддерево -> Корінь -> Праве піддерево */
    public static void printTree(TreeNode node) {
        if (node == null) return;

        printTree(node.left);  /* Крок 1: Рекурсія вліво */
        System.out.print(node.data + " "); /* Крок 2: Обробка даних */
        printTree(node.right); /* Крок 3: Рекурсія вправо */
    }

    public static void main(String[] args) {
        /* Побудова ієрархії вручну */
        TreeNode root = new TreeNode(1);
        root.left = new TreeNode(2);
        root.right = new TreeNode(3);
        root.left.left = new TreeNode(4);
        root.left.right = new TreeNode(5);

        /* Результат обходу: 4 2 5 1 3 */
        System.out.println("Дерево у порядку обходу \"всередину\":");
        printTree(root);
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Null Safety.** `TreeMap` не дозволяє використовувати `null` як ключ (викидає `NullPointerException`), оскільки неможливо порівняти `null` з іншими ключами.
>     
> - **Стек рекурсії.** При ручному обході дуже глибоких дерев (наприклад, незбалансованих) може виникнути `StackOverflowError`. Для таких випадків краще використовувати ітеративний підхід зі стеком.
>     
> - **Незмінність.** При ручній реалізації зміна `root.left = null` видаляє все ліве піддерево з пам'яті (Garbage Collector очистить його), що потребує обережного поводження з посиланнями.
>     

---

> [!tip] Коли використовувати
> 
> Так ✅:
> 
> - **TreeMap:** Котрібно зберігати пари ключ-значення з постійним сортуванням.
>     
> - **Custom TreeNode:** Для специфічних структур, таких як префіксні дерева (Trie) або дерева відрізків.
>     
> 
> Ні ❌:
> 
> - Якщо порядок елементів не має значення (використовуйте `HashMap`, він швидший — $O(1)$).
>     
> - Якщо дані часто змінюються і вимагають лише швидкого доступу за індексом.
>
