Ациклічний граф — це тип графа, у якому **відсутні цикли**. Це означає, що, почавши рух з будь-якої вершини і слідуючи вздовж ребер, ви ніколи не зможете повернутися в ту саму точку, не проходячи по одному й тому самому ребру двічі (або не повертаючись назад у випадку орієнтованих графів).

---

> [!abstract] Core Framework
> 
> Ациклічність — це структурне обмеження, яке гарантує ієрархічність або послідовність даних.
> 
> - **Неорієнтований ациклічний граф:** Більш відомий як **Дерево** (Tree) або **Ліс** (Forest — набір дерев). У ньому між будь-якими двома вершинами існує рівно один шлях.
>     
> - **Орієнтований ациклічний граф (DAG):** Граф, де ребра мають напрямок, але не утворюють замкнених петель.
>     

---

> [!info] Key Insights
> 
> - **Топологічне сортування.** Тільки ациклічні графи дозволяють впорядкувати вершини так, щоб для кожного ребра $u \to v$ вершина $u$ йшла перед $v$. Це критично для систем збирання проектів (Maven, Gradle) або планувальників задач.
>     
> - **Напівпорядок.** DAG визначає відношення часткового порядку між елементами.
>     
> - **Ієрархія.** Будь-яке дерево є ациклічним графом, але не кожен ациклічний граф є деревом (наприклад, DAG може мати кілька шляхів до однієї вершини, які не утворюють цикл).
>

> [!example] Практичне застосування

### Представлення DAG у Java

Для перевірки ациклічності зазвичай використовують алгоритм DFS з відстеженням поточного стека рекурсії.

```java
import java.util.*;

/* Приклад структури для DAG. 
   Важливо: при додаванні ребра потрібно перевіряти, чи не створює воно цикл.
*/
public class DAGChecker {
    /* Використання алгоритму DFS для виявлення циклів. 
       Якщо цикл знайдено — граф не є ациклічним.
    */
    private boolean hasCycle(int node, Map<Integer, List<Integer>> graph, 
                             Set<Integer> visited, Set<Integer> stack) {
        if (stack.contains(node)) return true; /* Цикл виявлено */
        if (visited.contains(node)) return false;

        visited.add(node);
        stack.add(node);

        for (Integer neighbor : graph.getOrDefault(node, new ArrayList<>())) {
            if (hasCycle(neighbor, graph, visited, stack)) return true;
        }

        stack.remove(node);
        return false;
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Динамічні зміни.** Додавання лише одного ребра в ациклічний граф може миттєво перетворити його на циклічний, що зламає алгоритми на кшталт топологічного сортування.
>     
> - **Складність перевірки.** Перевірка на ациклічність (Cycle Detection) має складність $O(V + E)$, що потрібно враховувати при роботі з великими даними.
>

> 
> Так ✅ (Завжди ациклічні):
> 
> - **Генеалогічне дерево:** (в біологічному сенсі ви не можете бути власним предком).
>     
> - **Системи контролю версій (Git):** комміти утворюють DAG.
>     
> - **Залежності в коді:** Клас А залежить від Б, Б від В (якщо з'явиться цикл — виникне помилка циклічної залежності).
>
