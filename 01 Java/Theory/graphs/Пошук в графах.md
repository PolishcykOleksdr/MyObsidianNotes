> Пошук у графах - це один з основних алгоритмів обробки графів, який використовується для знаходження шляхів між вершинами, перевірки наявності певних структур у графі, знаходження найкоротших шляхів, пошуку циклів та багато іншого. Тут наведено кілька найпоширеніших алгоритмів пошуку в графах:

---

> [!abstract] Core Framework
> 
> Алгоритми графів базуються на систематичному відвідуванні вершин ($V$) та ребер ($E$). Основні операції включають:
> 
> - **Traversing (Обхід)** — відвідування кожної вершини графа.
>     
> - **Pathfinding (Пошук шляху)** — знаходження маршруту між двома точками.
>     
> - **Sorting (Сортування)** — встановлення лінійної послідовності вершин на основі їхніх залежностей.
>     
> 
> Вибір алгоритму залежить від типу графа (орієнтований/неорієнтований), наявності ваг ребер та відсутності циклів (DAG).

---

> [!info] Key Insights
> 
> - **DFS vs BFS.** Вибір структури даних визначає стратегію: Стек (Stack) для занурення в глибину або Черга (Queue) для рівномірного розширення в ширину.
>     
> - **Обмеження Дейкстри.** Алгоритм не здатний коректно обробляти ребра з від'ємною вагою, що вимагає переходу до алгоритму Беллмана-Форда.
>     
> - **DAG (Directed Acyclic Graph).** Відсутність циклів дозволяє застосовувати специфічні методи, такі як топологічне сортування, які неможливі в циклічних графах.
>     
> - **Детекція циклів.** DFS є критично важливим для виявлення зациклень у системі (наприклад, взаємозалежності в пакетних менеджерах).
>     

---

> [!example] Практичне застосування

## 1. DFS (Depth-First Search) — Пошук у глибину

Алгоритм просувається максимально глибоко по гілці, перш ніж повернутися (backtracking).

```java
import java.util.*;

public class DepthFirstSearch {
    private int V; // Кількість вершин
    private LinkedList<Integer> adj[]; // Список суміжності

    // Конструктор класу
    DepthFirstSearch(int v) {
        V = v;
        adj = new LinkedList[v];
        for (int i = 0; i < v; ++i)
            adj[i] = new LinkedList();
    }

    // Додавання ребра до графа
    void addEdge(int v, int w) {
        adj[v].add(w);
    }

    // Рекурсивна функція DFS, що обходить вершини графа
    void DFSUtil(int v, boolean visited[]) {
        // Позначаємо поточну вершину як відвідану
        visited[v] = true;
        System.out.print(v + " ");

        // Рекурсивно відвідуємо всі суміжні вершини, які ще не були відвідані
        Iterator<Integer> i = adj[v].listIterator();
        while (i.hasNext()) {
            int n = i.next();
            if (!visited[n])
                DFSUtil(n, visited);
        }
    }

    // Публічний метод для початку обходу в глибину з вказаної вершини
    void DFS(int v) {
        // Ініціалізуємо масив для відстеження відвіданих вершин
        boolean visited[] = new boolean[V];

        // Викликаємо рекурсивну функцію DFSUtil для обходу графа
        DFSUtil(v, visited);
    }

    public static void main(String args[]) {
        DepthFirstSearch g = new DepthFirstSearch(4);

        // Додаємо ребра до графа
        g.addEdge(0, 1);
        g.addEdge(0, 2);
        g.addEdge(1, 2);
        g.addEdge(2, 0);
        g.addEdge(2, 3);
        g.addEdge(3, 3);

        System.out.println("DFS починаючи з вершини 2:");
        g.DFS(2);
    }
}
```

### Результат:

```
DFS починаючи з вершини 2:
2 0 1 3
```

Результат показує порядок, у якому вершини були відвідані під час обходу в глибину з початкової вершини 2.
## 2. BFS (Breadth-First Search) — Пошук у ширину

Розглядає спочатку всіх безпосередніх сусідів, перш ніж переходити на наступний рівень віддаленості.

```java
import java.util.*;

public class BreadthFirstSearch {
    private int V; // Кількість вершин
    private LinkedList<Integer> adj[]; // Список суміжності

    // Конструктор класу
    BreadthFirstSearch(int v) {
        V = v;
        adj = new LinkedList[v];
        for (int i = 0; i < v; ++i)
            adj[i] = new LinkedList();
    }

    // Додавання ребра до графа
    void addEdge(int v, int w) {
        adj[v].add(w);
    }

    // Публічний метод для початку обходу в ширину з вказаної вершини
    void BFS(int s) {
        // Ініціалізуємо масив для відстеження відвіданих вершин
        boolean visited[] = new boolean[V];

        // Створюємо чергу для зберігання вершин, які маємо відвідати
        LinkedList<Integer> queue = new LinkedList<Integer>();

        // Позначаємо поточну вершину як відвідану і додаємо її до черги
        visited[s] = true;
        queue.add(s);

        while (queue.size() != 0) {
            // Видаляємо вершину з черги та виводимо її
            s = queue.poll();
            System.out.print(s + " ");

            // Отримуємо всі суміжні вершини поточної вершини
            // Якщо суміжна вершина ще не відвідана, позначаємо її як відвідану і додаємо до черги
            Iterator<Integer> i = adj[s].listIterator();
            while (i.hasNext()) {
                int n = i.next();
                if (!visited[n]) {
                    visited[n] = true;
                    queue.add(n);
                }
            }
        }
    }

    public static void main(String args[]) {
        BreadthFirstSearch g = new BreadthFirstSearch(4);

        // Додаємо ребра до графа
        g.addEdge(0, 1);
        g.addEdge(0, 2);
        g.addEdge(1, 2);
        g.addEdge(2, 0);
        g.addEdge(2, 3);
        g.addEdge(3, 3);

        System.out.println("BFS починаючи з вершини 2:");
        g.BFS(2);
    }
}
```

### Результат:

```
BFS починаючи з вершини 2:
2 0 3 1
```

Результат показує порядок, у якому вершини були відвідані під час обходу в ширину з початкової вершини 2.
## 3. Пошук найкоротшого шляху (Dijkstra / Bellman-Ford)

**Алгоритми пошуку найкоротшого шляху (наприклад, Дейкстри або Алгоритм Беллмана-Форда)**:

- Ці алгоритми використовуються для знаходження найкоротшого шляху між двома вершинами у вагованому графі.
- Дейкстри - працює для графів з невід'ємними вагами ребер.
- Алгоритм Беллмана-Форда - працює для графів з можливими негативними вагами ребер, а також для виявлення циклів з від'ємною сумою ваг.
### Приклад алгоритму Дейкстри:

```java
import java.util.*;

public class DijkstraAlgorithm {
    private static final int V = 9; // Кількість вершин у графі

    // Знаходження вершини з найменшою відстанню до початкової вершини
    int minDistance(int dist[], Boolean sptSet[]) {
        int min = Integer.MAX_VALUE, min_index = -1;

        for (int v = 0; v < V; v++)
            if (!sptSet[v] && dist[v] <= min) {
                min = dist[v];
                min_index = v;
            }

        return min_index;
    }

    // Виведення результату пошуку найкоротших відстаней
    void printSolution(int dist[]) {
        System.out.println("Вершина \t Відстань від початкової вершини");
        for (int i = 0; i < V; i++)
            System.out.println(i + " \t\t " + dist[i]);
    }

    // Реалізація алгоритму Дейкстри для пошуку найкоротших шляхів у графі
    void dijkstra(int graph[][], int src) {
        int dist[] = new int[V]; // Масив для зберігання найкоротших відстаней

        // Масив для відстеження вершин, які вже включені до найкоротших шляхів
        Boolean sptSet[] = new Boolean[V];

        // Ініціалізація всіх відстаней як нескінченні, а також позначаємо всі вершини як невідвідані
        for (int i = 0; i < V; i++) {
            dist[i] = Integer.MAX_VALUE;
            sptSet[i] = false;
        }

        // Відстань від початкової вершини до самої себе завжди дорівнює 0
        dist[src] = 0;

        // Знаходимо найкоротший шлях для всіх вершин
        for (int count = 0; count < V - 1; count++) {
            int u = minDistance(dist, sptSet); // Вибираємо вершину з найменшою відстанню

            sptSet[u] = true; // Позначаємо вибрану вершину як відвідану

            // Оновлюємо відстані для сусідніх вершин вибраної вершини, якщо вони ще не відвідані і нова відстань коротша
            for (int v = 0; v < V; v++)
                if (!sptSet[v] && graph[u][v] != 0 &&
                        dist[u] != Integer.MAX_VALUE &&
                        dist[u] + graph[u][v] < dist[v])
                    dist[v] = dist[u] + graph[u][v];
        }

        // Виводимо результат
        printSolution(dist);
    }

    public static void main(String[] args) {
        // Приклад графу з вагами ребер
        int graph[][] = new int[][]{
                {0, 4, 0, 0, 0, 0, 0, 8, 0},
                {4, 0, 8, 0, 0, 0, 0, 11, 0},
                {0, 8, 0, 7, 0, 4, 0, 0, 2},
                {0, 0, 7, 0, 9, 14, 0, 0, 0},
                {0, 0, 0, 9, 0, 10, 0, 0, 0},
                {0, 0, 4, 14, 10, 0, 2, 0, 0},
                {0, 0, 0, 0, 0, 2, 0, 1, 6},
                {8, 11, 0, 0, 0, 0, 1, 0, 7},
                {0, 0, 2, 0, 0, 0, 6, 7, 0}
        };
        DijkstraAlgorithm t = new DijkstraAlgorithm();
        t.dijkstra(graph, 0);
    }
}
```

### Результат:
```
Вершина      Відстань від початкової вершини
0            0
1            4
2            12
3            19
4            21
5            11
6            9
7            8
8            14
```

Результат показує найкоротші відстані від початкової вершини (у даному випадку від вершини 0) до всіх інших вершин графу.

## 4.Алгоритм пошуку в ширину в орієнтованому ациклічному графі (Topological Sorting):

Використовується для визначення порядку вершин у графі без циклів, де кожне ребро вказує на спадне зв'язок.

```java
import java.util.*;

public class TopologicalSorting {
    private int V; // Кількість вершин у графі
    private LinkedList<Integer> adj[]; // Список суміжності

    // Конструктор класу
    TopologicalSorting(int v) {
        V = v;
        adj = new LinkedList[v];
        for (int i = 0; i < v; ++i)
            adj[i] = new LinkedList();
    }

    // Додавання ребра до графа
    void addEdge(int v, int w) {
        adj[v].add(w);
    }

    // Виконання BFS для знаходження топологічного сортування графа
    void topologicalSort() {
        Stack<Integer> stack = new Stack<Integer>();

        // Масив для відстеження відвіданих вершин
        boolean visited[] = new boolean[V];

        // Маркуємо всі вершини як невідвідані
        for (int i = 0; i < V; i++)
            if (!visited[i])
                topologicalSortUtil(i, visited, stack);

        // Виводимо вершини у порядку топологічного сортування
        System.out.println("Топологічне сортування графа:");
        while (!stack.isEmpty())
            System.out.print(stack.pop() + " ");
    }

    // Допоміжний метод для виконання рекурсивного пошуку в глибину
    void topologicalSortUtil(int v, boolean visited[], Stack<Integer> stack) {
        visited[v] = true;
        Integer i;

        // Рекурсивно викликаємо функцію для всіх сусідніх вершин поточної вершини
        Iterator<Integer> it = adj[v].iterator();
        while (it.hasNext()) {
            i = it.next();
            if (!visited[i])
                topologicalSortUtil(i, visited, stack);
        }

        // Після відвідання всіх сусідніх вершин, поточну вершину додаємо у стек
        stack.push(new Integer(v));
    }

    public static void main(String args[]) {
        TopologicalSorting g = new TopologicalSorting(6);
        g.addEdge(5, 2);
        g.addEdge(5, 0);
        g.addEdge(4, 0);
        g.addEdge(4, 1);
        g.addEdge(2, 3);
        g.addEdge(3, 1);

        System.out.println("Topological Sorting of the given graph:");
        g.topologicalSort();
    }
}
```
## Результат:

```
Топологічне сортування графа:
5 4 2 3 1 0
```

Результат показує порядок, у якому вершини були відсортовані у топологічному порядку. Кожна вершина розташована в такому порядку, щоб усі її сусідні вершини були розташовані перед нею.
## 5. Пошук у глибину в орієнтованому ациклічному графі (Depth-First Search on Directed Acyclic Graph - DFS on DAG):

Використовується для знаходження найкоротших шляхів у напрямленому ациклічному графі.

```java
import java.util.*;

public class DFSDirectedAcyclicGraph {
    private int V; // Кількість вершин у графі
    private LinkedList<Integer> adj[]; // Список суміжності

    // Конструктор класу
    DFSDirectedAcyclicGraph(int v) {
        V = v;
        adj = new LinkedList[v];
        for (int i = 0; i < v; ++i)
            adj[i] = new LinkedList();
    }

    // Додавання ребра до графа
    void addEdge(int v, int w) {
        adj[v].add(w);
    }

    // Рекурсивне виконання DFS
    void DFSUtil(int v, boolean visited[]) {
        // Позначаємо поточну вершину як відвідану
        visited[v] = true;
        System.out.print(v + " ");

        // Рекурсивно відвідуємо всі сусідні вершини, які ще не були відвідані
        Iterator<Integer> i = adj[v].listIterator();
        while (i.hasNext()) {
            int n = i.next();
            if (!visited[n])
                DFSUtil(n, visited);
        }
    }

    // Виконання DFS починаючи з усіх вершин графа
    void DFS() {
        // Ініціалізуємо масив для відстеження відвіданих вершин
        boolean visited[] = new boolean[V];

        // Виконуємо DFS для всіх вершин графа, що ще не були відвідані
        for (int i = 0; i < V; ++i)
            if (!visited[i])
                DFSUtil(i, visited);
    }

    public static void main(String args[]) {
        DFSDirectedAcyclicGraph g = new DFSDirectedAcyclicGraph(6);
        g.addEdge(0, 1);
        g.addEdge(0, 2);
        g.addEdge(1, 3);
        g.addEdge(1, 4);
        g.addEdge(2, 4);
        g.addEdge(3, 5);
        g.addEdge(4, 5);

        System.out.println("DFS у напрямленому ациклічному графі (DAG):");
        g.DFS();
    }
}
```

## Результат:

```
DFS у напрямленому ациклічному графі (DAG):
0 1 3 4 5 2
```

Результат показує порядок, у якому вершини були відвідані під час обходу в глибину у напрямленому ациклічному графі (DAG).

---

> [!warning] Обмеження та ризики
> 
> - **Від'ємні цикли.** Якщо у графі є цикл з від'ємною сумою ваг, алгоритми найкоротшого шляху можуть зациклитися (крім Беллмана-Форда, який це виявляє).
>     
> - **Пам'ять.** BFS може споживати значну кількість пам'яті для зберігання черги на широких графах ($O(V)$).
>     
> - **Стек викликів.** При рекурсивній реалізації DFS на дуже глибоких графах можливе переповнення стека (`StackOverflowError`).
>     

---

> [!tip] Коли використовувати
> 
> Так ✅:
> 
> - **BFS:** якщо потрібно знайти найкоротший шлях у кроках (невагований граф).
>     
> - **DFS:** для аналізу зв'язності та топологічних перевірок.
>     
> - **Bellman-Ford:** якщо в даних можуть бути фінансові транзакції (де можливі "негативні" ваги/знижки).
>     
> 
> Ні ❌:
> 
> - **Dijkstra:** якщо є ризик появи негативних ваг ребер.
>     
> - **Topological Sort:** якщо граф містить хоча б один цикл (алгоритм не спрацює).
>
