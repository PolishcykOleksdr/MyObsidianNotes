> **Буферизація (Buffering)** — це метод оптимізації введення-виведення, при якому дані спочатку накопичуються у тимчасовій ділянці пам'яті (буфері), а потім передаються великим блоком. Це значно швидше, ніж звертатися до диска чи мережі для кожного окремого байта чи символа.

---

> [!abstract] Core Framework
>  Основні класи для буферизації тексту:
> 
> - **BufferedWriter**: Огортає будь-який об'єкт `Writer`. Накопичує символи у внутрішньому масиві та записує їх у джерело лише тоді, коли буфер заповнюється або викликається метод `flush()`.
>     
> - **BufferedReader**: Огортає об'єкт `Reader`. Зчитує великий блок даних із джерела у пам'ять за один раз, що дозволяє програмі швидко отримувати дані з RAM.
>     

---

> [!info] Key Insights
> 
> - **Зменшення системних викликів**: Замість 1000 звернень до диска для запису 1000 символів, `BufferedWriter` може зробити лише одне звернення, що критично підвищує продуктивність.
>     
> - **Метод readLine()**: `BufferedReader` додає зручну можливість читати текст цілими рядками (до символу перенесення рядка `\n` або `\r`), що неможливо зробити базовим `FileReader`.
>     
> - **Метод newLine()**: `BufferedWriter` надає платформонезалежний спосіб запису перенесення рядка.
>

> [!example] Практичне застосування: BufferedWriter
> 
> Java
> 
> ```java
> import java.io.BufferedWriter;
> import java.io.FileWriter;
> import java.io.IOException;
> 
> public class Main {
>     private final static String BASE_PATH = "/Username/Files/";
> 
>     public static void main(String[] args) {
>         String fileName = "myfile.txt";
>         String myText = "Some information.";
>         System.out.println(handleFile(fileName, myText));
>     }
> 
>     private static String handleFile(String fileName, String text) {
>         /* BufferedWriter огортає FileWriter для підвищення ефективності */
>         try (BufferedWriter buffWriter = new BufferedWriter(
>                 new FileWriter(BASE_PATH + fileName))) {
>             buffWriter.write(text);
>             return "Success.";
>         } catch (IOException ex) {
>             return "Exception: " + ex.getMessage();
>         }
>     }
> }
> ```

---

> [!example] Практичне застосування: BufferedReader
> 
> Java
> 
> ```java
> import java.io.BufferedReader;
> import java.io.FileReader;
> import java.io.IOException;
> 
> public class Main {
>     private final static String BASE_PATH = "/Username/Files/";
> 
>     public static void main(String[] args) {
>         String path = BASE_PATH + "myfile.txt";
>         System.out.println(handleFile(path));
>     }
> 
>     private static String handleFile(String path) {
>         try (BufferedReader buffReader = new BufferedReader(new FileReader(path))) {
>             int sym;
>             StringBuilder stringBuilder = new StringBuilder();
>             /* Читання з буфера в пам'яті відбувається миттєво */
>             while ((sym = buffReader.read()) != -1) {
>                 stringBuilder.append((char) sym);
>             }
>             return stringBuilder.toString();
>         } catch (IOException ex) {
>             return "Error: " + ex.getMessage();
>         }
>     }
> }
> ```

---

> [!warning] Важливе про буфери
> 
> - **Flush**: Якщо ви записуєте дані через `BufferedWriter` і програма завершується некоректно (без закриття потоку), дані в буфері можуть бути втрачені. Використання `try-with-resources` автоматично викликає `close()`, який у свою чергу робить `flush()`.
>     
> - **Розмір буфера**: За замовчуванням розмір становить 8 КБ, що є оптимальним для більшості завдань, але його можна налаштувати в конструкторі.
>     
> - **Ланцюжки (Chaining)**: Буфери завжди використовуються як "обгортки" навколо базових потоків (`new BufferedReader(new FileReader(...))`).
>     

---

> [!tip] Коли використовувати 
> Так ✅:
> 
> - Завжди при роботі з файлами на диску, навіть якщо вони невеликі.
>     
> - Коли потрібно зчитувати файл рядково (`readLine()`).
>     
> 
> Ні ❌:
> 
> - Якщо ви працюєте з `StringBuilder` або масивами в пам'яті (`StringReader`), додаткова буферизація зазвичай не потрібна.
>
