> Робота з бінарними даними (зображеннями, аудіо, виконуваними файлами або сирим текстом) в Java здійснюється через ієрархію байтових потоків. В основі лежать два абстрактні класи: **InputStream** та **OutputStream**.

---

> [!abstract] Core Framework
> 
> Базові компоненти для роботи з байтами:
> 
> - **InputStream**: Абстрактний суперклас для всіх вхідних потоків байтів. Визначає метод `read()`.
>     
> - **OutputStream**: Абстрактний суперклас для всіх вихідних потоків байтів. Визначає метод `write()`.
>     
> - **FileInputStream**: Спеціалізація для читання байтів безпосередньо з файлу.
>     
> - **FileOutputStream**: Спеціалізація для запису байтів у файл.
>     

---

> [!info] Key Insights
> 
> - **Сирі дані**: Ці класи оперують байтами (`byte`), а не символами. Хоча текст можна записати через байтові потоки, для складної роботи з текстом (кодування, Unicode) краще підходять символьні потоки (`Reader`/`Writer`).
>     
> - **Кінець потоку**: Метод `read()` повертає ціле число (`int`). Якщо досягнуто кінця файлу, повертається значення $-1$.
>     
> - **Автоматизація**: Використання конструкції `try-with-resources` є кращою практикою, ніж ручне закриття у блоці `finally`, оскільки вона гарантує звільнення ресурсів навіть при виникненні помилок.
>     

---

> [!example] Практичне застосування: Запис у файл (FileOutputStream)
> 
> Java
> 
> ```java
> import java.io.FileOutputStream;
> import java.io.IOException;
> 
> public class Main {
>     /* Шлях до файлу (приклад для Unix-подібних систем або кореня диска) */
>     public final static String BASE_PATH = "/Username/Files/";
> 
>     public static void main(String[] args) {
>         String myFile = BASE_PATH + "myfile.txt";
> 
>         try {
>             FileOutputStream fout = new FileOutputStream(myFile);
>             String myText = "Some information.";
>             
>             /* Рядок необхідно перетворити на масив байтів перед записом */
>             byte[] someBytes = myText.getBytes();
>             
>             fout.write(someBytes);
>             fout.close(); /* Важливо закрити потік для збереження даних */
>             
>             System.out.println("Success :)");
>         } catch (IOException e) {
>             System.out.println("Exception: " + e.getMessage());
>             System.exit(0);
>         }
>     }
> }
> ```

---

> [!example] Практичне застосування: Читання з файлу (FileInputStream)
> 
> Java
> 
> ```java
> import java.io.FileInputStream;
> import java.io.IOException;
> 
> public class Main {
>     public final static String BASE_PATH = "/Username/Files/";
> 
>     public static void main(String[] args) {
>         String path = BASE_PATH + "myfile.txt";
>         FileInputStream fin = null;
> 
>         try {
>             fin = new FileInputStream(path);
>             int i;
>             /* Читаємо по одному байту до кінця файлу (-1) */
>             while ((i = fin.read()) != -1) {
>                 /* Конвертація байта (int) у символ для виводу */
>                 System.out.print((char) i);
>             }
>         } catch (IOException ex) {
>             System.out.println(ex.getMessage());
>         } finally {
>             /* Ручне закриття потоку в блоці finally */
>             try {
>                 if (fin != null) fin.close();
>             } catch (IOException ex) {
>                 System.out.println("Error closing stream: " + ex.getMessage());
>             }
>         }
>     }
> }
> ```

---

> [!warning] Важливе про роботу з файлами
> 
> - **Перезапис за замовчуванням**: `new FileOutputStream(path)` перезаписує файл. Щоб дописувати в кінець, використовуйте конструктор `new FileOutputStream(path, true)`.
>     
> - **Кодування**: При перетворенні `(char) i` слід пам'ятати, що цей метод коректно працює лише для ASCII. Для кирилиці (UTF-8) один символ може займати 2-4 байти, тому просте приведення типів у циклі `while` може вивести "биті" символи.
>     
> - **Продуктивність**: Читання по одному байту через `read()` створює велике навантаження. Для оптимізації варто використовувати `BufferedInputStream`.
>
