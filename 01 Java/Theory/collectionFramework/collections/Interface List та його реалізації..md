> Реалізації: ArrayList, LinkedList

> `Interface List<E>` , де `E` – тип елементів у цьому списку, містить, засновані на індексі, елементи, методи для вставки та видалення елементів.**
> 
> Популярні реалізації `Interface List` : `Class ArrayList` , `Class LinkedList`.

---

> [!info] **ArrayList**
> `Class ArrayList<E>`, де `E` – тип елементів у цьому списку, є простим списком об'єктів. Список, який дуже схожий на масив, але має довільну довжину та цілий ряд методів, які дозволяють додавати, видаляти, замінювати елементи списку у будь-який час та у будь-якому місці.
>

# Основні методи:

> [!info] **Методи**:
> 1. Додавання елементів
> 
> - **`add(element)`**: додає елемент у кінець списку.
>     
> - **`add(index, element)`**: вставляє елемент у конкретну позицію.
>     
> - **`addAll(collection)`**: додає всі елементи з іншої колекції в кінець вашого списку.
>     
>  2. Видалення та очищення
> 
> - **`remove(index)`**: видаляє елемент за його номером.
>     
> - **`remove(object)`**: знаходить і видаляє перший такий об'єкт (наприклад, видалити слово "apple").
>     
> - **`clear()`**: повністю очищає список, роблячи його порожнім.
>     
> - **`removeAll(collection)`**: видаляє всі елементи, які є в іншому списку.
>     
> 
>  3. Отримання та пошук
> 
> - **`get(index)`**: повертає елемент, який стоїть на вказаному місці.
>     
> - **`indexOf(object)`**: каже, під яким номером вперше зустрічається цей елемент (або -1, якщо його немає).
>     
> - **`contains(object)`**: просто відповідає "так" або "ні", чи є такий елемент у списку.
>     
> - **`size()`**: повертає кількість елементів у списку.
>     
> 
>  4. Зміна та підмножини
> 
> - **`set(index, element)`**: замінює елемент у вказаній позиції на новий (як перезапис комірки).
>     
> - **`subList(from, to)`**: вирізає шматочок списку від одного індексу до іншого.
>     
> 
>  5. Перевірка стану
> 
> - **`isEmpty()`**: повертає `true`, якщо в списку нуль елементів.

> [!info] Специфічні методи для `LinkedList`:
> 
> Оскільки `LinkedList` працює як ланцюжок, у нього є швидкі методи для роботи з краями:
> 
> - **`addFirst()` / `addLast()`**: додати на самий початок або в самий кінець.
>     
> - **`getFirst()` / `getLast()`**: глянути, що лежить першим або останнім.
>     
> - **`removeFirst()` / `removeLast()`**: забрати (видалити) перший або останній елемент.

---

> [!attention] Різниця між Array та ArrayList
> Масив (`Array`) – структура даних **фіксованого розміру**, що містить колекцію елементів **даних аналогічного типу**. `ArrayList` - структура даних **динамічного розміру**. Необхідно вказати розмір масиву при створенні об'єкта `Array`, а для `ArrayList` не потрібно вказувати його розмір.

# Приклад:

### Створення масиву та ArrayList:

```java
String[] products = new String[10];
List<String> productList = new ArrayList<>();
// або
ArrayList<String> productList = new ArrayList<>();
```
### Масив може містити примітиви (int, float тощо), а також об'єкт класу, що базується на визначенні масиву. ArrayList підтримує лише об'єкти:

```java
double[] prices = new double[10];
Product[] poductArray = new Product[10];
List<Double> prices = new ArrayList<>();
List<Product> productList = new ArrayList<>();
// або
ArrayList<Double> prices = new ArrayList<>();
ArrayList<Product> productList = new ArrayList<>();
```

---

> [!important] В чому різниця між
> ```java
> List<String> productList = new ArrayList<>();
> ```
> та 
> ```java
> ArrayList<String> productList = new ArrayList<>();
> ```
> 
> Використовуючи загальний інтерфейс `List`, можна легко замінити `ArrayList` на інший тип списку (наприклад, `LinkedList`), не змінюючи логіку в усій програмі. Це робить код універсальним, оскільки можна сказати програмі: "Мені потрібен будь-який список", а не "Мені потрібен тільки цей конкретний вид".

# Time complexity

| Method                      | ArrayList (Best / Worst) | LinkedList (Best / Worst) |   Array(Best / Worst)   |
| --------------------------- | :----------------------: | :-----------------------: | :---------------------: |
| get(int index)              |      $O(1) / O(1)$       |       $O(1) / O(n)$       |      $O(1) / O(1)$      |
| add(E element) (в кінець)   |      $O(1) / O(n)$       |       $O(1) / O(1)$       | N/A (фіксований розмір) |
| add(int index, E element)   |      $O(1) / O(n)$       |       $O(1) / O(n)$       |           N/A           |
| remove(int index)           |      $O(1) / O(n)$       |       $O(1) / O(n)$       |           N/A           |
| set(int index, E element)   |      $O(1) / O(1)$       |       $O(1) / O(n)$       |      $O(1) / O(1)$      |
| ListIterator.add(E element) |      $O(1) / O(n)$       |       $O(1) / O(n)$       |      $O(1) / O(n)$      |

> [!quote] Що де краще?
> **ArrayList** (та звичайний **Array**) найкраще підходять для швидкого доступу до елементів за індексом та додавання в кінець списку **(get, set, add (в кінець)).**
> 
> **LinkedList** вигідніший для постійних вставок або видалень у середину чи початок списку, оскільки він просто переписує посилання між вузлами без зсуву інших елементів **(add (за індексом), remove, ListIterator.add).**
