> **LinkedHashMap** — це гібридна структура даних, яка поєднує можливості хеш-таблиці та двобічно зв’язаного списку. Вона забезпечує передбачуваний порядок ітерації елементів, успадковуючи при цьому високу швидкість доступу від `HashMap`.

---

> [!abstract] Core Framework
> 
> `LinkedHashMap` розширює `HashMap`, додаючи механізм відстеження черговості елементів:
> 
> - **Порядок ітерацій**: За замовчуванням елементи обходяться у порядку їх вставки (Insertion order).
>     
> - **Структура**: Підтримує подвійний зв’язаний список (doubly-linked list), що проходить крізь усі записи.
>     
> - **Продуктивність**: Операції `add()`, `contains()`, `remove()` мають константну складність $O(1)$.
>     
> - **Пам'ять**: Потребує більше місця, ніж `HashMap`, для зберігання посилань на попередній та наступний елементи списку.
>     

---

> [!info] Key Insights: Механіка впорядкування
> 
> - **Зв'язки між елементами**: На відміну від `HashMap`, де порядок залежить від хеш-коду та розподілу по бакетах, `LinkedHashMap` пам'ятає, хто був доданий першим, а хто — останнім.
>     
> - **Продуктивність ітерації**: Ітерація по `LinkedHashMap` залежить від кількості елементів ($n$), тоді як у `HashMap` вона залежить від загальної ємності таблиці (кількості бакетів).
>     
> - **Використання**: Це ідеальний вибір для створення кешів (наприклад, LRU-кеш), де важливо знати порядок звернення до об'єктів.
>     

---

> [!example] Практичне застосування: Збереження порядку
> 
> **Вставка та автоматичне впорядкування:**
> 
> ```java
> Map<String, String> map = new LinkedHashMap<>();
> map.put("abc1", "orange");
> map.put("dvc3", "apricot");
> map.put("dvc3", "plum"); /* Оновлення значення не змінює позицію ключа в списку */
> map.put(null, "pepper");
> 
> /* Результат ітерації завжди буде: abc1 -> dvc3 -> null */
> 
> **Безпечне отримання значень:**
> /* Метод поверне "mango", якщо ключа "mye4" не існує в мапі */
> String val = map.getOrDefault("mye4", "mango"); 
> ```

---

> [!example] Практичне застосування: Модифікація та видалення
> 
> **Умовна заміна (`replace`):**
> 
> 
> ```java
> /* Замінить значення лише за умови повної відповідності ключа та старого значення */
> boolean replaced = map.replace("abc1", "orange", "avocado"); 
> ```
> 
> **Точкове видалення (`remove`):**
> 
> 
> ```java
> /* Видаляє запис, якщо ключ "abc1" асоційований саме з "orange" */
> map.remove("abc1", "orange"); 
> ```

# Основні методи:
### **Додавання (Put)**

- **`put(K key, V value)`** — додає пару "ключ-значення". Якщо такий ключ уже був, він просто **записує нове значення поверх старого**.
    
- **`putIfAbsent(K key, V value)`** — додає пару лише тоді, **коли такого ключа ще немає** (або там лежить `null`). Якщо ключ уже є, метод нічого не змінює.
    
- **`putAll(Map m)`** — бере всі дані з іншої мапи та «засипає» їх у поточну. Якщо ключі збігаються, дані з нової мапи **замінять** ваші старі.
### **Отримання (Get)**

- **`get(Object key)`** — дістає значення за ключем. Якщо такого ключа не існує — видасть `null`.
    
- **`getOrDefault(Object key, V defaultValue)`** — дістає значення, але якщо ключа немає, замість `null` видасть **ваше «запасне» значення**, яке ви вкажете.
### **Очищення (Clear/Empty)**

- **`clear()`** — повністю видаляє всі записи. Мапа стає порожньою.
    
- **`isEmpty()`** — просто перевіряє: чи є в мапі хоч щось? (Результат: `true` — порожньо, `false` — є дані).
### **Пошук (Contains)**

- **`containsKey(Object key)`** — запитує: "Чи є у нас такий ключ?".
    
- **`containsValue(Object value)`** — запитує: "Чи є у нас десь (хоч під якимсь ключем) таке значення?".
### **Заміна (Replace)**

- **`replace(K key, V value)`** — оновлює значення для ключа, але **тільки якщо цей ключ уже існує**. Якщо ключа немає — нічого не станеться.
    
- **`replace(K key, V oldValue, V newValue)`** — замінить старе значення на нове тільки за умови, що **старе значення точно збігається** з тим, що ви вказали.
### **Видалення (Remove)**

- **`remove(Object key)`** — видаляє ключ і його значення.
    
- **`remove(Object key, Object value)`** — видаляє запис лише тоді, коли і ключ, і значення **повністю збігаються** з тими, що ви вказали.

---

> [!warning] Обмеження
> 
> - **Потокобезпека**: Як і `HashMap`, цей клас не є синхронізованим. Якщо кілька потоків одночасно модифікують мапу, це призведе до помилок.
>     
> - **Об'єм пам'яті**: Через збереження додаткових посилань (`before`, `after`) у кожному вузлі, витрати пам'яті вищі приблизно на 2 посилання на кожен елемент.
>
