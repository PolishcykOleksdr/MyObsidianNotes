> **HashMap** — це одна з найпоширеніших структур даних у Java, яка реалізує інтерфейс `Map` на основі хеш-таблиці. Вона забезпечує високу швидкість доступу до даних, використовуючи пари «ключ-значення».

---

> [!abstract] Core Framework
> 
> `HashMap` базується на механізмі хешування і має такі характеристики:
> 
> - **Унікальність**: Ключі завжди унікальні. Якщо додати новий запис з існуючим ключем, старе значення буде замінено.
>     
> - **Null-сумісність**: Дозволяє один `null` ключ та безліч `null` значень.
>     
> - **Відсутність впорядкованості**: Не гарантує стабільного порядку елементів під час ітерації.
>     
> - **Несинхронізованість**: Об'єкт не є потокобезпечним (Thread-unsafe).
>     
> - **Продуктивність**: Основні операції (`put`, `get`, `remove`) виконуються за час $O(1)$ за умови гарного розподілу хеш-функції.
>     

---

> [!info] Key Insights: "Під капотом"
> 
> - **Структура Bucket-ів**: Дані зберігаються в сегментах (buckets). Спочатку кожен сегмент — це `LinkedList`.
>     
> - **Treeification**: У Java 8+, якщо кількість елементів в одному сегменті перевищує поріг `TREEIFY_THRESHOLD = 8`, зв'язний список перетворюється на **червоно-чорне дерево**. Це покращує швидкість пошуку з $O(n)$ до $O(\log n)$ у разі колізій.
>     
> - **Проблема змінних ключів**: Не можна використовувати змінні (mutable) об'єкти як ключі. Якщо стан об'єкта зміниться, його `hashCode` також може змінитись, і ви "втратите" значення, оскільки `HashMap` шукатиме його не в тому сегменті.
>     

---

> [!example] Практичне застосування: Базові операції
> 
> **Вставка та обробка дублікатів:**
> 
> 
> ```java
> Map<String, String> map = new HashMap<>();
> 
> map.put("abc1", "orange");
> map.put("dvc3", "apricot");
> map.put("dvc3", "plum"); // /* Старе значення "apricot" буде замінено на "plum" */
> map.put(null, "pepper"); // /* Дозволений null-ключ */
> ```
> 
> **Безпечне додавання (`putIfAbsent`):**
> 
> 
> ```java
> /* Додасть "cherry", тільки якщо ключа 322 немає або він null */
> map.putIfAbsent(322, "cherry"); 
> ```
> 
> **Отримання значень (`getOrDefault`):**
> 
> 
> ```java
> /* Поверне "mango", якщо ключа "mye4" не існує */
> String val = map.getOrDefault("mye4", "mango"); 
> ```

---

> [!example] Практичне застосування: Модифікація та Видалення
> 
> **Заміна значень:**
> 
> 
> ```java
> /* Замінить значення для "abc1" лише якщо зараз там "orange" */
> map.replace("abc1", "orange", "avocado"); 
> ```
> 
> **Видалення:**
> 
> 
> ```java
> /* Видалить запис за ключем */
> map.remove("dvc3"); 
> 
> /* Видалить лише якщо ключ "abc1" відповідає значенню "orange" */
> map.remove("abc1", "orange"); 
> ```

---

> [!example] Ітерація по HashMap
> 
> **1. Через entrySet (for-each):**
> 
> 
> ```java
> for (Map.Entry<String, String> entry : map.entrySet()) {
>     System.out.println(entry.getKey() + "=" + entry.getValue());
> }
> ```
> 
> **2. Через forEach (Lambda):**
> 
> 
> ```java
> map.forEach((key, value) -> System.out.println(key + "=" + value));
> ```

# Основні методи:
### **Додавання (Put)**

- **`put(K key, V value)`** — додає пару "ключ-значення". Якщо такий ключ уже був, він просто **записує нове значення поверх старого**.
    
- **`putIfAbsent(K key, V value)`** — додає пару лише тоді, **коли такого ключа ще немає** (або там лежить `null`). Якщо ключ уже є, метод нічого не змінює.
    
- **`putAll(Map m)`** — бере всі дані з іншої мапи та «засипає» їх у поточну. Якщо ключі збігаються, дані з нової мапи **замінять** ваші старі.
### **Отримання (Get)**

- **`get(Object key)`** — дістає значення за ключем. Якщо такого ключа не існує — видасть `null`.
    
- **`getOrDefault(Object key, V defaultValue)`** — дістає значення, але якщо ключа немає, замість `null` видасть **ваше «запасне» значення**, яке ви вкажете.
### **Очищення (Clear/Empty)**

- **`clear()`** — повністю видаляє всі записи. Мапа стає порожньою.
    
- **`isEmpty()`** — просто перевіряє: чи є в мапі хоч щось? (Результат: `true` — порожньо, `false` — є дані).
### **Пошук (Contains)**

- **`containsKey(Object key)`** — запитує: "Чи є у нас такий ключ?".
    
- **`containsValue(Object value)`** — запитує: "Чи є у нас десь (хоч під якимсь ключем) таке значення?".
### **Заміна (Replace)**

- **`replace(K key, V value)`** — оновлює значення для ключа, але **тільки якщо цей ключ уже існує**. Якщо ключа немає — нічого не станеться.
    
- **`replace(K key, V oldValue, V newValue)`** — замінить старе значення на нове тільки за умови, що **старе значення точно збігається** з тим, що ви вказали.
### **Видалення (Remove)**

- **`remove(Object key)`** — видаляє ключ і його значення.
    
- **`remove(Object key, Object value)`** — видаляє запис лише тоді, коли і ключ, і значення **повністю збігаються** з тими, що ви вказали.

---

> [!warning] Важливо знати
> 
> - **Initial Capacity & Load Factor**: `HashMap` має початкову ємність (за замовчуванням 16) і коефіцієнт завантаження (0.75). Коли таблиця заповнюється на 75%, відбувається **rehashing** (перебудування), що є витратною операцією.
>     
> - **Колізії**: Якщо різні ключі мають однаковий хеш-код, вони потрапляють в один сегмент. Саме тут спрацьовує логіка зі списками/деревами.
>
