> **Interface Map** — це фундаментальний інтерфейс Java Collections Framework, призначений для роботи зі структурами даних у форматі «ключ-значення». Важливою архітектурною особливістю є те, що він **не успадковується** від `Collection`, оскільки описує іншу модель даних (пари, а не поодинокі елементи).

---

> [!abstract] Core Framework
> 
> Interface Map визначає контракт для словників, де кожен ключ є унікальним і відображається на конкретне значення.
> 
> **Ієрархія інтерфейсів:**
> 
> - **Interface Map**: Базовий рівень для всіх словників.
>     
> - **Interface SortedMap**: Розширює Map, гарантуючи впорядкування ключів за їхнім природним порядком або за допомогою `Comparator`.
>     
> - **Interface NavigableMap**: Розширює SortedMap методами навігації (наприклад, пошук найближчого меншого або більшого ключа). Підтримує ітерацію в обох напрямках.
>     

---

> [!info] Key Insights
> 
> - **Відсутність спадковості від Collection**: Map оперує парами, тому методи на кшталт `add(E e)` замінені на `put(K key, V value)`.
>     
> - **Паралель із Set**: Реалізації Map за логікою роботи та продуктивністю ідентичні відповідним реалізаціям Set (`HashMap` $\approx$ `HashSet`), оскільки Set часто реалізується всередині через Map.
>     
> - **Навігаційні можливості**: `NavigableMap` дозволяє отримувати підмножини словника (`subMap`, `headMap`) та знаходити межі діапазонів ключів.
>

> [!example] Практичне застосування: Класи реалізації
> 
> **1. Class HashMap** > Використовується для досягнення максимальної швидкості операцій.
> 
> - Часова складність: $O(1)$ для основних операцій
> 
> - Порядок ітерації: не гарантується
> 
> **2. Class LinkedHashMap** > Поєднує швидкість `HashMap` із можливістю збереження порядку вставки.
> 
> - Корисно для створення LRU-кешів
> 
> - Порядок ітерації: за порядком додавання елементів
> 
> **3. Class TreeMap** > Реалізує `NavigableMap`. Використовується, коли дані мають бути завжди відсортовані за ключем.
> 
> - Часова складність: $O(log n)$ через структуру червоно-чорного дерева
> 
> - Порядок ітерації: відсортований за ключами
> 
> Приклад:
> ```java
> import java.util.*;
> 
> public class MapComparison {
> 
> public static void main(String[] args) {
> 
> 	// HashMap - швидкість, порядок неважливий
> 	Map<Integer, String> hashMap = new HashMap<>();
> 	
>     // LinkedHashMap - збереження черговості вставки
>     Map<Integer, String> linkedMap = new LinkedHashMap<>();
>     
>     // TreeMap - автоматичне сортування за ключем
>     SortedMap<Integer, String> treeMap = new TreeMap<>();
> }
> }
> ```

---

> [!tip] Вибір реалізації
> 
> |**Клас**|**Швидкість**|**Порядок ітерації**|
> |---|---|---|
> |**HashMap**|Найвища |Відсутній (хаотичний)|
> |**LinkedHashMap**|Висока|За порядком вставки 📥|
> |**TreeMap**|Середня |За сортуванням ключів 🔢|
