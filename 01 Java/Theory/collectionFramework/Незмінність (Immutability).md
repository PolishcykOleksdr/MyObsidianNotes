Це об'єкт, який після створення ніколи не змінює свій стан. Якщо вам потрібен такий самий, але з іншою властивістю — ви не переробляєте старий, а створюєте новий. Це гарантує, що дані не зміняться "за спиною" у програми.

---

> [!abstract] Core Framework
> **Імутабельність** — це архітектурний підхід, де стан об'єкта залишається константним протягом усього циклу життя.
> 
> **Візуальна метафора:** Це як **друкована книга**. Ви можете її читати, копіювати або видати нову редакцію, але ви не можете змінити текст безпосередньо на сторінках вже надрукованого примірника. На відміну від дошки з крейдою (mutable), книга — це зафіксована інформація.

---

>[!info] Key Insights
>	
>- **Безпека в багатопоточності.** Оскільки об'єкт не змінюється, кілька потоків можуть читати його одночасно без ризику виникнення помилок (Race Conditions).
>    
>- **Відсутність побічних ефектів.** Ви можете передавати об'єкт у будь-який метод, знаючи, що він повернеться в тому ж стані.
>    
>- **Спрощення логіки.** Зменшується кількість перевірок стану об'єкта під час виконання програми.
>    
>- **Ціна копіювання.** Будь-яка "зміна" — це створення нового об'єкта в пам'яті. Для величезних структур даних це може бути затратно.
>    

---

>[!example] Реалізація в Java
>
> Java
>
>```java
>// Створення незмінного представлення списку
>List<String> colors = new ArrayList<>(List.of("Red", "Green"));
>List<String> immutableColors = Collections.unmodifiableList(colors);
>
>// Спроба змінити структуру викине виняток:
>// immutableColors.add("Blue"); // UnsupportedOperationException
