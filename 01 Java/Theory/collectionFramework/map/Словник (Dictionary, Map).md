> **Словник** — це абстрактна структура даних, призначена для зберігання елементів у форматі пар **«ключ-значення»** (key-value). Головна ідея полягає у швидкому пошуку значення за його унікальним ідентифікатором — ключем.

---

> [!abstract] Core Framework
> 
> Словник є невпорядкованою колекцією, де кожен ключ однозначно вказує на відповідне значення. В Java ця концепція реалізована через інтерфейс `Map`.
> 
> **Базові операції:**
> 
> - **Вставка (Put)**: Додавання нової пари. Якщо ключ вже існує, старе значення замінюється новим.
>     
> - **Оновлення (Update)**: Зміна значення для вже існуючого ключа.
>     
> - **Видалення (Remove)**: Вилучення всієї пари за заданим ключем.
>     
> - **Пошук (Get/Contains)**: Перевірка наявності ключа або отримання значення за ним.
>     

---

> [!info] Key Insights
> 
> - **Унікальність ключа**: Ключі не можуть дублюватися. При спробі вставити дублікат, останнє значення перетирає попереднє.
>     
> - **Незмінність (Immutability)**: Ключ має бути незмінним (Immutable). У Java це зазвичай `String`, `Integer` або інші обгортки, оскільки зміна стану ключа після вставки призведе до неможливості його знаходження в пам'яті.
>     
> - **Чутливість до регістру**: Ключі `"KEY"` та `"key"` розглядаються як два абсолютно різні об'єкти.
>     
> - **Типи значень**: На відміну від ключа, значення може бути будь-яким: числом, рядком, об'єктом або навіть іншою колекцією (наприклад, `List` або інша `Map`).
>     

---

> [!example] Практичне застосування (Java Map)
> 
> В Java найпопулярнішою реалізацією словника є `HashMap`.
> 
> 
> ```java
> import java.util.HashMap;
> import java.util.Map;
> public class DictionaryExample {
> 
> public static void main(String[] args) {
> 
> // Створення словника
> 
> Map<String, String> dictionary = new HashMap<>();
>     // 1. Вставка даних
>     dictionary.put("Apple", "A fruit that grows on trees");
>     dictionary.put("Java", "A high-level programming language");
>     
>     // 2. Оновлення даних (заміна за ключем)
>     dictionary.put("Java", "A popular object-oriented language"); 
>     
>     // 3. Перевірка та отримання даних
>     if (dictionary.containsKey("Apple")) {
>         System.out.println("Meaning: " + dictionary.get("Apple"));
>     }
>     
>     // 4. Видалення даних
>     dictionary.remove("Apple");
>     
>     /* Результат: у словнику залишиться тільки Java */
> }
> 
> }
> ```

---

> [!warning] Обмеження та ризики
> 
> - **Втрата даних**: При вставці існуючого ключа старе значення видаляється без попередження.
>     
> - **Помилки видалення**: Спроба видалити ключ, якого не існує, може не викликати помилку в `Map.remove()`, але логічна перевірка перед видаленням є хорошою практикою.
>     
> - **Мутабельні об'єкти як ключі**: Використання масивів або списків як ключів є небезпечним, оскільки зміна їх вмісту змінить їхній `hashCode`, що зробить пару "невидимою" для пошуку.
>     

---

> [!tip] Складність операцій
> 
> Для більшості реалізацій (як `HashMap` в Java) часова складність операцій становить:
> 
> - **Пошук**: $O(1)$ в середньому випадку.
>     
> - **Вставка**: $O(1)$ в середньому випадку.
>     
> - **Видалення**: $O(1)$ в середньому випадку.
>     
> 
> Це робить словники однією з найефективніших структур для роботи з великими обсягами даних.
