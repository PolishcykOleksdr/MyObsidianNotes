> **Колекції (Collections)** – це **сховища** або **контейнери**, що підтримують різні способи накопичення та впорядкування об'єктів з метою забезпечення можливостей ефективного доступу до них.

#### **Колекції Java** - **класи**, основна мета яких - зберігати набір елементів.

> [!abstract] Core Framework
> **Колекції є реалізацією абстрактних структур даних, що підтримують основні операції:**
> - додавання нового елемента до колекції;
> - видалення елемента з колекції;
> - зміна елемента в колекції.

---

> [!tip] Для зберігання великої кількості однотипних даних можуть використовуватися **масиви**, але вони не завжди є ідеальним рішенням.
> - **Довжина масиву задається заздалегідь. Однак**, якщо кількість елементів заздалегідь невідома, доведеться або виділяти пам'ять «з запасом», або робити складні дії щодо виділення пам'яті для масиву.
> - **Елементи масиву мають жорстко задані місця розміщення**, тому, наприклад, видалення елемента з масиву перестає бути простою операцією.

### Щоб позбавитися цих недоліків використовуються рекурсивні типи даних, такі як списки і дерева.

---
### Стандартний набір колекцій Java служить для позбавлення необхідності самостійно реалізовувати ці типи даних і надає додаткові можливості.
