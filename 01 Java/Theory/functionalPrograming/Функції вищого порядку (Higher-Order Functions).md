Зазвичай функції працюють із числами або текстом. Функції вищого порядку йдуть далі: вони працюють з іншими функціями. Це дозволяє створювати універсальні інструменти, де ви даєте "двигун" (алгоритм), а функція-контейнер просто його запускає.

---

> [!abstract] Core Framework **Функція вищого порядку** — це функція, яка виконує хоча б одну з двох дій: приймає іншу функцію як вхідний параметр або повертає функцію як результат.
> 
> **Візуальна метафора:** Це як **кухонний комбайн**. Сам по собі комбайн (функція вищого порядку) просто крутить вал. Але те, що він зробить — наріже кубиками, натре чи зб'є — залежить від **насадки** (функції-аргументу), яку ви в нього вставите.

---

> [!info] Key Insights
> 
> - **Універсальність.** Замість створення десяти різних методів (фільтрація парних, фільтрація додатних тощо), ви створюєте один метод `filter`, який приймає умову.
>     
> - **Декларативність.** Ви кажете "що" зробити (відфільтрувати за умовою), а не "як" перебирати масив вручну.
>     
> - **Гнучкість.** Функції стають "цеглинками", які можна комбінувати між собою.
>     
> - **Інструментарій Java.** У Java 8+ цей принцип реалізовано через інтерфейси `Predicate`, `Function`, `Consumer` та `Stream API`.
>     

---

> [!example] Приклад: Функція як деталь конструктора
> 
> Java
> 
> ```java
> // Функція вищого порядку 'filter' приймає логіку 'predicate'
> public static List<Integer> filter(List<Integer> list, Predicate<Integer> predicate) {
>     List<Integer> result = new ArrayList<>();
>     for (Integer n : list) {
>         if (predicate.test(n)) result.add(n); // Виконуємо передану функцію
>     }
>     return result;
> }
> ```
> Виклик: передаємо "насадку" у вигляді лямбди List\<Integer> even = filter(numbers, n -> n % 2 == 0);
