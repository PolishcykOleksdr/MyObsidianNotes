> Лямбда-вираз — це спосіб передати "чисту дію" (функцію) так само легко, як ми передаємо число чи текст. Вам не потрібно будувати цілу "машину" (клас), щоб просто натиснути на кнопку; ви просто описуєте саму дію.

---

> [!abstract] Core Framework 
> **Лямбда-вираз** — це анонімна функція (функція без імені), яка реалізує єдиний метод функціонального інтерфейсу. Це лаконічна заміна анонімним класам.
> 
> **Візуальна метафора:** Це як **дистанційне керування**. Замість того, щоб підходити до кожного пристрою і перемикати важелі вручну (створювати класи), ви просто натискаєте кнопку на пульті (лямбда), яка посилає конкретну команду.

---

> [!info] Key Insights
> 
> - **Синтаксичний цукор.** Лямбда робить код чистішим, прибираючи "сміття" у вигляді назв методів та типів даних, які компілятор може вгадати сам.
>     
> - **Функціональний стиль.** Дозволяє думати категоріями "що зробити", а не "як організувати об'єкт".
>     
> - **Вимога до контексту.** Лямбду можна використати лише там, де очікується **функціональний інтерфейс** (той, що має лише один абстрактний метод).
>     
> - **Ефективність.** Використовує інструкцію `invokedynamic`, що часто працює швидше та споживає менше пам'яті, ніж анонімні класи.
>     

---

> [!example] Еволюція коду **До Java 8 (Анонімний клас):**
> 
> Java
> 
> ```java
> button.addActionListener(new ActionListener() {
>     @Override
>     public void actionPerformed(ActionEvent e) {
>         System.out.println("Клік!");
>     }
> });
> ```
> 
> **З Java 8 (Лямбда):**
> 
> Java
> 
> ```java
> button.addActionListener(e -> System.out.println("Клік!"));
> ```

---

> [!warning] Обмеження та ризики
> 
> - **Важкість відлагодження.** У стек-трейсі помилка в лямбді виглядає як анонімний виклик, що важче ідентифікувати.
>     
> - **Тільки effectively final.** Лямбда може використовувати змінні ззовні, але не може їх змінювати.
>     
> - **Читабельність.** Занадто довга лямбда (більше 3-5 рядків) перетворює код на "макарони". Краще винести таку логіку в окремий метод.
>
