# До Java 8 інтерфейси були "суворими контрактами": якщо ви додавали новий метод, усі класи, що його 
використовували, миттєво "ламалися", бо не мали його реалізації. **Default** методи дозволили додавати нові функції вже готовим інтерфейсам, а **static** методи — створювати допоміжні інструменти прямо в самому інтерфейсі.

---

> [!abstract] Core Framework
>  **Default методи** — це методи в інтерфейсах, які мають тіло (реалізацію) за замовчуванням. Їх не обов'язково переписувати в класі. 
>  **Static методи** — це методи, які належать самому інтерфейсу, а не об'єкту. Їх можна викликати, не створюючи екземпляр класу.
> 
> **Візуальна метафора:** Уявіть **стандартний договір оренди**. **Default** — це пункти, які вже прописані для всіх (наприклад, "не палити"), але за бажанням ви можете їх обговорити й змінити. **Static** — це номер телефону юридичної підтримки в кінці аркуша: він стосується самого договору як документа, а не ваших стосунків з квартирою.

---

> [!info] Key Insights
> 
> - **Еволюція без болю.** Default методи дозволяють розширювати інтерфейси (наприклад, `forEach` у `Collection`), не переписуючи тисячі існуючих класів.
>     
> - **Множинне наслідування.** Клас може реалізувати кілька інтерфейсів із default методами. Якщо імена методів збігаються, Java змусить вас вручну вказати, яку саме версію використовувати.
>     
> - **Утиліти в інтерфейсах.** Static методи ідеальні для створення фабрик або допоміжних перевірок (валідації), які стосуються лише цього інтерфейсу.
>     
> - **Виклик.** Static методи викликаються через `InterfaceName.methodName()`, а default — через об'єкт класу `object.methodName()`.
>     

---

> [!example] Приклад у коді
> 
> Java
> 
> ```java
> interface Vehicle {
>    // Звичайний метод (треба реалізувати)
>    void move();
>     
> // Default метод (вже готовий)
> default void horn() {
>     System.out.println("Beep beep!");
> }
> 
> // Static метод (допоміжний)
> static boolean isElectric(String type) {
>     return "Tesla".equals(type);
> }
> ```

---

> [!warning] Важливе правило (Diamond Problem) 
> Якщо два інтерфейси мають однаковий `default` метод, і один клас реалізує обидва — виникне конфлікт. Вам доведеться перевизначити метод у класі та вказати: `InterfaceName.super.methodName();`
