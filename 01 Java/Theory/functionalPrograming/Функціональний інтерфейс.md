
> Функціональний інтерфейс — це "контракт" у Java, який містить тільки одну дію (метод). Завдяки цьому можна писати код коротше, використовуючи лямбда-вирази (→). Це як шаблон з одним отвором — туди можна вставити тільки одну функцію, і це робить код простішим.

---

## Головна ідея

> [!abstract] Що таке функціональний інтерфейс?
> 
> **Правило "один метод"**: Інтерфейс з одним абстрактним методом. Може мати додаткові методи (default або static), але головний — тільки один.
> 
> **Навіщо потрібен**: Дозволяє передавати поведінку (функцію) як параметр. Замість створення цілого класу пишемо одну коротку функцію.
> 
> **Мітка якості**: Анотація `@FunctionalInterface` каже компілятору: "Перевір, чи справді тут один метод".

---

## Як використовувати

> [!info] Лямбда-вирази
> 
> Замість довгого коду пишемо короткий:
> ```java
> // Створюємо функцію "піднести до квадрату"
> Function<Integer, Integer> square = (x) -> x * x;
> int result = square.apply(5); // = 25
> ```
> 
> **Читається як**: "Візьми число x і поверни x помножене на x"

> [!info] Готові інтерфейси в Java
> 
> Не треба створювати свої — Java вже має набір у пакеті `java.util.function`:
> 
> - **Function** — перетворює одне на інше
> - **Predicate** — перевіряє умову (так/ні)
> - **Consumer** — робить щось із даними
> 
> ```java
> // Predicate = "фільтр"
> Predicate<String> довгеСлово = s -> s.length() > 5;
> boolean результат = довгеСлово.test("Hello"); // = false
> ```

## ✅ Переваги

> [!success] Чому це зручно
> 
> **Менше коду** → замість класу на 10 рядків пишемо 1 рядок  
> **Паралелізм** → легко обробляти дані одночасно (Stream API)  
> **Читабельність** → код виглядає як людська мова

---

## ⚠️ Обмеження

> [!warning] Про що памʼятати
> 
> 1. **Тільки один метод** — не можна розширювати (жорстке правило)
> 2. **Багато інтерфейсів** — для кожної задачі може знадобитися новий інтерфейс
> 3. **Ламкість** — якщо змінити інтерфейс, поламається весь код
> 4. **Прив'язка до типу** — інтерфейс працює з конкретним типом даних
> 5. **Складно новачкам** — треба розуміти функціональне програмування

## 🌐 Де використовується

> [!example] Практичне застосування
> 
> - **Stream API** — обробка списків/колекцій  
>   ```java
>   list.filter(x -> x > 10).map(x -> x * 2)
>   ```
> 
> - **Обробка подій** — кнопки у додатках  
>   ```java
>   button.onClick(() -> showMessage())
>   ```
