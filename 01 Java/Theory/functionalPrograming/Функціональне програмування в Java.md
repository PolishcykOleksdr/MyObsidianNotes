> Функціональне програмування дозволяє писати код, який працює як конвеєр: дані входять з одного боку, проходять через низку перетворень і виходять готовим результатом. Замість того, щоб постійно змінювати змінні (як у традиційному підході), ми створюємо ланцюжки функцій, які не псують вихідні дані. Це робить код передбачуваним, безпечним для багатопоточності та легким для тестування.

---

> [!abstract] Core Framework
> 
> **Функціональне програмування** — парадигма, де програма складається з функцій, які не змінюють дані, а створюють нові.
> 
> **Метафора:** Уявіть фабрику з конвеєрними стрічками. Кожна станція (функція) бере деталь, обробляє її і передає далі, не змінюючи попередні деталі. У фінальній точці ви отримуєте готовий продукт, а всі проміжні стани залишаються недоторканими.
> 
> У Java (традиційно об'єктно-орієнтованій мові) функціональний підхід з'явився завдяки лямбда-виразам і Stream API (з версії 8).

---

> [!info] Key Insights
> 
> **Три кити функціонального підходу:**
> 
> - **[[Незмінність (Immutability)]]** — дані не змінюються після створення. Кожна операція повертає новий об'єкт замість зміни існуючого.
> - **[[Функції вищого порядку (Higher-Order Functions)]]** — функції, які приймають інші функції як параметри або повертають їх як результат. Дозволяють будувати складну логіку з простих блоків.
> - **[[Замикання (closures)]]** — функції, які "запам'ятовують" змінні з зовнішнього оточення. Працюють як капсула з контекстом.
> 
> **Інструменти в Java:**
> 
> - **[[Лямбда-вирази]]** — короткий спосіб написання анонімних функцій. Замість 5 рядків коду пишемо один: `(x, y) -> x + y`
> - **[[Стріми (Stream API)]]** — конвеєр для обробки колекцій даних. Операції `map` (перетворення), `filter` (фільтрація), `reduce` (згортка) замінюють звичайні цикли.
> - **[[Паралельні стріми (Parrallel Stream API)]]** — автоматичний розподіл роботи між процесорними ядрами через `parallelStream()`. Потік даних обробляється одночасно кількома потоками.
> 
> **Переваги:**
> 
> - Код стає коротшим і читабельнішим (менше шаблонного коду)
> - Легше знаходити помилки (функції не мають побічних ефектів)
> - Автоматична підтримка паралелізму (без ручного створення потоків)

---

> [!example] Практичне застосування
> 
> **Послідовна обробка:**  
> ```java
> list.stream().filter(x -> x > 10).map(x -> x * 2).collect(Collectors.toList())
> ```
> 
> **Паралельна обробка:**  
> ```java
> list.parallelStream()
> ``` 
> — та сама логіка, але Java автоматично розподіляє роботу між ядрами процесора.
> 
> **Різниця:** Паралельний підхід прискорює обробку великих обсягів даних, але додає накладні витрати на синхронізацію. Використовуйте для колекцій з 1000+ елементів.
