Анонімні функції — це функції без імені, які створюються прямо там, де потрібні. Замість того, щоб оголошувати окремий метод, який використаєте лише один раз, ви пишете логіку "на місці". Це схоже на те, як ви можете сказати "той високий чоловік у синьому светрі" замість того, щоб давати йому ім'я, коли вказуєте на когось у натовпі.

---

> [!abstract] Core Framework
> 
> **Анонімна функція** — функція без власного імені, яка визначається та використовується безпосередньо в місці виклику.
> 
> **Метафора:** Уявіть, що ви наймаєте працівника для одноразового завдання (прибрати кімнату). Вам не потрібно знати його ім'я, посаду чи біографію — важливо лише те, що він зробить роботу. Після виконання завдання цей "працівник" зникає. Так само анонімна функція виконує свою роботу і більше не потрібна.
> 
> **У Java до версії 8:** анонімні функції реалізовувались через анонімні класи (багато шаблонного коду).  
> **У Java 8+:** з'явились лямбда-вирази — компактний синтаксис для анонімних функцій.

---

> [!info] Key Insights
> 
> **Основні характеристики:**
> 
> - **Не мають імені** — ви не можете викликати їх повторно за назвою
> - **Створюються "на льоту"** — визначаються прямо в коді, де потрібні
> - **Використовуються один раз** — зазвичай для короткочасних операцій
> - **Зберігають контекст** — можуть "бачити" змінні з зовнішнього оточення
> 
> **Коли використовувати:**
> 
> - Передача поведінки як параметра (сортування, фільтрація, обробка подій)
> - Одноразова логіка, яку не потрібно повторювати
> - Callback-функції (код, який виконається після певної події)
> 
> **Переваги:**
> 
> - **Менше коду** — не потрібно створювати окремий метод
> - **Читабельність** — логіка знаходиться там, де використовується
> - **Гнучкість** — швидка зміна поведінки без рефакторингу
> 
> **Недоліки:**
> 
> - Складну логіку важко читати в одному рядку
> - Неможливо повторно використати без копіювання коду

---

> [!example] Практичне застосування
> 
> **Старий підхід (анонімний клас):**
> 
> java
> 
> ```java
> button.addActionListener(new ActionListener() {
>     public void actionPerformed(ActionEvent e) {
>         System.out.println("Кнопку натиснуто");
>     }
> });
> ```
> 
> **Новий підхід (лямбда-вираз):**
> 
> java
> 
> ```java
> button.addActionListener(e -> System.out.println("Кнопку натиснуто"));
> ```
> 
> **Фільтрація списку:**
> 
> java
> 
> ```java
> // Залишити тільки парні числа
> numbers.stream()
>        .filter(n -> n % 2 == 0)  // анонімна функція перевіряє парність
>        .collect(Collectors.toList());
> ```
> 
> **Сортування:**
> 
> java
> 
> ```java
> // Відсортувати користувачів за віком
> users.sort((u1, u2) -> u1.getAge() - u2.getAge());
> ```

---

> [!tip] Правило простоти
> 
> Якщо анонімна функція займає більше 2-3 рядків або використовується у кількох місцях — краще створити звичайний іменований метод. Анонімні функції найкращі для простих, одноразових операцій.
