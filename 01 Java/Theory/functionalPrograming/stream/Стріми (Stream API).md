> Stream API — це конвеєр для обробки даних у Java. Уяви: замість того, щоб перебирати список вручну (цикл for), ти створюєш "трубу", через яку пропускаєш дані та на кожному етапі щось з ними робиш (фільтруєш, змінюєш, об'єднуєш). Для прикладу уяви, що маєш коробку з кульками і хочеш залишити тільки сині, великі, та впорядкувати їх за розміром. Це робить код коротшим і дозволяє Java автоматично розподіляти роботу між процесорами.

---

> [!abstract] Що таке Stream API?
> 
> **Що це**: Інструмент для обробки колекцій через ланцюжок операцій.
> 
> **Метафора**: Stream API — це як конвеєр на фабриці. Дані входять з одного боку, проходять через станції обробки (map, filter, reduce), і на виході ти отримуєш готовий результат.
> 
> **Принцип роботи**:
> ```
> Джерело даних → Операція 1 → Операція 2 → Результат
> ```
> 
> **Ключова особливість**: Операції можуть виконуватися послідовно (одна за одною) або паралельно (одночасно на кількох ядрах процесора).
>
>Три кити Stream API:
>
>- **Stream** — послідовність елементів (сам конвеєр)
>- **Проміжні операції** — `filter`, `map`, `sorted` — трансформують дані, але не запускають обробку
>- **Термінальні операції** — `forEach`, `collect`, `count` — запускають весь конвеєр і дають результат

---
> [!info] Key Insights
> **Лінивість** — конвеєр не працює, поки не викликана термінальна операція. Додаєш фільтри безкоштовно.
> 
> **Декларативність** — пишеш *що* зробити, не *як* ітерувати. Код коротший і читабельніший.
> 
> **Паралельність** — `.parallelStream()` замість `.stream()` автоматично розподіляє роботу між ядрами CPU.
> 
> Приклад: фрукти довші за 5 символів -> верхній регістр -> сортування -> вивід
> ```java
> fruits.stream()
>     .filter(fruit -> fruit.length() > 5)
>     .map(String::toUpperCase)
>     .sorted()
>     .forEach(System.out::println);
> ```

---

> [!info] Переваги та недоліки
> **Переваги**:
> - **Менше коду** — замість циклів пишеш ланцюжок операцій
> - **Читабельність** — код виглядає як опис дій, а не інструкції
> - **Паралелізм** — можна автоматично розподілити роботу на кілька ядер
> - **Незмінність** — оригінальні дані не змінюються
> 
> **Недоліки**:
> - **Одноразовість** — кожен Stream можна використати тільки один раз
> - **Крива навчання** — треба розуміти функціональне програмування
> - **Надлишковість** — для простих операцій звичайний цикл може бути зрозумілішим

> [!example] Практичний приклад
> 
> **Завдання**: З списку чисел отримати подвоєні парні числа
> 
> ```java
> List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
> 
> // Подвоїти кожне число
> List<Integer> doubled = numbers.stream()
>     .map(n -> n * 2)
>     .collect(Collectors.toList());
> // Результат: [2, 4, 6, 8, 10]
> 
> // Відібрати парні числа
> List<Integer> evens = numbers.stream()
>     .filter(n -> n % 2 == 0)
>     .collect(Collectors.toList());
> // Результат: [2, 4]
> ```
> 
> **Пояснення**:
> - `.stream()` — створює потік з колекції
> - `.map()` / `.filter()` — обробляють дані
> - `.collect()` — перетворює потік назад у список

---

> [!tip] Коли використовувати
> 
> **Так**:
> - Обробка великих колекцій
> - Складні трансформації даних
> - Потреба в паралельній обробці
> - Функціональний стиль коду
> 
> **Ні**:
> - Проста ітерація по списку
> - Модифікація елементів на місці
> - Потреба багаторазово використовувати потік
