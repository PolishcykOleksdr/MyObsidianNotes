> Посилання на метод — це ярлик у Java, який дозволяє передати готову функцію як параметр. Замість того, щоб писати `x -> doSomething(x)`, пишемо просто `Class::doSomething`. Це робить код коротшим. Працює як пульт дистанційного керування: натискаєш кнопку — викликається потрібна функція.

---

> [!abstract] Що таке посилання на методи?
> 
> **Що це**: Спосіб передати метод як аргумент, використовуючи символ `::` (подвійна двокрапка).
> 
> **Формула**: `Клас::метод` замість `(параметр) -> Клас.метод(параметр)`
> 
> **Метафора**: Це як замовлення готової страви в ресторані замість приготування з нуля. Ти просто вказуєш на пункт меню (метод), а не пишеш рецепт (лямбду).
> 
> **Синтаксис**:
> ```java
> // Замість лямбди
> words.forEach(word -> System.out.println(word));
> 
> // Посилання на метод
> words.forEach(System.out::println);
> ```

---

> [!info] Key Insights
> 
> **Три типи посилань**:
> 
> 1. **Статичний метод** → `Клас::статичнийМетод`
>    ```java
>    words.sort(String::compareToIgnoreCase);
>    ```
>    Читається: "Візьми метод compareToIgnoreCase з класу String"
> 
> 2. **Метод екземпляра** → `Клас::методЕкземпляра`
>    ```java
>    words.stream().map(String::length);
>    ```
>    Читається: "Для кожного слова викликай його метод length"
> 
> 3. **Метод конкретного об'єкта** → `об'єкт::метод`
>    ```java
>    words.forEach(System.out::println);
>    ```
>    Читається: "Передай кожен елемент у метод println об'єкта System.out"
> 
> ---
> 
> **Переваги**:
> - **Компактність** → `String::length` замість `s -> s.length()`
> - **Читабельність** → одразу видно, який метод використовується
> - **Сумісність** → працює з Stream API та функціональними інтерфейсами
> 
> **Обмеження**:
> - **Жорстка структура** → метод має точно відповідати параметрам
> - **Фіксована логіка** → не можна додати додаткові дії
> - **Крива навчання** → новачкам треба розуміти функціональне програмування
> - **Ламкість** → зміна методу ламає всі посилання на нього


---

> [!example] Практичні приклади
> 
> **Виведення списку**:
> ```java
> List<String> words = Arrays.asList("apple", "orange", "banana");
> words.forEach(System.out::println);
> ```
> 
> **Сортування без урахування регістру**:
> ```java
> words.sort(String::compareToIgnoreCase);
> ```
> 
> **Отримання довжини слів**:
> ```java
> List<Integer> lengths = words.stream()
>     .map(String::length)
>     .collect(Collectors.toList());
> ```

---

> [!tip]  Коли використовувати
> 
> **Так** ✅:
> - Лямбда просто викликає готовий метод
> - Потрібна читабельність коду
> - Робота з Stream API
> 
> **Ні** ❌:
> - Потрібна додаткова логіка всередині
> - Метод має складну сигнатуру
> - Посилання робить код менш зрозумілим

---

> [!warning] Typical Pitfalls
> 
> - **Плутанина типів**: розрізняй статичні методи та методи екземпляра
> - **Невідповідність параметрів**: метод має точно підходити до функціонального інтерфейсу
> - **Надмірна абстракція**: іноді звичайна лямбда зрозуміліша
