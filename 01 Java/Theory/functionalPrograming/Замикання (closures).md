>Замикання дозволяє функції захопити змінні з навколишнього середовища і "забрати їх з собою". Це як взяти з собою в дорогу термос із чаєм: чай був заварений на кухні (в контексті методу), але ви п'єте його в дорозі (в іншому місці коду), де кухні вже немає.

---

> [!abstract] Core Framework **Замикання** — це функція, яка посилається на вільні змінні у своєму лексичному контексті. Вона "замикає" в собі область видимості, де була створена.
> **Візуальна метафора:** Це як **рюкзак мандрівника**. Коли функція (мандрівник) покидає свою домівку (метод), вона бере в рюкзак потрібні їй речі (змінні). Тепер, куди б вона не пішла, ці речі завжди під рукою, навіть якщо домівка вже далеко.

---

> [!info] Key Insights
> 
> - **Захоплення контексту.** Функція бачить змінні, які не були передані їй як аргументи, але знаходилися поруч у момент її створення.
>     
> - **Ефект імутабельності.** У Java замикання можуть захоплювати лише змінні, які є `final` або **effectively final** (їх значення не змінюється після ініціалізації).
>     
> - **Життєвий цикл.** Навіть якщо метод, де була створена змінна, завершив роботу, замикання продовжує "тримати" цю змінну в пам'яті.
>     
> - **Інструменти реалізації.** Java реалізує цей механізм через **лямбда-вирази** та **анонімні класи**.
>     

---

> [!example] Приклад захоплення змінної
> 
> Java
> 
> ```java
> int limit = 100; // Змінна в оточенні
> // Лямбда "замикає" у собі змінну limit 
> Predicate<Integer> isOverLimit = (value) -> value > limit;
> // Навіть в іншому методі isOverLimit знатиме, що limit = 100 
> boolean result = isOverLimit.test(150); // true
> ```
