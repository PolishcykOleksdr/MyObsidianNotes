> **Одиночка (Singleton)** — це **породжувальний патерн**, який гарантує, що клас має лише **один екземпляр**, і надає **глобальну точку доступу** до цього екземпляра з будь-якого місця програми.

---

> [!abstract] Core Framework 
> **Singleton забезпечує єдиність екземпляра через три механізми:**
> 
> - **Приватний конструктор** — забороняє створення екземплярів ззовні класу;
> - **Статичне поле** — зберігає єдиний екземпляр усередині самого класу;
> - **Статичний метод `getInstance()`** — надає глобальну точку доступу і створює екземпляр при першому зверненні.

---

> [!info] Key Insights
> 
> - **Ледаче створення (Lazy Initialization).** Екземпляр створюється не при завантаженні класу, а лише при першому виклику `getInstance()`, що економить ресурси.
>     
> - **Глобальна точка доступу.** Будь-яка частина програми може отримати доступ до єдиного екземпляра через статичний метод, не передаючи посилання вручну.
>     
> - **Приватний конструктор.** Ключовий механізм захисту — жоден зовнішній код не може викликати `new Singleton()`.
>     

---

> [!example] Практичне застосування
> 
> **Структура патерну:**
> 
> - `instance` — приватне статичне поле, що зберігає єдиний екземпляр;
> - `Singleton()` — приватний конструктор, недоступний ззовні;
> - `getInstance()` — статичний метод, що повертає існуючий або створює новий екземпляр.

```java
// Клас Одиночка (Singleton)
public class Singleton {
    // Приватне статичне поле для зберігання єдиного екземпляра класу
    private static Singleton instance;

    // Приватний конструктор, щоб заборонити створення екземплярів класу ззовні
    private Singleton() {
    }

    // Метод для отримання єдиного екземпляра класу
    public static Singleton getInstance() {
        // Якщо екземпляр ще не був створений, створюємо його
        if (instance == null) {
            instance = new Singleton();
        }
        // Повертаємо єдиний екземпляр класу
        return instance;
    }

    // Додаткові методи та поля класу
    public void showMessage() {
        System.out.println("Hello, I am a Singleton!");
    }
}

public class Main {
    public static void main(String[] args) {
        // Отримання єдиного екземпляра класу Singleton
        Singleton singleton = Singleton.getInstance();
        // Виклик методу showMessage для показу роботи Одиночка
        singleton.showMessage();
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Проблеми з багатопотоковістю.** Наведена реалізація не є потокобезпечною — два потоки можуть одночасно пройти перевірку `instance == null` і створити два екземпляри. Для вирішення використовують `synchronized` або подвійну перевірку блокування.
>     
> - **Прихована залежність.** Код, що використовує Singleton, приховано залежить від нього, що ускладнює тестування та підміну реалізації.
>     
> - **Порушення принципу єдиної відповідальності.** Клас сам керує своїм життєвим циклом на додаток до основної бізнес-логіки.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Потрібен єдиний об'єкт для координації дій у системі (логгер, конфігурація, пул з'єднань)
> - Необхідна глобальна точка доступу до спільного ресурсу
> - Створення об'єкта є дорогим і має відбуватись лише один раз
> 
> **Ні** ❌:
> 
> - Потрібно кілька незалежних екземплярів з різним станом
> - Код має бути легко тестованим (Singleton ускладнює моки)
