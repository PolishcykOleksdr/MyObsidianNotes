> **Прототип (Prototype)** — це **породжувальний патерн**, який дозволяє створювати нові об'єкти шляхом **копіювання вже існуючих екземплярів** (клонування), уникаючи повторного проходження складного процесу ініціалізації.

---

> [!abstract] Core Framework 
> **Prototype делегує логіку копіювання самому об'єкту:**
> 
> - **Абстрактний Prototype** — оголошує метод `clone()`, обов'язковий для всіх прототипів;
> - **ConcretePrototype** — реалізує `clone()`, знаючи власну структуру і копіюючи всі свої поля;
> - **Клієнт** — створює новий об'єкт викликом `clone()` на існуючому екземплярі, не знаючи його конкретного класу.

---

> [!info] Key Insights
> 
> - **Клонування замість конструктора.** Новий об'єкт отримується не через `new`, а через копіювання існуючого — це корисно, коли створення «з нуля» є складним або дорогим.
>     
> - **Інкапсуляція логіки копіювання.** Кожен клас сам знає, як себе копіювати, — клієнту не потрібно знати внутрішню структуру об'єкта.
>     
> - **Незалежність клонів.** Після клонування обидва об'єкти — оригінал і копія — існують незалежно і зміни в одному не впливають на інший.
>     

---

> [!example] Практичне застосування
> 
> **Структура патерну:**
> 
> - `Prototype` — абстрактний клас із полем `name` і абстрактним методом `clone()`;
> - `ConcretePrototype` — розширює `Prototype`, додає поле `value`, реалізує `clone()` через конструктор копіювання;
> - `Main` — демонструє створення оригіналу і його незалежного клону.

```java
// Абстрактний клас Прототипу
abstract class Prototype {
    protected String name;

    // Конструктор класу
    public Prototype(String name) {
        this.name = name;
    }

    // Метод клонування
    public abstract Prototype clone();
}

// Конкретний клас Прототипу
class ConcretePrototype extends Prototype {
    private int value;

    // Конструктор класу
    public ConcretePrototype(String name, int value) {
        super(name);
        this.value = value;
    }

    // Реалізація методу клонування
    public Prototype clone() {
        return new ConcretePrototype(this.name, this.value);
    }

    // Метод для виведення інформації про об'єкт у консоль
    public void printInfo() {
        System.out.println("Ім'я: " + this.name + ", Значення: " + this.value);
    }
}

public class Main {
    public static void main(String[] args) {
        // Створюємо початковий об'єкт
        ConcretePrototype original = new ConcretePrototype("Прототип", 100);
        original.printInfo(); // Виводимо інформацію про початковий об'єкт

        // Клонуємо об'єкт
        ConcretePrototype clone = (ConcretePrototype) original.clone();
        clone.printInfo(); // Виводимо інформацію про клонований об'єкт
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Поверхневе копіювання (Shallow Copy).** Якщо об'єкт містить поля-посилання (масиви, колекції, інші об'єкти), просте копіювання скопіює лише посилання, а не самі об'єкти. Потрібно реалізовувати глибоке копіювання (Deep Copy) вручну.
>     
> - **Складність клонування з циклічними посиланнями.** Якщо об'єкти посилаються один на одного, реалізація `clone()` значно ускладнюється.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Процес створення нового об'єкта є складним або дорогим (наприклад, читання з БД або мережі)
> - Потрібно створювати багато схожих об'єктів із незначними відмінностями
> - Конкретний клас об'єкта, що створюється, має бути невідомим клієнту
> 
> **Ні** ❌:
> 
> - Об'єкт містить складні посилання і потребує глибокого копіювання — накладні витрати можуть переважити користь
> - Простіше і зрозуміліше скористатися звичайним конструктором
