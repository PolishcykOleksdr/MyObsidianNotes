> **Міст (Bridge)** — це **структурний патерн**, який **розділяє абстракцію від її реалізації** так, щоб обидві частини могли змінюватися та розширюватися незалежно одна від одної.

---

> [!abstract] Core Framework 
> **Bridge розриває жорсткий зв'язок між абстракцією та реалізацією через композицію:**
> 
> - **Abstraction** — абстрактний клас, що містить посилання на об'єкт реалізації (`Shape` → `Color`);
> - **RefinedAbstraction** — конкретна абстракція, що розширює базову (`Rectangle`, `Circle`);
> - **Implementor** — інтерфейс реалізації (`Color`);
> - **ConcreteImplementor** — конкретна реалізація інтерфейсу (`RedColor`, `GreenColor`).

---

> [!info] Key Insights
> 
> - **Композиція замість успадкування.** Абстракція не успадковує реалізацію, а тримає на неї **посилання** — це дозволяє підмінити реалізацію в рантаймі.
>     
> - **Незалежне масштабування.** Нові фігури додаються без зміни кольорів, нові кольори — без зміни фігур. Кількість класів зростає лінійно, а не як декартовий добуток.
>     
> - **Уникнення комбінаторного вибуху класів.** Без Bridge для $n$ фігур і $m$ кольорів потрібно $n \times m$ класів. З Bridge — лише $n + m$.
>     

---

> [!example] Практичне застосування
> 
> **Структура патерну:**
> 
> - `Shape` — абстракція з полем `color` типу `Color` і абстрактним методом `draw()`;
> - `Color` — інтерфейс реалізації з методом `applyColor()`;
> - `Rectangle`, `Circle` — конкретні абстракції, що реалізують `draw()` і делегують колір;
> - `RedColor`, `GreenColor` — конкретні реалізації кольору.

```java
// Абстракція
abstract class Shape {
    protected Color color;

    // Конструктор абстракції, який приймає об'єкт реалізації
    public Shape(Color color) {
        this.color = color;
    }

    // Абстрактний метод для відображення фігури
    abstract void draw();
}

// Реалізація абстракції - червоний колір
class RedColor implements Color {
    public void applyColor() {
        System.out.println("Застосування червоного кольору");
    }
}

// Реалізація абстракції - зелений колір
class GreenColor implements Color {
    public void applyColor() {
        System.out.println("Застосування зеленого кольору");
    }
}

// Інтерфейс для реалізації кольору
interface Color {
    void applyColor();
}

// Прямокутник
class Rectangle extends Shape {
    // Конструктор прямокутника, який приймає об'єкт реалізації кольору
    public Rectangle(Color color) {
        super(color);
    }

    // Перевизначення методу для відображення прямокутника
    @Override
    void draw() {
        System.out.print("Відображення прямокутника: ");
        color.applyColor(); // Виклик методу для застосування кольору
    }
}

// Круг
class Circle extends Shape {
    // Конструктор круга, який приймає об'єкт реалізації кольору
    public Circle(Color color) {
        super(color);
    }

    // Перевизначення методу для відображення круга
    @Override
    void draw() {
        System.out.print("Відображення круга: ");
        color.applyColor(); // Виклик методу для застосування кольору
    }
}

public class Main {
    public static void main(String[] args) {
        // Створення червоного прямокутника
        Shape redRectangle = new Rectangle(new RedColor());
        redRectangle.draw(); // Виклик методу для відображення прямокутника

        // Створення зеленого круга
        Shape greenCircle = new Circle(new GreenColor());
        greenCircle.draw(); // Виклик методу для відображення круга
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Складність для простих випадків.** Якщо абстракція має лише одну реалізацію і розширення не планується — Bridge додає зайву структуру.
>     
> - **Непряме звернення.** Виклик методу абстракції завжди делегується через поле-посилання, що ускладнює розуміння потоку виконання.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Є дві осі варіативності (наприклад, форма і колір), які мають розширюватися незалежно
> - Потрібно уникнути комбінаторного вибуху підкласів при поєднанні кількох вимірів
> - Реалізацію потрібно підмінювати динамічно в рантаймі
> 
> **Ні** ❌:
> 
> - Абстракція має єдину стабільну реалізацію
> - Система не передбачає розширення по кількох осях
