> **Адаптер (Adapter)** — це **структурний патерн**, який дозволяє об'єктам із **несумісними інтерфейсами** працювати разом, виступаючи посередником-перекладачем між ними без зміни їхнього вихідного коду.

---

> [!abstract] Core Framework 
> **Adapter забезпечує сумісність через обгортку:**
> 
> - **Target (цільовий інтерфейс)** — інтерфейс, який очікує клієнт (`OldInterface`);
> - **Adaptee (адаптований клас)** — існуючий клас із несумісним інтерфейсом (`NewClass`);
> - **Adapter** — клас-обгортка, що реалізує цільовий інтерфейс і делегує виклики до Adaptee.

---

> [!info] Key Insights
> 
> - **Без змін вихідного коду.** Ні старий, ні новий клас не змінюються — Adapter є єдиною точкою адаптації між ними.
>     
> - **Делегування замість успадкування.** Adapter тримає посилання на об'єкт нового класу і делегує йому виклики, перетворюючи сигнатуру методу.
>     
> - **Прозорість для клієнта.** Клієнт бачить лише звичний `OldInterface` і не знає, що насправді викликається метод іншого класу.
>     

---

> [!example] Практичне застосування
> 
> **Структура патерну:**
> 
> - `NewInterface` / `NewClass` — новий клас із методом `request()`;
> - `OldInterface` — старий інтерфейс із методом `specificRequest()`;
> - `Adapter` — реалізує `OldInterface`, всередині викликає `request()` об'єкта `NewClass`.

```java
// Інтерфейс нового класу
interface NewInterface {
    void request();
}

// Клас, що реалізує новий інтерфейс
class NewClass implements NewInterface {
    public void request() {
        System.out.println("Виклик методу request() з нового класу");
    }
}

// Інтерфейс старого класу
interface OldInterface {
    void specificRequest();
}

// Клас старого класу
class OldClass {
    public void specificRequest() {
        System.out.println("Виклик методу specificRequest() зі старого класу");
    }
}

// Адаптер для забезпечення сумісності між старим та новим інтерфейсами
class Adapter implements OldInterface {
    private NewInterface newObject;

    // Конструктор адаптера, що приймає об'єкт нового класу
    public Adapter(NewInterface newObject) {
        this.newObject = newObject;
    }

    // Реалізація методу старого інтерфейсу, який викликає метод нового класу
    public void specificRequest() {
        newObject.request();
    }
}

public class Main {
    public static void main(String[] args) {
        // Створення об'єкта нового класу
        NewClass newClass = new NewClass();

        // Створення адаптера, що забезпечує сумісність інтерфейсів
        Adapter adapter = new Adapter(newClass);

        // Виклик методу старого класу через адаптер
        adapter.specificRequest();
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Додатковий рівень непрямості.** Кожен виклик проходить через Adapter, що ускладнює читання стек-трейсу і може заплутати при відлагодженні.
>     
> - **Розростання адаптерів.** Якщо несумісних інтерфейсів багато, кількість класів-адаптерів може суттєво зрости.
>     

---

> [!tip] Коли використовувати
> 
> **Так** ✅:
> 
> - Потрібно інтегрувати сторонню бібліотеку або legacy-код із несумісним інтерфейсом
> - Не можна або не бажано змінювати вихідний код існуючих класів
> - Потрібно повторно використати існуючий клас у новому контексті
> 
> **Ні** ❌:
> 
> - Інтерфейси можна узгодити безпосередньо (рефакторингом)
> - Різниця між інтерфейсами настільки велика, що адаптер перетворюється на повноцінну бізнес-логіку
