> Будівельні блоки UML — це ключові елементи, з яких складаються діаграми. Вони дозволяють розробникам створювати моделі, що точно відображають структуру та поведінку системи, забезпечуючи ефективну комунікацію всередині проекту.

---

> [!abstract] Core Framework Основні компоненти моделювання включають:
> 
> - **Клас та Об'єкт:** Клас визначає структуру (атрибути) та поведінку (методи), а об'єкт є його конкретним екземпляром у пам'яті.
>     
> - **Інтерфейс:** Контракт, що визначає набір методів без їх реалізації.
>     
> - **Пакет:** Засіб групування елементів для керування складністю моделі.
>     
> - **Відношення (Стрілки):** Графічне представлення зв'язків та взаємодії між елементами.
>     
> - **Абстрактний клас:** Містить як реалізовані, так і нереалізовані методи, виступаючи базою для інших класів.
>     

---

> [!info] Key Insights: Семантика зв'язків В UML тип стрілки визначає характер зв'язку між об'єктами:
> 
> - **Асоціація:** Базовий зв'язок, що вказує на можливість доступу одного об'єкта до іншого.
>     
> - **Агрегація:** Відношення "ціле-частина", де частина може існувати незалежно від цілого.
>     
> - **Композиція:** Суворий тип агрегації; частина не може існувати без цілого (спільний життєвий цикл).
>     
> - **Залежність:** Тимчасовий зв'язок, де зміна в одному елементі може вплинути на інший (наприклад, використання об'єкта як параметра методу).
>

> [!example] Практичне застосування: Типи відношень у Java
> 
> ### 1. Відношення асоціації
> 
> Зв'язок між `Customer` та `Order`, де кожен об'єкт зберігає посилання на інший.
> 
> ```java
> /* Клас, який представляє замовлення */
> public class Order {
>     private Customer customer; /* Поле, що зберігає замовника */
> 
>     /* Конструктор, що приймає замовника */
>     public Order(Customer customer) {
>         this.customer = customer;
>     }
> }
> 
> /* Клас, який представляє клієнта */
> public class Customer {
>     private List<Order> orders; /* Поле, що зберігає список замовлень */
> 
>     /* Конструктор, що ініціалізує список замовлень */
>     public Customer() {
>         this.orders = new ArrayList<>();
>     }
> 
>     /* Метод для додавання замовлення до списку */
>     public void addOrder(Order order) {
>         orders.add(order);
>     }
> }
> ```
> 
> ### 2. Відношення агрегації
> 
> `Order` містить список `Product`, але продукти існують незалежно від замовлення.
> 
> ```java
> import java.util.List;
> 
> /* Клас, який представляє продукт */
> public class Product {
>     private String name;
> 
>     /* Конструктор класу Product */
>     public Product(String name) {
>         this.name = name;
>     }
> 
>     /* Метод для отримання назви продукту */
>     public String getName() {
>         return name;
>     }
> }
> 
> /* Клас, який представляє замовлення */
> public class Order {
>     private List<Product> products; /* Список продуктів в замовленні */
> 
>     /* Конструктор класу Order (агрегація: продукти передаються ззовні) */
>     public Order(List<Product> products) {
>         this.products = products;
>     }
> 
>     /* Метод для отримання списку продуктів в замовленні */
>     public List<Product> getProducts() {
>         return products;
>     }
> }
> ```
> 
> ### 3. Відношення композиції
> 
> Об'єкти `Engine` та `Wheel` створюються всередині `Car` і не можуть існувати без нього.
> 
> ```java
> /* Клас, що представляє двигун */
> public class Engine {
>     /* Реалізація двигуна */
> }
> 
> /* Клас, що представляє колесо */
> public class Wheel {
>     /* Реалізація колеса */
> }
> 
> /* Клас, що представляє автомобіль */
> public class Car {
>     private Engine engine; /* Об'єкт класу Engine */
>     private List<Wheel> wheels; /* Список об'єктів класу Wheel */
> 
>     /* Конструктор класу Car (композиція: компоненти створюються разом із цілим) */
>     public Car() {
>         this.engine = new Engine(); /* Створення двигуна при створенні авто */
>         this.wheels = new ArrayList<>();
>         for (int i = 0; i < 4; i++) {
>             wheels.add(new Wheel()); /* Створення коліс при створенні авто */
>         }
>     }
> 
>     public Engine getEngine() { return engine; }
>     public List<Wheel> getWheels() { return wheels; }
> }
> ```

> [!warning] Важливе про стрілки
> 
> - **Напрямок:** Стрілка вказує на напрямок доступу. В асоціації це може бути навігація від замовлення до клієнта.
>     
> - **Життєвий цикл:** У композиції (зафарбований ромб) видалення об'єкта `Car` автоматично призводить до видалення `Engine` з пам'яті (garbage collection), оскільки посилання ініціалізовано всередині.
>     
> - **Слабкі зв'язки:** Залежність (пунктирна стрілка) — це найслабший зв'язок, що зазвичай не передбачає зберігання посилання як поля класу.
>     

---

> [!tip] Візуалізація блоків
> 
> - **Клас:** Прямокутник з трьома секціями (Назва, Атрибути, Методи).
>     
> - **Інтерфейс:** Позначається ключовим словом `<<interface>>`.
>     
> - **Абстрактний клас:** Назва зазвичай пишеться курсивом або додається тег `<<abstract>>`.
>
