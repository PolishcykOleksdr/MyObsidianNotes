> State Machine діаграма (або Statechart) — це динамічна діаграма UML, яка описує життєвий цикл одного об'єкта, фокусуючись на зміні його станів у відповідь на зовнішні або внутрішні події. Вона ідеально підходить для моделювання систем зі складною логікою переходів.

---

> [!abstract] Core Framework 
> Поведінкова модель об'єкта будується на таких елементах:
> 
> - **State (Стан):** Умова або ситуація в житті об'єкта, протягом якої він задовольняє певну умову або чекає на подію. Позначається прямокутником із закругленими кутами.
>     
> - **Transition (Перехід):** Відношення між двома станами, що вказує на зміну стану при настанні певної події.
>     
> - **Initial & Final State:** Точка входу (чорне коло) та точка завершення життєвого циклу (коло з крапкою).
>     
> - **Event (Подія):** Тригер, що викликає перехід (виклик методу, таймер, зміна сигналу).
>     
> - **Guard (Умова):** Булевий вираз `[condition]`, який має бути істинним, щоб перехід відбувся.
>     

---

> [!info] Key Insights
> 
> - **Фокус на об'єкті.** На відміну від Activity-діаграми, яка описує процес, State-діаграма описує зміни конкретного екземпляра класу.
>     
> - **Deterministic Behavior.** Діаграма допомагає гарантувати, що об'єкт не потрапить у непередбачений стан (наприклад, замовлення не може бути "Оплачене" до того, як воно було "Створене").
>     
> - **Внутрішні дії.** Стан може містити дії: `entry` (при вході), `exit` (при виході), `do` (під час перебування).
>     
> - **Складні стани.** Можливість створення ієрархічних станів (стан всередині стану).
>

> [!example] Практичне застосування Моделювання статусу виборів для вашого проекту **"school-elections"**.

```java
/*
 * Реалізація логіки станів через Enum та Pattern State
 * Об'єкт: Election (Вибори)
 */

public enum ElectionState {
    CREATED,    /* Створено */
    ONGOING,    /* Триває голосування */
    FINISHED    /* Завершено */
}

public class Election {
    private ElectionState currentState = ElectionState.CREATED;

    public void startElection() {
        /* Перехід: CREATED -> ONGOING 
         * Подія: Виклик методу start
         */
        if (currentState == ElectionState.CREATED) {
            currentState = ElectionState.ONGOING;
            System.out.println("Voting has started.");
        }
    }

    public void castVote(String studentId) {
        /* Умова (Guard): Голосувати можна лише у стані ONGOING */
        if (currentState == ElectionState.ONGOING) {
            // логіка збереження голосу в PostgreSQL
            System.out.println("Vote registered for: " + studentId);
        } else {
            throw new IllegalStateException("Voting is not active!");
        }
    }
}
```

> [!warning] Обмеження та ризики
> 
> - **Комбінаторний вибух.** Якщо у об'єкта забагато станів і переходів, діаграма стає нечитабельною ("спагетті-діаграма").
>     
> - **Неправильне використання.** Не намагайтеся описати станами всю систему; використовуйте їх лише для об'єктів із чіткою подієво-залежною логікою.
>     
> - **Синхронізація.** Важливо стежити, щоб логіка в коді (наприклад, статуси в БД) точно відповідала станам на діаграмі.
>     

---

> [!tip] Коли використовувати 
> **Так ✅:**
> 
> - Реалізація складних бізнес-процесів (статуси замовлень, заявок).
>     
> - Проектування систем реального часу або вбудованого ПЗ.
>     
> - Опис протоколів мережевої взаємодії (TCP Handshake).
>     
> 
> **Ні ❌:**
> 
> - Для об'єктів, які просто зберігають дані (DTO) і не мають поведінки.
>     
> - Для опису лінійних алгоритмів без розгалужень за подіями.
>
