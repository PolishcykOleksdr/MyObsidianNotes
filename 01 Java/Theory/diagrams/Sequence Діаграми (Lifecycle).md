> **Sequence Diagram (Діаграма послідовності)** — це тип діаграми взаємодії, що зосереджена на часовій упорядкованості повідомлень. Вона ілюструє, як об’єкти в системі взаємодіють один з одним у межах конкретного сценарію виконання.

---

> [!abstract] Core Framework
> 
> Основна структура діаграми базується на вертикальних та горизонтальних осях:
> 
> - **Вертикальна вісь (Час):** Відображає плин часу зверху донизу.
>     
> - **Горизонтальна вісь (Учасники):** Перелічує об’єкти або акторів, залучених до взаємодії.
>     
> - **Лінія життя (Lifeline):** Пунктирна вертикальна лінія, що показує період існування об’єкта.
>     
> - **Смуга активації (Activation Bar):** Тонкий прямокутник на лінії життя, що вказує на виконання об’єктом певної операції.
>

> [!info] Key Insights: Типи учасників та повідомлень
> 
> - **Учасники:** Окрім звичайних **Об’єктів** та **Акторів**, виділяють **Boundary** (інтерфейси/екрани), **Control** (менеджери/контролери) та **Entity** (системні дані).
>     
> - **Синхронні повідомлення:** Відправник очікує відповіді (зафарбована стрілка $\rightarrow$).
>     
> - **Асинхронні повідомлення:** Відправник продовжує роботу, не чекаючи відповіді (відкрита стрілка $\rightarrow$).
>     
> - **Повернення (Return):** Передача результату назад викликаючій стороні (пунктирна стрілка $\dashleftarrow$).
>     

---

> [!example] Практичне застосування: Фрагменти керування
> 
> Для моделювання складної логіки (цикли, умови) використовуються комбіновані фрагменти:
> 
> |**Оператор**|**Опис**|**Приклад використання**|
> |---|---|---|
> |**alt**|Альтернатива (if-else)|Вибір методу оплати (карта або готівка).|
> |**opt**|Опція (if)|Додавання подарункового пакування до замовлення.|
> |**loop**|Цикл (for/while)|Обробка кожного товару в кошику.|
> |**ref**|Посилання|Виклик іншої діаграми (наприклад, "Процес авторизації").|

---

> [!quote] Фрагмент `par` на діаграмі послідовності
> 
> Абревіатура від **parallel**. Цей блок використовується для опису дій, які відбуваються **одночасно**. Порядок повідомлень всередині різних секцій цього блоку не має значення — вони виконуються паралельно.

> [!example] Мапінг на Java-код
> 
> Послідовність повідомлень на діаграмі напряму трансформується у виклики методів:

```java
/* Актор ініціює дію через Boundary (UI) */
public class OrderController { /* Control */
    private PaymentService paymentService;
    private InventoryService inventory; /* Entity/Service */

    public void checkout(Order order) {
        /* loop: перевірка кожного товару */
        for (Item item : order.getItems()) {
            inventory.checkStock(item); /* Стрілка від Control до Entity */
        }

        /* alt: вибір сценарію оплати */
        if (order.getPaymentType() == Card) {
            paymentService.processCard(order.getAmount()); /* Синхронний виклик */
        }
        
        /* return: повернення статусу успіху */
    }
}
```

---

> [!warning] Обмеження та ризики
> 
> - **Захаращеність:** Не намагайтеся відобразити кожну дрібну операцію чи ініціалізацію змінної. Діаграма має бути читабельною.
>     
> - **Логічна стрілка:** Початок стрілки **завжди** має торкатися лінії життя відправника, а кінець — лінії життя одержувача.
>     
> - **Спрямованість:** Основний потік повідомлень будується зліва направо, а повернення — справа наліво.
>

>
