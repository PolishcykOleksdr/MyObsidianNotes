> **Діаграма класів UML** — це статична структурна діаграма, що описує структуру системи шляхом візуалізації її класів, їхніх атрибутів, методів та взаємозв'язків між об'єктами. Вона є фундаментом об'єктно-орієнтованого моделювання і використовується для документування архітектури програмного забезпечення.

---

> [!abstract] Core Framework 
> Стандартне графічне представлення класу складається з прямокутника, поділеного на три секції:
> 
> 1. **Верхня секція:** Назва класу (обов'язкова).
>     
> 2. **Середня секція:** Атрибути класу (характеристики/поля).
>     
> 3. **Нижня секція:** Операції або методи (поведінка/функції).
>

> [!info] Key Insights: 
> Модифікатори доступу та Елементи Для визначення видимості членів класу в UML використовуються спеціальні символи:
> 
> - `+` **Public:** Доступний для всіх.
>     
> - `-` **Private:** Доступний лише всередині класу.
>     
> - `#` **Protected:** Доступний класу та його нащадкам.
>     
> - `~` **Package:** Доступний лише всередині пакета.
>     
> - `_underline_` **Static:** Член класу (статичний).
>     
> - `/` **Derived:** Похідний атрибут, що обчислюється на основі інших.
>     
> 
> **Додаткові будівельні блоки:**
> 
> - **Інтерфейси:** Набір сигнатур операцій (позначаються як `<<interface>>`).
>     
> - **Перерахування (Enumeration):** Типи даних, визначені користувачем (`<<enumeration>>`).
>     
> - **Об'єкти:** Конкретні екземпляри класів.
>

> [!quote] Association Class (Асоціативний клас) 
> 
> Це особливий елемент, який поєднує в собі властивості **зв'язку** (асоціації) та **класу**. Його використовують, коли сам факт зв'язку між двома об'єктами має власні характеристики.
> 
> - **Приклад:** Клас `Студент` і клас `Курс`. Зв'язок між ними — «Зарахування». Але де зберігати оцінку студента за цей курс? Вона не належить суто студенту чи суто курсу. Вона з'являється лише в момент їхньої взаємодії. Тут ми створюємо `Association Class`, який містить атрибут `оцінка`.

---

> [!quote] Множинність `0..1` 
> 
> Це вказівник на кількість об'єктів, що беруть участь у зв'язку. `0..1` означає «нуль або один».
> 
> - Це типово для полів, які можуть бути порожніми (`null` у програмуванні). Наприклад, у `Людини` може бути `0..1` об'єкт `Автомобіль`.

> [!example] Приклад структури Java-класу в UML

```java
/* Приклад класу з атрибутами та методами */
public class Student {
    private String name;      /* - name: String */
    public int age;           /* + age: int */
    protected double gpa;     /* # gpa: double */

    public void study() {     /* + study(): void */
        /* Логіка навчання */
    }

    private void calculateGrade() { /* - calculateGrade(): void */
        /* Внутрішні розрахунки */
    }
}
```

> [!warning] Важливе про використання
> 
> - **Статичність:** Діаграма класів не показує, _як_ об'єкти взаємодіють у часі (для цього використовуйте Sequence Diagram), вона описує лише _будову_ системи.
>     
> - **Рівень деталізації:** Не обов'язково вказувати всі гетери/сетери або приватні допоміжні методи, якщо вони не несуть архітектурної цінності. Захаращення діаграми ускладнює її читання.
>
