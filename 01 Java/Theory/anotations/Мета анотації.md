> Мета-анотації — це спеціальні анотації, які застосовуються для анотування інших анотацій. Вони надають метадані про самі анотації, визначаючи їхнє призначення, зону дії, життєвий цикл та інші характеристики в контексті JVM та компілятора.

---

> [!abstract] Core Framework
> 
> Мета-анотації дозволяють визначати, як саме анотації повинні поводитися в програмі та як їх слід використовувати. Вони визначають "правила гри":
> 
> - **Зона дії**: де саме в коді можна розмістити анотацію.
>     
> - **Час життя**: як довго анотація залишається доступною (в сирцях, байт-коді чи пам'яті).
>     
> - **Документування**: чи буде анотація частиною публічного API в Javadoc.
>     
> - **Успадкування**: чи будуть підкласи автоматично отримувати анотацію від батьків.
>     

---

> [!info] Key Insights
> 
> - **@Target**: Вказує на типи елементів Java (класи, методи, поля), до яких може бути застосована анотація.
>     
> - **@Retention**: Керує видимістю анотації. Найважливіший рівень — `RUNTIME`, оскільки він дозволяє аналізувати код під час виконання програми.
>     
> - **@Documented**: Гарантує, що використання анотації буде зафіксовано в технічній документації, що важливо для розробників, які використовують ваш API.
>     
> - **@Inherited**: Впливає на ієрархію класів, дозволяючи анотаціям "прокидатися" вниз до нащадків.
>     

---

> [!example] Практичне застосування
> 
> **1. @Target** > Визначає, до яких елементів коду можна застосувати анотацію (методи, класи, поля тощо).
> 
> Java
> 
> ```java
> @Target(ElementType.METHOD) 
> public @interface MyAnnotation {
>     /* Цю анотацію можна поставити ТІЛЬКИ над методом */
> }
> ```
> 
> **2. @Retention** > Вказує, як довго анотація "живе":
> 
> - `SOURCE`: Тільки в сирцевому коді (зникає після компіляції).
>     
> - `CLASS`: Зберігається у `.class` файлі, але JVM її ігнорує (недоступна в Runtime).
>     
> - `RUNTIME`: Доступна під час виконання через Reflection API.
>     
> Java
> 
> ```java
> @Retention(RetentionPolicy.RUNTIME)
> public @interface MyAnnotation {
>     /* Буде доступна під час роботи програми (через рефлексію) */
> }
> ```
> 
> **3. @Documented** > Включає анотацію в документацію Javadoc.
> 
> Java
> 
> ```java
> @Documented
> public @interface MyAnnotation {
>     /* Відобразиться в документації до класу/методу */
> }
> ```
> 
> **4. @Inherited** > Дозволяє анотації успадковуватися підкласами.
> 
> Java
> 
> ```java
> @Inherited
> public @interface MyAnnotation {
>     /* Якщо позначити цією анотацією Class A, то Class B (нащадок) теж її матиме */
> }
> ```

---

> [!tip] Коротке резюме
> 
> |**Мета-анотація**|**Що вона контролює?**|
> |---|---|
> |**@Target**|Де писати (метод, клас, поле)|
> |**@Retention**|Як довго зберігати (код, байт-код, пам'ять)|
> |**@Documented**|Чи показувати в Javadoc|
> |**@Inherited**|Чи передавати нащадкам по спадковості|
