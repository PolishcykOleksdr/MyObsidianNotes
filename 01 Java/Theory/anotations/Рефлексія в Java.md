> **Рефлексія** — це механізм, який дозволяє програмі під час виконання (Runtime) отримувати детальну інформацію про класи, методи, поля та анотації, а також маніпулювати ними. Це процес, за якого програма аналізує власну структуру або взаємодіє з динамічно завантаженим кодом.

---

> [!abstract] Core Framework
> 
> Рефлексія надається через пакет `java.lang.reflect` та клас `java.lang.Class`. Вона дозволяє реалізувати динамічну поведінку, що неможлива при звичайній компіляції:
> 
> - **Отримання метаданих**: доступ до імен, типів, модифікаторів доступу та анотацій.
>     
> - **Маніпулювання об'єктами**: створення екземплярів через `Constructor`, читання/запис значень через `Field`.
>     
> - **Виклик методів**: запуск методів через `Method.invoke()`, навіть якщо їх назви невідомі на етапі написання коду.
>     

---

> [!info] Key Insights
> 
> - **Доступ до private**: Рефлексія дозволяє ігнорувати модифікатори доступу (`setAccessible(true)`), що дає змогу працювати з приватними полями та методами.
>     
> - **Динамічність**: Програма може завантажувати класи за їх текстовим ім'ям, що критично для плагінних архітектур.
>     
> - **Продуктивність**: Операції рефлексії значно повільніші за прямі виклики, оскільки JVM не може застосувати стандартні оптимізації (наприклад, інлайнінг).
>     
> - **Безпека**: Використання рефлексії може порушити інкапсуляцію та цілісність об'єкта, що робить код вразливим до помилок.
>

> [!example] Практичне застосування та приклади
> 
> **1. Spring Framework**
> 
> Використовує рефлексію для сканування пакетів та знаходження компонентів (`@Component`, `@Service`). Це дозволяє контейнеру автоматично створювати об'єкти та впроваджувати залежності.
> 
> **2. Hibernate ORM**
> 
> Сканує класи, анотовані `@Entity`, та через рефлексію відображає поля об'єктів на колонки БД. Дозволяє заповнювати приватні поля об'єктів даними з бази без використання сеттерів.
> 
> **3. JUnit**
> 
> Автоматично знаходить методи з анотацією `@Test` у класі та викликає їх під час виконання тестів, позбавляючи розробника необхідності реєструвати кожен тест вручну.
> 
> **4. Java Reflection API (Базовий приклад)**
> 
> Java
> 
> ```java
> /* Отримання об'єкта класу */
> Class<?> clazz = MyClass.class;
> 
> /* Отримання та виклик методу за ім'ям */
> Method method = clazz.getDeclaredMethod("myMethod");
> 
> method.invoke(myObjectInstance);
> 
> /* Доступ до приватного поля */
> Field field = clazz.getDeclaredField("privateSecret");
> 
> field.setAccessible(true);
> 
> field.set(myObjectInstance, "New Value");
> ```

---

> [!warning] Ризики та обмеження
> 
> - **Погіршення продуктивності**: Витрати ресурсів на аналіз метаданих у Runtime.
>     
> - **Складність підтримки**: Код стає менш прозорим ("магічним"), що ускладнює відлагодження.
>     
> - **Runtime помилки**: Помилки (наприклад, `NoSuchMethodException`) виникають лише під час роботи програми, а не під час компіляції.
>
