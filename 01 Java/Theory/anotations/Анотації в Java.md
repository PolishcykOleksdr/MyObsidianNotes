> Анотації в Java — це спеціальні маркери (метадані), які додаються до елементів програмного коду (класів, методів, полів, параметрів тощо) для надання додаткової інформації про ці елементи або їх обробку компілятором, інструментами чи середовищем виконання.

---

> [!abstract] Core Framework Основна мета анотацій — надання метаінформації, яка може використовуватися для конфігурування, створення документації, визначення контрактів коду та керування поведінкою програми в різних контекстах (тестування, веб-розробка). Вони визначаються ключовим словом `@interface`, а застосовуються через символ `@` перед назвою анотації.

--------------------------------------------------------------------------------

> [!info] Key Insights Глибока логіка анотацій базується на механізмі **Retention Policy**, який визначає "час життя" метаданих у пам'яті та їх доступність для JVM:
> 
> - **RetentionPolicy.SOURCE**: Метадані існують лише у файлі `.java`. Вони ігноруються компілятором під час створення байт-коду і не потрапляють у файл `.class`.
> - **RetentionPolicy.CLASS**: Анотації зберігаються в байт-коді як частина метаданих класу, але вони не завантажуються в пам'ять JVM під час виконання програми.
> - **RetentionPolicy.RUNTIME**: Анотації зберігаються в байт-коді та стають доступними через Reflection API під час виконання програми, що дозволяє динамічно зчитувати їх значення.
>

## 🛠 Практичне застосування: @Deprecated

Використання стандартної анотації для маркування елементів, які більше не рекомендуються до використання.

```java
@Deprecated
public class MyClass {
    // Список методів та полів класу
}
```

## ⏱ Політики утримання (RetentionPolicy)

Рівень "життя" анотації визначає, на якому етапі вона буде доступна: від сирцевого коду до моменту виконання.

### 1. RetentionPolicy.SOURCE

Анотація доступна лише на етапі написання коду; ігнорується компілятором.

```java
@Retention(RetentionPolicy.SOURCE)
@interface MyAnnotation {
    String value();
}

public class MyClass {
    @MyAnnotation("Видно лише у файлі .java")
    public void myMethod() { }
}
```

### 2. RetentionPolicy.CLASS

Зберігається у `.class` файлі (байт-коді), але недоступна через Reflection під час виконання. **Це значення за замовчуванням.**

```java
@Retention(RetentionPolicy.CLASS)
@interface MyAnnotation {
    String value();
}
```

### 3. RetentionPolicy.RUNTIME

Найпотужніший тип. Дозволяє динамічно зчитувати дані анотації під час роботи програми.

```java
@Retention(RetentionPolicy.RUNTIME)
@interface MyRuntimeAnnotation {
    String value();
}
```

## 🎯 Target: Куди можна ставити анотації?

### Анотування типів (Типи даних)

Надання метаінформації для класів, інтерфейсів та перерахувань (`ElementType.TYPE`).

```java
@Retention(RetentionPolicy.RUNTIME) 
@Target(ElementType.TYPE) 
@interface ClassInfo { 
    String author(); 
    String date(); 
}

@ClassInfo(author = "John Doe", date = "2026-03-07") 
public class MyClass { }

@Target(ElementType.TYPE) 
@interface EnumInfo { String description(); }

@EnumInfo(description = "Рівні доступу") 
public enum AccessLevel { ADMIN, GUEST }
```

### Анотування членів класу та параметрів

Опис полів (`FIELD`), методів (`METHOD`), конструкторів та їхніх аргументів.

```java
public class Example {
    @FieldInfo(name = "id", size = 10)
    private int id;

    @ConstructorInfo(author = "Admin")
    public Example(@ParameterInfo(description = "ID value") int id) {
        this.id = id;
    }

    @MethodInfo(author = "John")
    public void processData() { }
}
```

### Специфічні цілі

- **LOCAL_VARIABLE**: Локальні змінні всередині методів.
    
- **PACKAGE**: Визначається у файлі `package-info.java`.
    
- **ANNOTATION_TYPE**: Створення мета-анотацій (анотацій для анотацій).

## 📊 Порівняльна характеристика

|**Характеристика**|**Опис**|
|---|---|
|✅ **Метаінформація**|Зручне документування автора, версії та дати безпосередньо в коді.|
|✅ **Гнучкість**|Три рівні збереження дозволяють оптимізувати продуктивність.|
|✅ **Автоматизація**|База для роботи Spring, Hibernate та інструментів генерації коду (Lombok).|
|❌ **Обмеження Reflection**|Анотації `SOURCE` та `CLASS` неможливо дістати під час Runtime.|
|❌ **Рідкісні кейси**|Анотування пакетів та локальних змінних важко піддається автоматичній обробці.|

---

> [!WARNING] Важливо пам'ятати
> 
> Хоча ви можете анотувати локальні змінні, стандартна Java Reflection **не дозволяє** отримати їх значення під час виконання програми. Це зазвичай використовується лише інструментами статичного аналізу коду (наприклад, Checkstyle).
