> Кастомні анотації — це анотації, які розробник створює самостійно для позначення елементів програми або надання їм додаткових метаданих. Вони дозволяють впроваджувати власні правила, які можна застосовувати до класів, методів, полів тощо.

---

> [!abstract] Core Framework
> 
> Для створення кастомної анотації використовується ключове слово `@interface` у поєднанні з мета-анотаціями, що керують її поведінкою.
> 
> Основні операції при роботі з кастомними анотаціями:
> 
> - **Визначення структури**: оголошення полів (методів) анотації та значень за замовчуванням.
>     
> - **Конфігурація**: використання мета-анотацій (`@Retention`, `@Target`) для налаштування доступності.
>     
> - **Маркування**: застосування створеної анотації до цільових елементів коду.
>     

---

> [!info] Key Insights
> 
> - **Метадані в коді**: Анотації дозволяють вбудовувати інформацію безпосередньо в сирцевий код, що полегшує його організацію.
>     
> - **Рефлексія (Reflection)**: Саме завдяки анотаціям, доступним у `RUNTIME`, програма може "читати" саму себе під час виконання, отримуючи додаткові інструкції.
>     
> - **Архітектурний інструмент**: Кастомні анотації є фундаментом для Dependency Injection (ін'єкції залежностей) та аспектно-орієнтованого програмування.
>     
> - **Валідація**: Можливість автоматизованої перевірки вхідних параметрів або станів об'єктів на відповідність певним умовам.
>     

---

> [!example] Практичне застосування
> 
> **Створення кастомної анотації:**
> 
> Java
> 
> ```java
> import java.lang.annotation.ElementType;
> import java.lang.annotation.Retention;
> import java.lang.annotation.RetentionPolicy;
> import java.lang.annotation.Target;
> ```
> @Retention(RetentionPolicy.RUNTIME) - Доступна під час виконання через Reflection API @Target(ElementType.METHOD) - Можна застосовувати тільки до методів _
> ```java
> public @interface MyCustomAnnotation {
>  String value() default ""; - Поле анотації зі значенням за замовчуванням _
>  int priority() default 0; - Інше поле анотації 
>  }
> ```
> 
> ```java
> **Використання анотації в класі:**
> public class MyClass {
>     @MyCustomAnnotation(value = "Some value", priority = 1)
>     public void myMethod() {
>         /* Тіло методу, позначене метаданими */
>     }
> }
> ```

> [!tip] Переваги використання
> 
> |**✅ Переваги**|**Опис**|
> |---|---|
> |**Читабельність**|Слугують вбудованою документацією про призначення коду.|
> |**Гнучкість архітектури**|Реалізація складних паттернів (DI, AOP) у зручний спосіб.|
> |**Валідація**|Перевірка полів та параметрів на відповідність бізнес-логіці.|
> |**Екосистема**|Легка інтеграція зі сторонніми фреймворками через конфігурацію анотаціями.|
