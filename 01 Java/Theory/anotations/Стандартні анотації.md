> Стандартні анотації в Java — це вбудовані в мову маркери, які надають певні метадані для програми на етапах компіляції, виконання або використання сторонніми інструментами розробки.

---

> [!abstract] Core Framework
>  Основна мета стандартних анотацій полягає у покращенні якості, читабельності та безпеки коду. Вони діють як інструкції для компілятора (для виявлення помилок чи придушення попереджень) або як інформаційні вказівки для користувачів API.

---

> [!info] Key Insights Глибока логіка роботи стандартних анотацій базується на взаємодії з компілятором та інструментами аналізу:
> 
> - **Валідація контрактів**: Анотації на кшталт `@Override` активують механізми перевірки відповідності сигнатур методів у ієрархії успадкування під час компіляції.
> - **Керування попередженнями**: Шляхом використання анотацій можна вибірково керувати діагностичними повідомленнями компілятора, ігноруючи специфічні типи зауважень (наприклад, "unchecked").
> - **Маркування життєвого циклу**: Анотації дозволяють позначати елементи коду, які готуються до видалення, забезпечуючи зворотний зв'язок для розробників через попередження.

> [!example] Практичне застосування анотацій Java
> 
> Анотації допомагають компілятору та розробнику краще розуміти структуру коду, запобігаючи помилкам на етапі написання.
> 
> ### 1. Перевизначення методу (`@Override`)
> 
> Використовується для інформування компілятора про намір змінити логіку базового методу. Це запобігає помилкам, якщо сигнатура методу була вказана неправильно (наприклад, помилка в назві або типах аргументів).
> 
> Java
> 
> ```java
> @Override
> public void someMethod() {
>     /* Перевизначений метод */
> }
> ```
> 
> ### 2. Застарілі елементи (`@Deprecated`)
> 
> Позначає класи, методи або поля, які вважаються застарілими. Це слугує попередженням для інших розробників, що в майбутніх версіях ці елементи можуть бути видалені, і варто шукати альтернативу.
> 
> Java
> 
> ```java
> @Deprecated
> public void oldMethod() {
>     /* Старий метод */
> }
> ```
> 
> ### 3. Придушення попереджень (`@SuppressWarnings`)
> 
> Інструктує компілятор ігнорувати визначені типи попереджень (warnings) під час обробки конкретного фрагмента коду. Це корисно, коли ви свідомо використовуєте "небезпечний" код (наприклад, старі бібліотеки без Generic-типів).
> 
> Java
> 
> ```java
> @SuppressWarnings("unchecked")
> List list = new ArrayList();
> ```
> 
> ### 4. Функціональні інтерфейси (`@FunctionalInterface`)
> 
> Застосовується до інтерфейсів, які містять **рівно один** абстрактний метод. Це гарантує, що інтерфейс можна використовувати для Lambda-виразів.
> 
> Java
> 
> ```java
> @FunctionalInterface
> public interface MyFunctionalInterface {
>     void doSomething();
> }
> ```
---

> **Порада:** Використання `@Override` є правилом "хорошого тону" в Java. Навіть якщо метод перевизначається автоматично, ця анотація робить код значно зрозумілішим для ваших колег.

---

> [!warning] Обмеження та ризики
> 
> - **Ризик видалення**: Використання елементів з анотацією `@Deprecated` може призвести до проблем у майбутньому через їх нестабільність у нових версіях API.
> - **Помилки компіляції**: Якщо метод, позначений `@Override`, не знайдено у базовому класі, програма не буде скомпільована.
> - **Приховування проблем**: Надмірне використання `@SuppressWarnings` може призвести до ігнорування важливих попереджень, що потенційно впливають на безпеку типів.

---

|**✅ Переваги**|**❌ Недоліки / Ризики**|
|---|---|
|**Виявлення помилок** на етапі компіляції (завдяки `@Override`)|**Ризик використання нестабільного API** при ігноруванні `@Deprecated`|
|**Підвищення безпеки типів** та загальної якості коду|**Приховування реальних проблем** через надмірне використання `@SuppressWarnings`|
|**Покращення читабельності** та полегшення підтримки коду колегами|**Необхідність суворого дотримання сигнатур** для коректного перевизначення
