```java
public class Test {  
    public static void main(String[] args) {  
        ((Test) null).call();  
  
        Test test = null;  
        test.call();  
        call();  
    }  
    static void call(){  
        System.out.println("Привіт..");  
    }  
}
```

```
Output:
Hello..
Hello..
Hello..
```

---

>[!question] Чому це працює?
> 
> В Java, коли ви викликаєте `static` метод, компілятору байдуже, чи є посилання (reference) порожнім (`null`). Він використовує **тип посилання**, а не сам об'єкт у пам'яті, щоб знайти метод.

> [!note] Ось що відбувається за лаштунками для кожного вашого виклику:
> 1. **`((Test) null).call();`**
>     Ви явно кажете компілятору: "Тут є щось типу `Test`". Оскільки `call()` — це статичний метод, Java просто дивиться на тип (`Test`) і викликає метод безпосередньо з класу. Сама адреса в пам'яті (яка тут `null`) навіть не перевіряється.
> 2. **`test.call();` (де `test` це `null`)**
>     Це виглядає як звернення до об'єкта, але для компілятора це лише "синтаксичний цукор". Він бачить, що змінна `test` має тип `Test`, знає, що `call()` — статичний, і перетворює цей виклик на `Test.call()`.
> 3. **`call();`**
>     Це класичний виклик статичного методу всередині того ж класу.

---
### Як це бачить JVM

Коли ви пишете `test.call()`, під час компіляції це фактично перетворюється на звернення до класу:

|**Тип методу**|**Як викликається**|**Чи перевіряється null?**|
|---|---|---|
|**Static**|Через клас (`ClassName.method()`)|**Ні**. Об'єкт не потрібен.|
|**Instance** (звичайний)|Через об'єкт у купі (heap)|**Так**. Якщо об'єкт `null` — буде помилка.|

---

Хоча Java дозволяє викликати статичні методи через екземпляри (як у вашому прикладі), це вважається **дуже поганою практикою**. Це заплутує розробників, створюючи ілюзію, що стан об'єкта має значення.
