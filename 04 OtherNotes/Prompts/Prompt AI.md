# Загальне про тему
Приклад:
> **Колекції (Collections)** – це **сховища** або **контейнери**, що підтримують різні способи накопичення та впорядкування об'єктів з метою забезпечення можливостей ефективного доступу до них.

# Core Framework (завжди у [!abstract] та огортай через >)
Приклад:

> [!abstract] Core Framework
> **Колекції є реалізацією абстрактних структур даних, що підтримують основні операції:**
> - додавання нового елемента до колекції;
> - видалення елемента з колекції;
> - зміна елемента в колекції.

---
# Key Insights (завжди у [!info] та огортай через >)
Приклад:
> [!info] Key Insights
> 
> - **Вхідний захист.** У конструкторі ми копіюємо дані, щоб зовнішній код не міг змінити їх пізніше через своє посилання.
>     
> - **Вихідний захист.** У геттері ми повертаємо копію, щоб той, хто викликав метод, не зміг видалити або змінити елементи нашого внутрішнього списку.
>     
> - **Справжня імутабельність.** Без захисного копіювання клас із полями-колекціями (як `List`) ніколи не буде по-справжньому незмінним.
>     
> - **Витрати.** Створення копій споживає пам'ять, тому для величезних масивів даних шукають компроміси (наприклад, `Collections.unmodifiableList`).

# Практичне застосування (завжди у [!example] та огортай через >)
Приклад:
> [!example] Практичне застосування
> 
> **Послідовна обробка:**  
> ```java
> list.stream().filter(x -> x > 10).map(x -> x * 2).collect(Collectors.toList())
> ```
> 
> **Паралельна обробка:**  
> ```java
> list.parallelStream()
> ``` 
> — та сама логіка, але Java автоматично розподіляє роботу між ядрами процесора.
> 
> **Різниця:** Паралельний підхід прискорює обробку великих обсягів даних, але додає накладні витрати на синхронізацію. Використовуйте для колекцій з 1000+ елементів.

# Або у вигляді: (завжди у трьох ` (якщо код то +java до них після)) 

## Проміжні Операції:

## 1. **filter(Predicate<T\> predicate)**

Відбір елементів за певною умовою.

```java
List<String> fruits = Arrays.asList("apple", "orange", "banana", "grape");

List<String> filteredFruits = fruits.stream()
                                   .filter(fruit -> fruit.length() > 5)
                                   .collect(Collectors.toList());
// Результат: ["orange", "banana"]
```

---
## 2. **map(Function<T, R> mapper)**

Трансформація елементів за допомогою заданої функції.

```java
List<String> fruits = Arrays.asList("apple", "orange", "banana", "grape");

List<String> upperCaseFruits = fruits.stream()
                                     .map(String::toUpperCase)
                                     .collect(Collectors.toList());
// Результат: ["APPLE", "ORANGE", "BANANA", "GRAPE"]
```

# Важливе (завжди у [!warning] та огортай через >)
Приклад:
> [!warning] Обмеження та ризики
> 
> - **Важкість відлагодження.** У стек-трейсі помилка в лямбді виглядає як анонімний виклик, що важче ідентифікувати.
>     
> - **Тільки effectively final.** Лямбда може використовувати змінні ззовні, але не може їх змінювати.
>     
> - **Читабельність.** Занадто довга лямбда (більше 3-5 рядків) перетворює код на "макарони". Краще винести таку логіку в окремий метод.
